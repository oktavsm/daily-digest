## ⚡ Competitive Programming Concept: Segment Tree (Pohon Segmen)

**Category:** Data Structure
**TL;DR:** Segment Tree adalah struktur data pohon yang memungkinkan query pada rentang dan update elemen secara efisien dalam waktu logaritmik.

Segment Tree adalah struktur data pohon yang sangat fleksibel dan sering digunakan dalam competitive programming untuk menyelesaikan masalah yang melibatkan operasi query pada rentang (range query) dan pembaruan (update) pada elemen-elemen dalam sebuah array. Contoh umum adalah mencari nilai minimum/maksimum dalam suatu rentang, atau menghitung jumlah elemen dalam rentang. Keunggulan utamanya adalah kemampuannya untuk melakukan kedua jenis operasi ini dengan efisiensi waktu logaritmik, menjadikannya pilihan ideal untuk masalah dengan banyak query dan update.

Struktur Segment Tree dibangun di atas sebuah array. Setiap node dalam pohon mewakili sebuah segmen atau rentang dari array asli. Node root mewakili seluruh rentang array, dan setiap node internal memiliki dua child yang masing-masing mewakili setengah dari rentang node parent-nya. Proses ini berlanjut hingga node leaf, yang masing-masing mewakili sebuah elemen tunggal dari array. Dengan demikian, Segment Tree memiliki tinggi logaritma dari ukuran array, yaitu O(log N).

Operasi `build` untuk membangun Segment Tree dari array awal membutuhkan waktu O(N) karena setiap elemen array dikunjungi sekali. Operasi `query` untuk mencari informasi pada rentang `[L, R]` (misalnya, sum atau min) membutuhkan waktu O(log N). Demikian pula, operasi `update` untuk mengubah nilai pada indeks tertentu atau bahkan rentang tertentu (dengan lazy propagation) juga memiliki kompleksitas waktu O(log N). Ini membuat Segment Tree sangat efisien untuk skenario di mana banyak operasi perlu dilakukan pada array.

```
/* Asumsi 'arr' adalah array asli, 'tree' adalah array Segment Tree. */
/* Asumsi 'node' adalah indeks node saat ini, [L, R] adalah rentang node. */
/* Asumsi [qL, qR] adalah rentang query yang diinginkan. */

int query_sum(int node, int L, int R, int qL, int qR) {
    if (qL > R || qR < L) return 0; // Tidak ada tumpang tindih
    if (qL <= L && R <= qR) return tree[node]; // Tumpang tindih penuh

    int mid = (L + R) / 2;
    int p1 = query_sum(2 * node, L, mid, qL, qR);
    int p2 = query_sum(2 * node + 1, mid + 1, R, qL, qR);
    return p1 + p2;
}
```

💡 **Fun Fact:** Segment Tree seringkali bisa digantikan oleh Fenwick Tree (BIT) jika hanya dibutuhkan operasi sum/prefix sum dan update titik, namun Segment Tree lebih fleksibel untuk berbagai jenis query dan operasi pada rentang (misalnya min/max, GCD, dll.).
