## 🌐 Networking Concept: EtherChannel: Agregasi Link Fisik

**Category:** Switching
**TL;DR:** Menggabungkan beberapa port switch fisik menjadi satu link logis untuk meningkatkan bandwidth dan ketersediaan, menghindari loop STP.

Di jaringan modern, bandwidth antar switch atau antara switch dan server seringkali menjadi bottleneck. Selain itu, kegagalan satu link fisik dapat menyebabkan gangguan layanan. EtherChannel, atau disebut juga Link Aggregation Control Protocol (LACP) atau Port Aggregation Protocol (PAgP) oleh Cisco, adalah teknologi yang memungkinkan Anda menggabungkan (bundling) beberapa link Ethernet fisik menjadi satu link logis. Ini menciptakan sebuah "port-channel" virtual yang meningkatkan total bandwidth yang tersedia dan menyediakan redundansi otomatis jika salah satu link fisik gagal.

Ketika beberapa port fisik dikonfigurasi sebagai EtherChannel, switch akan memperlakukannya sebagai satu antarmuka logis. Hal ini memiliki beberapa keuntungan signifikan. Pertama, bandwidth total akan menjadi jumlah bandwidth dari semua link fisik yang tergabung (misalnya, dua link Gigabit Ethernet akan menjadi satu link 2 Gbps logis). Kedua, EtherChannel menyediakan redundansi: jika satu link fisik dalam grup gagal, traffic akan secara otomatis didistribusikan ke link-link lain yang masih aktif dalam grup tanpa downtime. Ketiga, dari sudut pandang Spanning Tree Protocol (STP), EtherChannel hanya dianggap sebagai satu port logis, yang mencegah terjadinya loop STP tanpa memblokir link fisik tambahan yang bermanfaat.

EtherChannel dapat diimplementasikan menggunakan beberapa mode. LACP (Link Aggregation Control Protocol) adalah standar industri (IEEE 802.3ad) dan menawarkan mode "active" (selalu mencoba membentuk EtherChannel) dan "passive" (hanya membentuk EtherChannel jika switch di ujung lain bernegosiasi). PAgP (Port Aggregation Protocol) adalah proprietary Cisco, dengan mode "desirable" (mencoba membentuk) dan "auto" (hanya membentuk jika switch di ujung lain desirable). Ada juga mode "on" yang memaksa EtherChannel terbentuk tanpa negosiasi, namun kurang disarankan karena tidak ada mekanisme verifikasi link. Konfigurasi yang tepat memastikan semua link fisik memiliki karakteristik yang sama (misalnya, mode duplex, kecepatan, VLAN) agar dapat bergabung dalam EtherChannel.

```
channel-group 1 mode active
```

💡 **Fun Fact:** EtherChannel dapat menggabungkan hingga delapan link fisik aktif, ditambah hingga delapan link cadangan, semuanya dianggap sebagai satu port logis oleh Spanning Tree Protocol (STP), secara signifikan menyederhanakan topologi.
