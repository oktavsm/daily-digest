## ⚡ Competitive Programming Concept: Segment Tree Range Query Update

**Category:** Data Structure
**TL;DR:** Segment Tree adalah struktur data pohon yang efisien untuk melakukan query agregat (seperti sum, min, max) dan update elemen pada range tertentu dari sebuah array.

Segment Tree adalah struktur data berbasis pohon biner yang digunakan untuk menyimpan informasi tentang interval atau "segmen". Tujuannya adalah untuk secara efisien melakukan operasi query pada sub-range (misalnya, mencari jumlah, nilai minimum, atau maksimum dalam rentang indeks tertentu) dan juga melakukan operasi update pada elemen individual atau rentang elemen dalam array dasar. Setiap node dalam Segment Tree merepresentasikan sebuah interval, di mana node root merepresentasikan seluruh array, dan node anak merepresentasikan setengah dari interval parent-nya, secara rekursif hingga node daun yang merepresentasikan elemen tunggal.

Operasi query, misalnya mencari jumlah dalam rentang `[L, R]`, bekerja dengan memecah rentang query menjadi beberapa interval yang direpresentasikan oleh node-node di pohon. Algoritma akan melakukan traversal pohon, dimulai dari root. Jika interval node saat ini sepenuhnya berada dalam rentang `[L, R]`, nilai agregat dari node tersebut akan langsung dikembalikan. Jika tidak, operasi akan secara rekursif mencari di node anak kiri dan/atau kanan yang beririsan dengan `[L, R]`, lalu menggabungkan hasilnya. Proses ini memastikan bahwa hanya node-node yang relevan yang diperiksa.

Operasi update, seperti mengubah nilai elemen pada indeks `idx`, juga dilakukan dengan traversal pohon dari root ke daun. Ketika mencapai node daun yang merepresentasikan `idx`, nilainya diperbarui. Setelah itu, algoritma akan kembali ke atas (backtrack) ke node-node parent, memperbarui nilai agregatnya berdasarkan nilai anak-anak yang baru. Misalnya, jika Segment Tree menyimpan jumlah, node parent akan mengambil jumlah dari anak kiri dan kanannya. Baik operasi query maupun update memiliki kompleksitas waktu **O(log N)**, di mana N adalah ukuran array, karena setiap operasi hanya menelusuri jalur dari root ke leaf (atau jalur yang serupa).

Pembangunan Segment Tree awal membutuhkan waktu **O(N)** karena setiap node dikunjungi sekali untuk menghitung nilai agregatnya. Sementara itu, kompleksitas ruang untuk Segment Tree adalah **O(N)**, karena pohon biner yang lengkap atau mendekati lengkap untuk N elemen membutuhkan sekitar 2N hingga 4N node tergantung implementasi dan padding.

```
int N_arr[100], N_tree[400];
void build(int n, int s, int e) {
    if (s == e) N_tree[n] = N_arr[s];
    else { int m = (s + e) / 2; build(2*n, s, m); build(2*n+1, m+1, e); N_tree[n] = N_tree[2*n] + N_tree[2*n+1]; }
}
void update(int n, int s, int e, int idx, int val) {
    if (s == e) { N_arr[idx] = val; N_tree[n] = val; }
    else { int m = (s + e) / 2; (s <= idx && idx <= m) ? update(2*n, s, m, idx, val) : update(2*n+1, m+1, e, idx, val); N_tree[n] = N_tree[2*n] + N_tree[2*n+1]; }
}
int query(int n, int s, int e, int L, int R) {
    if (R < s || e < L) return 0;
    if (L <= s && e <= R) return N_tree[n];
    int m = (s + e) / 2; return query(2*n, s, m, L, R) + query(2*n+1, m+1, e, L, R);
}
```

💡 **Fun Fact:** Segment Tree sering dianggap sebagai salah satu struktur data "Swiss Army Knife" dalam competitive programming karena fleksibilitasnya. Variasi seperti Lazy Propagation memungkinkan update pada seluruh rentang dengan efisiensi yang sama, membuka pintu untuk memecahkan lebih banyak masalah kompleks.
