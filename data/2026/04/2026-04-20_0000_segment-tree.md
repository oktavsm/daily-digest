## ⚡ Competitive Programming Concept: Segment Tree

**Category:** Data Structure
**TL;DR:** Segment Tree adalah struktur data pohon biner yang efisien untuk melakukan query pada range (misalnya mencari sum, min, max) dan update nilai pada array dalam waktu logaritmis.

Segment Tree adalah struktur data pohon biner yang digunakan untuk menyimpan informasi tentang interval atau segmen dari sebuah array. Tujuan utamanya adalah untuk memungkinkan operasi query yang cepat pada rentang tertentu (seperti mencari nilai minimum, maksimum, atau jumlah dalam subarray) dan update nilai individual atau rentang dalam array. Tanpa Segment Tree, operasi-operasi ini mungkin memerlukan waktu linier (O(N)), tetapi dengan Segment Tree, kompleksitas waktu dapat dikurangi secara signifikan menjadi logaritmis (O(log N)). Hal ini menjadikannya alat yang sangat berguna dalam masalah competitive programming di mana ada banyak query dan update pada array yang besar.

Setiap node dalam Segment Tree merepresentasikan sebuah interval dalam array asli. Node root merepresentasikan seluruh array dari indeks 0 hingga N-1. Setiap node internal kemudian dibagi menjadi dua child, di mana masing-masing child merepresentasikan setengah dari interval node parent-nya. Proses ini berlanjut secara rekursif hingga mencapai node daun, yang masing-masing merepresentasikan sebuah elemen tunggal dalam array. Saat membangun Segment Tree, kita mulai dari node daun dan naik ke atas, menggabungkan hasil dari child-node untuk mengisi nilai pada parent-node. Misalnya, untuk Segment Tree yang mencari jumlah, node parent akan menyimpan jumlah dari interval child-nya. Proses pembangunan awal (initial build) Segment Tree ini membutuhkan waktu O(N), di mana N adalah ukuran array.

Untuk melakukan operasi query pada range tertentu `[L, R]`, kita menelusuri pohon dari root. Kita membandingkan interval node saat ini dengan range query yang diminta. Jika interval node sepenuhnya berada di dalam range query, kita mengembalikan nilai node tersebut. Jika ada tumpang tindih sebagian, kita secara rekursif mencari di child-child node dan menggabungkan hasilnya. Jika tidak ada tumpang tindih sama sekali, kita mengembalikan nilai identitas (misalnya 0 untuk sum, atau infinity untuk min). Untuk melakukan operasi update pada sebuah elemen di indeks `idx`, kita menelusuri pohon dari root hingga node daun yang merepresentasikan `idx`, mengubah nilainya, lalu kembali ke atas, memperbarui nilai-nilai pada node parent yang terpengaruh. Baik operasi query maupun update pada Segment Tree memiliki kompleksitas waktu O(log N) karena tinggi pohon logaritmis terhadap ukuran array, membuatnya sangat efisien.

```
int tree[4 * MAX_N]; // Ukuran sekitar 4 kali MAX_N
int arr[MAX_N];

// Fungsi untuk query range sum
int query(int node, int start, int end, int l, int r) {
    if (r < start || end < l) { // Segmen node di luar range query
        return 0; // Nilai identitas untuk penjumlahan
    }
    if (l <= start && end <= r) { // Segmen node sepenuhnya di dalam range query
        return tree[node];
    }
    // Tumpang tindih sebagian, rekursif ke child
    int mid = (start + end) / 2;
    int p1 = query(2 * node, start, mid, l, r);
    int p2 = query(2 * node + 1, mid + 1, end, l, r);
    return p1 + p2;
}
```

💡 **Fun Fact:** Segment trees sering kali dilengkapi dengan teknik 'lazy propagation' untuk mengoptimalkan update pada seluruh range sekaligus, di mana perubahan ditunda dan diterapkan hanya ketika sebuah node benar-benar diakses oleh query. Ini memperluas kemampuannya untuk menangani jenis masalah yang lebih kompleks.
