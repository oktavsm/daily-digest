## ☕ Java Concept: Penanganan Exception (try-catch)

**Category:** Exception
**TL;DR:** Mencegah program error dengan menangani kesalahan yang tak terduga menggunakan blok `try` dan `catch`.

Di dunia pemrograman, seringkali terjadi hal-hal di luar kendali kita, seperti input pengguna yang tidak valid, file yang tidak ditemukan, atau masalah jaringan. Ketika kesalahan-kesalahan semacam ini terjadi, program Java Anda bisa berhenti secara tiba-tiba (crash) jika tidak ditangani dengan baik. Konsep "Exception Handling" hadir untuk mencegah hal ini, memungkinkan program kita untuk "memulihkan diri" atau setidaknya memberikan pesan yang informatif kepada pengguna alih-alih langsung crash. Ini membuat aplikasi kita lebih tangguh dan mudah digunakan.

Penanganan exception di Java umumnya dilakukan menggunakan blok `try` dan `catch`. Blok `try` adalah tempat Anda menempatkan kode yang Anda curigai berpotensi menyebabkan exception. Jika ada exception yang terjadi di dalam blok `try`, eksekusi kode di dalam blok `try` akan segera dihentikan, dan kendali akan dialihkan ke blok `catch` yang sesuai. Ini memastikan bahwa meskipun ada masalah, program tidak berhenti begitu saja.

Setelah exception terjadi di blok `try`, Java akan mencari blok `catch` yang mampu menangani jenis exception tersebut. Setiap blok `catch` mendeklarasikan jenis exception yang bisa ditanganinya (misalnya, `ArithmeticException` untuk pembagian dengan nol, atau `NumberFormatException` untuk mengonversi teks ke angka yang gagal). Di dalam blok `catch`, Anda bisa menulis kode untuk merespons exception tersebut, misalnya menampilkan pesan kesalahan kepada pengguna, mencatat error untuk debugging, atau mencoba tindakan alternatif untuk melanjutkan program.

Dengan menggunakan `try-catch`, kita tidak hanya membuat program kita lebih stabil, tetapi juga meningkatkan pengalaman pengguna. Daripada melihat program tiba-tiba tertutup, pengguna akan mendapatkan pesan yang jelas tentang apa yang salah dan mungkin cara memperbaikinya. Ini adalah praktik fundamental untuk menulis aplikasi Java yang profesional dan andal, dan akan sangat sering Anda temui serta gunakan dalam pengembangan selanjutnya.

```
public class ExceptionDemo {
    public static void main(String[] args) {
        try {
            int result = 10 / 0; // Ini akan menyebabkan ArithmeticException
            System.out.println("Hasil: " + result); // Baris ini tidak akan dieksekusi
        } catch (ArithmeticException e) {
            System.err.println("Terjadi kesalahan aritmatika: " + e.getMessage());
            System.err.println("Tidak bisa membagi dengan nol!");
        }
        System.out.println("Program selesai dengan baik.");
    }
}
```

💡 **Fun Fact:** Di Java, ada dua kategori utama exception: Checked Exceptions (yang harus ditangani atau dideklarasikan) dan Unchecked Exceptions (seperti `RuntimeException` yang tidak wajib ditangani). Desain ini dimaksudkan untuk membantu developer mengidentifikasi potensi masalah serius di awal pengembangan.
