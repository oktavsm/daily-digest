## ⚡ Competitive Programming Concept: Segment Tree

**Category:** Data Structure
**TL;DR:** Segment Tree adalah struktur data pohon yang efisien untuk melakukan query pada rentang (range query) dan update pada titik tunggal (point update) atau rentang (range update) pada sebuah array.

Segment Tree adalah struktur data berbentuk pohon biner yang digunakan untuk menyimpan informasi tentang interval atau segmen dari sebuah array. Setiap node dalam pohon merepresentasikan suatu rentang (interval) dari array asli. Node root mewakili seluruh array, dan node-node anaknya masing-masing mewakili setengah bagian dari rentang node induknya, terus menerus hingga ke node daun yang mewakili elemen tunggal dari array. Dengan struktur ini, kita bisa melakukan query seperti mencari nilai minimum, maksimum, atau jumlah dalam suatu rentang [L, R] dan juga melakukan update pada elemen tertentu atau seluruh rentang.

Kelebihan utama Segment Tree terletak pada efisiensinya. Operasi membangun pohon (build) membutuhkan waktu O(N), di mana N adalah ukuran array. Untuk query pada rentang tertentu, seperti mencari sum atau minimum, dan juga untuk point update (mengubah nilai pada indeks tertentu), kompleksitas waktunya adalah O(log N). Ini jauh lebih cepat daripada O(N) jika kita melakukannya secara naif untuk setiap query.

Segment Tree sangat serbaguna dan dapat dimodifikasi untuk mendukung berbagai jenis operasi. Misalnya, selain point update, kita juga bisa melakukan range update (update semua elemen dalam rentang [L, R] dengan nilai tertentu). Teknik ini seringkali membutuhkan tambahan yang dikenal sebagai 'lazy propagation' untuk menjaga kompleksitas waktu O(log N) per operasi. Kemampuan untuk melakukan range query dan update dengan efisiensi logaritmik menjadikannya pilihan utama untuk banyak masalah di competitive programming.

Implementasi Segment Tree biasanya menggunakan array untuk merepresentasikan pohon. Jika sebuah node merepresentasikan rentang [L, R] dan disimpan di indeks `idx` dalam array representasi pohon, maka anak kirinya akan merepresentasikan [L, (L+R)/2] dan disimpan di `2*idx`, sementara anak kanannya merepresentasikan [(L+R)/2 + 1, R] dan disimpan di `2*idx + 1`. Ukuran array representasi pohon biasanya sekitar 4N untuk menampung semua node yang mungkin. Ini menjadikannya alat yang sangat penting dalam kompetisi pemrograman untuk masalah yang melibatkan query dan update rentang.



💡 **Fun Fact:** Segment Tree pertama kali diperkenalkan oleh Jon Bentley pada tahun 1977 sebagai bagian dari risetnya tentang range queries. Meskipun namanya "Segment Tree", struktur ini sebenarnya lebih mirip dengan "Interval Tree" dalam beberapa konteks.
