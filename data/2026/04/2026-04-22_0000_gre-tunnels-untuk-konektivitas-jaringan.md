## 🌐 Networking Concept: GRE Tunnels untuk Konektivitas Jaringan

**Category:** Routing
**TL;DR:** GRE tunnel mengenkapsulasi paket IP dalam paket IP lain untuk membuat jalur komunikasi point-to-point virtual melalui jaringan IP.

Hari ini kita akan menjelajahi konsep Generic Routing Encapsulation (GRE) Tunnels. GRE adalah protokol tunneling yang dikembangkan oleh Cisco yang memungkinkan enkapsulasi berbagai protokol jaringan di dalam paket IP. Bayangkan Anda memiliki dua kantor cabang yang terpisah jauh, masing-masing dengan jaringan internalnya sendiri, dan Anda ingin membuat mereka berkomunikasi seolah-olah mereka berada di satu jaringan yang sama, melewati internet publik yang tidak aman atau jaringan MPLS provider. GRE menyediakan cara untuk menciptakan jalur komunikasi point-to-point virtual antara dua router.

Cara kerja GRE cukup sederhana: ia mengambil seluruh paket IP (termasuk header dan payload aslinya) dari jaringan sumber, menambahkan header GRE, lalu menambahkan header IP baru di depannya. Paket "hasil" ini kemudian dikirim melalui jaringan fisik (seperti internet) ke router tujuan. Router tujuan akan menghapus header IP dan GRE tambahan, mengungkapkan kembali paket IP asli, dan meneruskannya ke tujuan akhirnya di jaringan lokalnya. Ini memungkinkan protokol routing dinamis seperti OSPF atau EIGRP berjalan di atas tunnel GRE, menghubungkan domain routing yang terpisah.

Untuk mengimplementasikan GRE di Cisco Packet Tracer, Anda perlu mengkonfigurasi interface tunnel virtual pada router. Interface tunnel ini memerlukan alamat IP sendiri (biasanya dari subnet yang terpisah dan unik untuk tunnel), alamat IP sumber (interface fisik router yang menghadap jaringan eksternal), dan alamat IP tujuan (interface fisik router peer di ujung tunnel yang lain). Setelah tunnel terbentuk, Anda bisa mengkonfigurasi routing (misalnya static atau OSPF) untuk mengarahkan traffic melalui tunnel tersebut.

GRE tunnel sangat berguna untuk berbagai skenario, termasuk menghubungkan subnet yang tidak contiguous melalui internet, menyediakan path untuk multicast traffic, atau sebagai dasar untuk membangun VPN IPsec yang lebih aman (IPsec dapat melindungi payload GRE). Meskipun GRE sendiri tidak menyediakan keamanan (enkripsi atau otentikasi), ia adalah fondasi yang sangat fleksibel untuk arsitektur jaringan yang lebih kompleks.

```
Router(config-if)#tunnel mode gre ip
```

💡 **Fun Fact:** GRE pertama kali dikembangkan oleh Cisco pada tahun 1990-an dan distandarisasi oleh IETF. Meskipun sering digunakan bersama IPsec untuk keamanan (IPsec dapat mengenkripsi payload GRE), GRE sendiri hanya menyediakan enkapsulasi, bukan enkripsi atau otentikasi.
