## 🌐 Networking Concept: HSRP: Redundansi Gateway Default

**Category:** Routing
**TL;DR:** HSRP (Hot Standby Router Protocol) memastikan ketersediaan gateway default yang tinggi dengan mengizinkan dua atau lebih router berbagi satu alamat IP virtual.

Dalam banyak jaringan, kegagalan pada router yang berfungsi sebagai gateway default bisa melumpuhkan komunikasi perangkat dalam satu subnet ke jaringan luar. HSRP hadir sebagai solusi untuk masalah 'single point of failure' ini. Dengan HSRP, Anda dapat mengkonfigurasi sepasang atau lebih router untuk bekerja sama dan menyajikan diri sebagai satu gateway virtual kepada host di LAN. Ini berarti jika salah satu router fisik mengalami masalah, router cadangan (standby) akan mengambil alih peran gateway tanpa gangguan yang signifikan pada komunikasi.

HSRP beroperasi dengan menetapkan peran 'Aktif' dan 'Standby' pada router dalam suatu grup. Router Aktif adalah router yang secara aktif meneruskan lalu lintas data dan merespons permintaan ARP untuk alamat IP virtual gateway. Sementara itu, router Standby memantau router Aktif dan siap untuk mengambil alih peran tersebut jika router Aktif gagal. Semua host di subnet dikonfigurasi untuk menggunakan alamat IP virtual sebagai gateway default mereka, sehingga mereka tidak perlu tahu router fisik mana yang sedang aktif. HSRP menggunakan pesan Hello yang dikirim secara berkala antar router untuk memantau status satu sama lain.

Konfigurasi dasar HSRP melibatkan penetapan nomor grup HSRP, alamat IP virtual yang akan digunakan sebagai gateway default oleh klien, dan prioritas untuk menentukan router mana yang akan menjadi Aktif (dengan prioritas lebih tinggi). Fitur 'preempt' juga penting, yang memungkinkan router dengan prioritas lebih tinggi untuk mengambil kembali peran Aktif setelah ia pulih dari kegagalan. Ini memastikan bahwa router yang lebih disukai atau memiliki kapasitas lebih besar selalu bertindak sebagai gateway utama jika memungkinkan.

Dengan Cisco Packet Tracer, Anda dapat dengan mudah mensimulasikan dan memverifikasi kerja HSRP. Anda bisa mengkonfigurasi dua router dengan HSRP, melihat bagaimana peran Aktif dan Standby ditetapkan, dan yang paling menarik, mensimulasikan kegagalan pada router Aktif (misalnya, mematikan interface atau router secara fisik). Anda akan melihat bagaimana router Standby dengan cepat mengambil alih peran Aktif, dan perangkat klien tetap dapat mengakses jaringan luar tanpa perlu rekonfigurasi atau intervensi manual, menunjukkan transisi yang mulus dan meningkatkan keandalan jaringan secara signifikan.

```
interface GigabitEthernet0/1
 ip address 192.168.1.1 255.258.1.0
 standby 1 ip 192.168.1.254
```

💡 **Fun Fact:** HSRP adalah protokol proprietary milik Cisco, sementara standar terbuka yang melakukan fungsi serupa dikenal sebagai VRRP (Virtual Router Redundancy Protocol). Meskipun HSRP lebih tua, HSRPv2 yang lebih baru mendukung alamat MAC virtual yang diperluas dan lebih banyak grup, memungkinkan skalabilitas yang lebih baik.
