## 🤖 Android Concept: Activity: Layar Utama Aplikasi

**Category:** UI
**TL;DR:** Activity adalah komponen fundamental yang menyediakan layar tunggal untuk berinteraksi dengan pengguna dalam aplikasi Android.

Dalam pengembangan aplikasi Android, Activity adalah salah satu blok bangunan paling dasar dan krusial. Bayangkan setiap layar yang Anda lihat di aplikasi, seperti layar login, daftar produk, atau halaman pengaturan; setiap layar tersebut biasanya diwakili oleh sebuah Activity. Tugas utama Activity adalah untuk menampilkan antarmuka pengguna (UI) dan mengelola interaksi pengguna dengan UI tersebut. Ini adalah 'rumah' bagi semua elemen visual seperti tombol, teks, gambar, dan elemen interaktif lainnya.

Setiap Activity memiliki siklus hidup (lifecycle) yang unik, mulai dari saat ia dibuat, terlihat oleh pengguna, berinteraksi, hingga akhirnya dihancurkan. Fungsi paling penting yang akan sering Anda gunakan adalah `onCreate()`. Ini adalah metode yang dipanggil ketika Activity pertama kali dibuat, tempat di mana Anda akan menyiapkan sebagian besar tampilan UI-nya. Anda akan menggunakan `setContentView(R.layout.nama_layout_anda)` untuk menghubungkan kode Kotlin Anda dengan file XML layout yang mendefinisikan tata letak visual layar tersebut.

Activity tidak hanya tentang menampilkan UI, tetapi juga memungkinkan navigasi antar layar dalam aplikasi Anda. Ketika Anda ingin berpindah dari satu layar ke layar lain—misalnya, dari halaman beranda ke halaman detail produk—Anda akan memulai Activity baru menggunakan sebuah Intent. Activity yang baru akan ditempatkan di atas Activity sebelumnya dalam tumpukan tugas (back stack), memungkinkan pengguna untuk kembali ke layar sebelumnya dengan tombol 'Back'.

Memahami Activity adalah langkah pertama yang sangat penting dalam perjalanan Anda sebagai developer Android. Ini adalah fondasi di mana semua interaksi pengguna dan presentasi data terjadi, membentuk pengalaman pengguna secara keseluruhan. Menguasai konsep ini akan membuka jalan bagi Anda untuk mempelajari komponen Android lain yang lebih kompleks dan membangun aplikasi yang fungsional dan responsif.

```
package com.example.myfirstapp

import android.os.Bundle
import androidx.appcompat.app.AppCompatActivity

class MainActivity : AppCompatActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main) // Menghubungkan ke file layout XML
        // Di sinilah kamu akan menambahkan logika untuk interaksi UI seperti tombol
    }
}
```

💡 **Fun Fact:** Tahukah kamu, di awal-awal Android, Activity menjadi satu-satunya cara untuk mengelola UI, bahkan untuk bagian kecil dari layar. Konsep Fragment muncul kemudian untuk mengatasi keterbatasan tersebut, terutama untuk tablet dengan layar lebih besar!
