## 🤖 Android Concept: Activity: Layar Interaksi Pengguna

**Category:** UI
**TL;DR:** Activity adalah komponen dasar yang menyediakan layar tunggal untuk berinteraksi dengan pengguna di aplikasi Android.

Activity adalah fondasi dasar dari antarmuka pengguna (UI) di aplikasi Android. Setiap kali Anda melihat sebuah 'layar' dalam aplikasi, seperti daftar email, halaman pengaturan, atau tampilan kamera, kemungkinan besar itu adalah sebuah instance dari Activity. Activity bertanggung jawab untuk menyediakan jendela di mana aplikasi Anda menggambar UI dan menerima interaksi pengguna. Ini adalah titik awal utama aplikasi Anda saat diluncurkan atau saat berpindah ke layar baru, bertindak sebagai "pintu masuk" visual untuk sebagian besar fitur aplikasi.

Salah satu konsep penting terkait Activity adalah siklus hidupnya (lifecycle). Saat sebuah Activity dibuat, di-pause, dilanjutkan, atau dihancurkan, sistem Android akan memanggil berbagai metode callback seperti `onCreate()`, `onStart()`, `onResume()`, `onPause()`, `onStop()`, dan `onDestroy()`. Metode `onCreate()` dipanggil pertama kali saat Activity dibuat dan menjadi tempat ideal untuk menginisialisasi komponen UI serta mempersiapkan data awal. Memahami lifecycle membantu kita mengelola sumber daya dan perilaku aplikasi dengan benar di berbagai kondisi, misalnya saat pengguna beralih aplikasi atau menerima panggilan telepon.

Activity terhubung erat dengan file layout XML Anda yang mendefinisikan struktur UI. Dengan memanggil `setContentView(R.layout.nama_layout_anda)` di dalam metode `onCreate()`, Anda memberi tahu Activity layout XML mana yang harus ditampilkan kepada pengguna. Setelah layout terhubung, Anda bisa mencari dan memanipulasi elemen UI (seperti `Button`, `TextView`, `ImageView`) dari kode Kotlin Anda, merespons klik pengguna, dan memperbarui tampilan secara dinamis. Ini adalah jembatan utama antara desain visual yang Anda buat di XML dan logika program yang Anda tulis di Kotlin.

```
import androidx.appcompat.app.AppCompatActivity
import android.os.Bundle

class MainActivity : AppCompatActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main) // Menghubungkan layout XML
        // Di sinilah Anda biasanya menginisialisasi tampilan atau logika awal
    }
}
```

💡 **Fun Fact:** Activity adalah salah satu komponen Android paling awal dan fundamental, diperkenalkan sejak API level 1. Mereka menjadi tulang punggung cara Google membayangkan aplikasi dengan UI visual.
