## ⚡ Competitive Programming Concept: Segment Tree

**Category:** Data Structure
**TL;DR:** Struktur data pohon yang efisien untuk melakukan query rentang dan update titik pada array.

Segment Tree adalah struktur data pohon biner yang digunakan untuk menyimpan informasi tentang rentang atau segmen dalam sebuah array. Tujuannya adalah untuk mendukung operasi query rentang (misalnya, mencari sum, min, atau max dalam sub-array tertentu) dan update titik (mengubah nilai elemen tunggal dalam array) secara efisien, jauh lebih cepat daripada metode brute-force. Ini adalah salah satu tool fundamental dalam competitive programming untuk masalah yang melibatkan array dan operasi pada rentang.

Secara konseptual, Segment Tree merepresentasikan array asli sebagai daun-daun di pohon, di mana setiap node internal menyimpan hasil komputasi dari gabungan rentang yang dicover oleh anak-anaknya. Misalnya, jika sebuah node menyimpan jumlah, maka nilainya adalah jumlah dari nilai anak kiri dan anak kanan. Struktur ini dibangun secara rekursif, membagi rentang array menjadi dua hingga mencapai elemen tunggal. Ukuran pohon biasanya sekitar 2N hingga 4N node untuk array berukuran N.

Ada tiga operasi utama: `build`, `update`, dan `query`. Operasi `build` membutuhkan waktu O(N) untuk menginisialisasi seluruh pohon. Operasi `update` mengubah nilai pada indeks tertentu di array asli dan memperbarui node-node yang terpengaruh di jalur dari daun ke root, membutuhkan waktu O(log N). Operasi `query` mencari informasi tentang rentang tertentu, juga membutuhkan waktu O(log N) karena hanya mengunjungi jalur yang relevan di pohon.

Kompleksitas waktu untuk `build` adalah O(N), sedangkan untuk `update` dan `query` adalah O(log N). Kompleksitas ruang yang dibutuhkan adalah O(N) karena jumlah node dalam Segment Tree proporsional dengan ukuran array asli. Segment Tree sangat serbaguna dan dapat digunakan untuk berbagai jenis query rentang seperti sum, minimum, maximum, GCD, dan bahkan operasi yang lebih kompleks asalkan operasi penggabungan bersifat asosiatif.

```
long long tree[4*MAXN]; // Asumsikan MAXN adalah ukuran maksimum array

// Fungsi query untuk mencari sum dalam rentang [l, r]
long long query_sum(int node, int start, int end, int l, int r) {
    if (r < start || end < l) return 0; // Rentang tidak overlap
    if (l <= start && end <= r) return tree[node]; // Rentang node sepenuhnya dalam query
    int mid = (start + end) / 2;
    long long p1 = query_sum(2 * node, start, mid, l, r);
    long long p2 = query_sum(2 * node + 1, mid + 1, end, l, r);
    return p1 + p2;
}
```

💡 **Fun Fact:** Segment Tree bisa digeneralisasi menjadi 2D atau bahkan lebih tinggi dimensi untuk menyelesaikan masalah query rentang pada matriks atau ruang multi-dimensi, meskipun dengan peningkatan kompleksitas yang signifikan.
