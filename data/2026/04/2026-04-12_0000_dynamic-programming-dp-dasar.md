## ⚡ Competitive Programming Concept: Dynamic Programming (DP) Dasar

**Category:** DP
**TL;DR:** DP adalah teknik optimasi dengan memecah masalah menjadi sub-masalah yang lebih kecil, menyimpan hasilnya untuk menghindari perhitungan ulang.

Dynamic Programming (DP) adalah sebuah teknik pemecahan masalah yang powerful dalam competitive programming. Ia digunakan untuk mengoptimalkan solusi masalah yang memiliki dua karakteristik utama: *overlapping subproblems* (sub-masalah yang sama dihitung berulang kali) dan *optimal substructure* (solusi optimal dari masalah besar dapat dibangun dari solusi optimal sub-masalahnya). DP bekerja dengan memecah masalah kompleks menjadi sub-masalah yang lebih kecil yang saling bergantung satu sama lain.

Inti dari DP adalah menyimpan hasil dari sub-masalah yang telah dipecahkan ke dalam tabel (biasanya array atau matriks) untuk menghindari perhitungan ulang. Ada dua pendekatan utama: *Top-down* (atau memoization), di mana kita memulai dari masalah utama dan secara rekursif memecahkannya sambil menyimpan hasil; dan *Bottom-up* (atau tabulation), di mana kita membangun solusi dari sub-masalah terkecil hingga masalah utama secara iteratif.

Proses berpikir dalam DP melibatkan identifikasi "state" (apa yang perlu kita ketahui untuk mendefinisikan sub-masalah), "transisi" (bagaimana kita bergerak dari satu state ke state lain, atau bagaimana state saat ini bergantung pada state sebelumnya), dan "base case" (kasus-kasus terkecil yang solusinya sudah diketahui). Kompleksitas waktu DP seringkali berupa O(N*M) atau O(N^k) tergantung pada dimensi state dan jumlah transisi. Misalnya, menghitung deret Fibonacci ke-N dengan DP bottom-up hanya memerlukan O(N) waktu, dibanding O(2^N) untuk rekursif murni.

DP adalah fundamental untuk menyelesaikan berbagai jenis masalah, mulai dari optimasi, pencarian jalur terpendek di DAG, hingga penghitungan banyaknya cara. Variasinya sangat luas, mencakup 1D DP, 2D DP, Tree DP, Digit DP, dan Profile DP. Menguasai konsep dasar DP adalah langkah krusial untuk menjadi programmer kompetitif yang handal.

```
// Menghitung Fibonacci ke-N dengan DP bottom-up
// Asumsi N <= 99
int calculate_fib_dp(int n) {
    int fib[100]; 
    fib[0] = 0;
    fib[1] = 1;
    for (int i = 2; i <= n; ++i) {
        fib[i] = fib[i-1] + fib[i-2];
    }
    return fib[n];
}
```

💡 **Fun Fact:** Istilah Dynamic Programming diciptakan pada tahun 1950-an oleh Richard Bellman, yang awalnya mencoba menyembunyikan penelitian matematikanya dari atasannya dengan menggunakan kata 'dynamic' agar terdengar lebih serius dan kurang berbau riset murni.
