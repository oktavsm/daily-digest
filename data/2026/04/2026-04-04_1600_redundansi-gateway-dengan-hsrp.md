## 🌐 Networking Concept: Redundansi Gateway dengan HSRP

**Category:** Routing
**TL;DR:** HSRP menyediakan redundansi gateway default untuk host di LAN dengan memungkinkan beberapa router berbagi alamat IP virtual yang sama.

Dalam jaringan lokal (LAN), setiap perangkat membutuhkan gateway default untuk dapat berkomunikasi dengan jaringan di luar LAN mereka. Jika gateway default ini hanya satu router dan router tersebut mengalami kegagalan, maka seluruh perangkat di LAN akan kehilangan konektivitas ke internet atau jaringan eksternal lainnya. Ini menciptakan "single point of failure" yang kritikal. Hot Standby Router Protocol (HSRP) hadir sebagai solusi untuk masalah ini, menyediakan mekanisme redundansi yang memungkinkan beberapa router untuk bertindak sebagai gateway default yang sama.

HSRP bekerja dengan mengkonfigurasi dua atau lebih router untuk berbagi satu alamat IP virtual dan alamat MAC virtual yang sama. Salah satu router akan menjadi router "Active" yang bertanggung jawab untuk meneruskan paket data dan merespons permintaan ARP untuk alamat IP virtual tersebut. Router lainnya akan menjadi router "Standby" yang secara aktif memantau status router Active. Jika router Active gagal, router Standby akan secara otomatis mengambil alih peran Active, memastikan layanan gateway tetap berjalan tanpa gangguan bagi perangkat di LAN. Ini dilakukan melalui pertukaran pesan "hello" secara berkala antara router-router dalam grup HSRP.

Konfigurasi HSRP melibatkan penetapan nomor grup standby pada antarmuka router, kemudian mendefinisikan alamat IP virtual yang akan dibagikan oleh grup tersebut. Opsional, kita dapat mengatur prioritas untuk menentukan router mana yang lebih disukai sebagai Active (router dengan prioritas tertinggi menjadi Active) dan mengaktifkan fitur `preempt` agar router dengan prioritas lebih tinggi yang baru pulih dapat kembali mengambil alih peran Active. Keunggulan utama HSRP adalah peningkatan ketersediaan jaringan yang signifikan. Pengguna akhir tidak akan menyadari adanya failover, karena alamat IP gateway mereka tidak berubah. Ini memastikan kelangsungan bisnis dan pengalaman pengguna yang mulus meskipun ada kegagalan perangkat keras router.

```
standby 1 ip 192.168.1.254
```

💡 **Fun Fact:** HSRP adalah protokol milik Cisco yang bersifat proprietary. Versi standar terbuka yang setara adalah VRRP (Virtual Router Redundancy Protocol).
