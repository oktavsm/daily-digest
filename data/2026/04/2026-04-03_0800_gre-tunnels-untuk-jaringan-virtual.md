## 🌐 Networking Concept: GRE Tunnels untuk Jaringan Virtual

**Category:** Protocol
**TL;DR:** Membangun koneksi point-to-point virtual di atas jaringan IP lain dengan membungkus paket data.

Hari ini kita akan menjelajahi Generic Routing Encapsulation (GRE) Tunnels, sebuah metode untuk menciptakan koneksi point-to-point virtual melalui jaringan IP yang sudah ada, seperti internet. Bayangkan Anda memiliki dua kantor yang terpisah jauh, dan Anda ingin membuat mereka seolah-olah terhubung langsung secara logis, memungkinkan routing protocol atau multicast traffic berjalan mulus di antara keduanya. Di sinilah GRE berperan, mengatasi batasan jaringan perantara yang mungkin tidak mendukung jenis trafik tersebut.

GRE bekerja dengan membungkus (encapsulating) paket IP asli di dalam sebuah paket IP baru. Paket IP asli (termasuk header dan payload-nya) menjadi payload dari paket GRE, yang kemudian ditempatkan di dalam header IP yang baru. Header IP terluar ini memiliki alamat IP dari endpoint tunnel (interface fisik router) sebagai source dan destination-nya. Ini memungkinkan paket yang terenkapsulasi GRE untuk melintasi jaringan perantara seolah-olah itu adalah paket IP biasa, sampai mencapai ujung tunnel lainnya di mana proses dekapsulasi terjadi.

Dalam Cisco Packet Tracer, kita bisa mensimulasikan skenario ini dengan mudah. Anda akan mengkonfigurasi interface tunnel virtual pada dua router yang terhubung melalui jaringan perantara. Setelah tunnel GRE aktif, Anda bisa menjalankan routing protocol seperti OSPF atau EIGRP di atas tunnel tersebut, memungkinkan kedua router untuk saling bertukar informasi routing seolah-olah mereka terhubung langsung, tanpa perlu khawatir dengan topologi fisik jaringan perantara. Ini sangat berguna untuk membangun VPN dasar atau menghubungkan jaringan terisolasi secara logis.

Penting untuk diingat bahwa GRE sendiri tidak menyediakan fitur keamanan seperti enkripsi atau autentikasi. Ia murni mekanisme enkapsulasi. Oleh karena itu, dalam implementasi dunia nyata, GRE seringkali digabungkan dengan IPsec untuk menyediakan lapisan keamanan yang kuat, membentuk solusi VPN yang lengkap dan aman. Meskipun demikian, pemahaman dasar GRE adalah fondasi penting untuk konsep jaringan yang lebih kompleks dan membangun solusi konektivitas yang fleksibel.

```
interface Tunnel 0
 ip address 10.0.0.1 255.255.255.252
 tunnel source GigabitEthernet0/0
 tunnel destination 203.0.113.2
 tunnel mode gre ip
```

💡 **Fun Fact:** Kata "Generic" pada GRE menunjukkan kemampuannya untuk membungkus hampir semua jenis protokol Layer 3 di dalamnya, bukan hanya IP, menjadikannya sangat fleksibel!
