## 🌐 Networking Concept: Redundansi Gateway dengan HSRP

**Category:** Protocol
**TL;DR:** HSRP adalah protokol milik Cisco yang menyediakan redundansi gateway default untuk host di LAN dengan memungkinkan beberapa router berbagi satu alamat IP virtual dan MAC address.

Selamat pagi mahasiswa sekalian! Hari ini kita akan menjelajahi salah satu konsep krusial dalam desain jaringan yang andal: redundansi. Bayangkan jika router yang bertindak sebagai default gateway untuk seluruh jaringan Anda tiba-tiba mati. Seluruh komunikasi keluar dari LAN akan terhenti, menyebabkan *downtime* yang signifikan. Untuk mengatasi *single point of failure* ini, Cisco mengembangkan Hot Standby Router Protocol (HSRP). HSRP memungkinkan dua atau lebih router bekerja sama untuk menyediakan satu alamat IP virtual yang berfungsi sebagai default gateway bagi host di LAN, memastikan layanan jaringan tetap tersedia meskipun salah satu router mengalami kegagalan.

Bagaimana HSRP mencapai hal ini? Dalam satu grup HSRP, satu router diidentifikasi sebagai router "Aktif" (Active Router) yang bertanggung jawab meneruskan paket untuk alamat IP virtual tersebut. Router kedua bertindak sebagai router "Standby" (Standby Router) yang siap mengambil alih peran aktif jika router aktif gagal. Router lain bisa berada dalam status "Listen". Mereka berkomunikasi satu sama lain menggunakan pesan "Hello" multicast secara periodik. Jika router aktif tidak mengirimkan pesan Hello dalam periode waktu tertentu, router standby akan mempromosikan dirinya menjadi router aktif baru. Penentuan peran ini didasarkan pada prioritas; router dengan prioritas tertinggi akan menjadi aktif, dan fitur *preemption* dapat diaktifkan agar router dengan prioritas lebih tinggi yang pulih setelah kegagalan bisa merebut kembali peran aktifnya.

Dengan HSRP, *failover* dari router aktif ke standby terjadi secara transparan bagi perangkat akhir. Host di jaringan tidak perlu mengubah konfigurasi default gateway mereka; mereka akan terus mengirimkan paket ke alamat IP virtual yang sama, tanpa menyadari bahwa router fisik di belakangnya telah berganti. Ini menghasilkan ketersediaan jaringan yang sangat tinggi, penting untuk aplikasi bisnis kritis. Di Cisco Packet Tracer, kita bisa mensimulasikan HSRP dengan mudah: siapkan dua router, satu switch, dan beberapa *end device*. Anda dapat mengkonfigurasi HSRP pada *interface* LAN kedua router, lalu coba matikan salah satu *interface* router aktif untuk melihat bagaimana router standby mengambil alih peran aktif secara otomatis. Ini adalah cara praktis untuk memahami pentingnya redundansi dalam jaringan modern.

```
standby 1 ip 192.168.1.254
```

💡 **Fun Fact:** HSRP adalah salah satu protokol FHRP (First Hop Redundancy Protocol) pertama yang dikembangkan, dan tetap menjadi pilihan populer di jaringan Cisco karena kesederhanaan dan efektivitasnya. Ada juga standar terbuka serupa bernama VRRP.
