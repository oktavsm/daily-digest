## ☕ Java Concept: Penanganan Exception (try-catch)

**Category:** Exception
**TL;DR:** Mekanisme Java untuk menangani kesalahan atau kejadian tak terduga yang terjadi saat program berjalan agar tidak crash.

Setiap program bisa mengalami masalah saat berjalan, seperti mencoba membagi dengan nol, mengakses elemen di luar batas array, atau membaca file yang tidak ada. Kejadian tak terduga ini disebut "exception" (pengecualian). Jika tidak ditangani, exception akan menyebabkan program berhenti (crash) dan menampilkan pesan error yang mungkin membingungkan pengguna.

Java menyediakan mekanisme `try-catch` untuk "menangkap" dan "menangani" exception ini secara elegan. Tujuannya adalah agar program tetap berjalan dengan aman meskipun ada masalah, atau setidaknya memberikan pesan error yang lebih informatif kepada pengguna. Ini sangat penting untuk membangun aplikasi yang kokoh dan ramah pengguna.

Blok `try` berisi kode yang berpotensi menimbulkan exception. Jika sebuah exception terjadi di dalam blok `try`, eksekusi kode di dalam `try` akan langsung berhenti, dan kontrol akan beralih ke blok `catch` yang sesuai. Blok `catch` berisi kode yang akan dijalankan untuk menanggapi exception tersebut, misalnya mencetak pesan error, menulis log, atau meminta input ulang dari pengguna.

Contoh praktisnya, jika kita mencoba mengubah teks menjadi angka menggunakan `Integer.parseInt()` tetapi teksnya bukan format angka yang valid, maka akan terjadi `NumberFormatException`. Dengan `try-catch`, kita bisa "menangkap" exception ini dan memberi tahu pengguna untuk memasukkan input yang benar, daripada program langsung crash.

```
public class DemoException {
    public static void main(String[] args) {
        String inputStr = "bukanangka";
        try {
            int angka = Integer.parseInt(inputStr);
            System.out.println("Angka: " + angka);
        } catch (NumberFormatException e) {
            System.err.println("Error: Input bukan angka!");
            System.out.println("Detail: " + e.getMessage());
        }
        System.out.println("Program tetap berjalan.");
    }
}
```

💡 **Fun Fact:** Tahukah kamu, di Java, ada perbedaan mendasar antara `Exception` dan `Error`? `Exception` (seperti `NumberFormatException`) adalah masalah yang seringkali bisa kamu tangani di kode, sedangkan `Error` (seperti `OutOfMemoryError`) biasanya adalah masalah serius yang menunjukkan kerusakan sistem dan sulit diatasi oleh program itu sendiri.
