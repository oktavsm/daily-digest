## 🤖 Android Concept: Daur Hidup Activity

**Category:** Architecture
**TL;DR:** Daur hidup Activity menjelaskan serangkaian status yang dilewati sebuah Activity dari dibuat hingga dihancurkan, sangat penting untuk mengelola sumber daya dan perilaku aplikasi dengan benar.

Setiap kali pengguna berinteraksi dengan aplikasi Android, mereka sebenarnya berinteraksi dengan satu atau lebih `Activity`. Sebagai developer, penting sekali untuk memahami "daur hidup" atau `lifecycle` dari sebuah `Activity`. Daur hidup ini adalah serangkaian status yang dilewati `Activity` sejak pertama kali dibuat, menjadi aktif di layar, beralih ke latar belakang, hingga akhirnya dihancurkan oleh sistem. Memahami daur hidup ini membantu kita menulis kode yang stabil, responsif, dan efisien, terutama dalam mengelola data dan sumber daya.

Ketika sebuah `Activity` pertama kali dibuat, sistem akan memanggil metode `onCreate()`. Ini adalah tempat paling umum untuk melakukan inisialisasi awal, seperti mengatur tampilan (layout) dengan `setContentView()`, atau mendeklarasikan variabel yang akan digunakan sepanjang daur hidup `Activity`. Setelah `onCreate()`, `onStart()` dipanggil saat `Activity` menjadi terlihat oleh pengguna, dan `onResume()` dipanggil saat `Activity` benar-benar berada di latar depan dan siap menerima interaksi pengguna. Pada titik ini, `Activity` dianggap "aktif" dan pengguna dapat berinteraksi dengannya.

Namun, `Activity` bisa saja kehilangan fokus. Misalnya, ketika panggilan telepon masuk, aplikasi lain terbuka di atasnya, atau pengguna menekan tombol "Home". Dalam kasus ini, `onPause()` dipanggil saat `Activity` sebagian tersembunyi, diikuti oleh `onStop()` jika `Activity` tidak lagi terlihat sama sekali. Terakhir, `onDestroy()` dipanggil tepat sebelum `Activity` sepenuhnya dihancurkan dari memori. Setiap metode ini adalah kesempatan bagi developer untuk menyimpan data, menghentikan operasi yang memakan sumber daya, atau melepaskan referensi untuk mencegah kebocoran memori.

Memahami dan memanfaatkan metode `lifecycle` ini dengan benar adalah kunci untuk membangun aplikasi Android yang tangguh. Anda dapat memastikan bahwa sumber daya penting dibebaskan saat tidak dibutuhkan, data disimpan secara otomatis sebelum `Activity` dihancurkan, dan aplikasi dapat pulih dengan baik dari perubahan konfigurasi (seperti rotasi layar) atau interupsi sistem. Dengan begitu, pengalaman pengguna akan tetap mulus dan aplikasi Anda tidak mudah crash.

```
import android.os.Bundle
import android.util.Log
import androidx.appcompat.app.AppCompatActivity

class MainActivity : AppCompatActivity() {

    private val TAG = "MainActivityLifecycle"

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)
        Log.d(TAG, "onCreate() dipanggil")
    }

    override fun onStart() {
        super.onStart()
        Log.d(TAG, "onStart() dipanggil")
    }

    override fun onResume() {
        super.onResume()
        Log.d(TAG, "onResume() dipanggil")
    }

    override fun onPause() {
        super.onPause()
        Log.d(TAG, "onPause() dipanggil")
    }

    override fun onStop() {
        super.onStop()
        Log.d(TAG, "onStop() dipanggil")
    }

    override fun onDestroy() {
        super.onDestroy()
        Log.d(TAG, "onDestroy() dipanggil")
    }
}
```

💡 **Fun Fact:** Kesalahan umum yang sering dibuat pemula adalah lupa menyimpan data saat `onPause()` atau `onStop()` dipanggil, sehingga data bisa hilang jika sistem mematikan `Activity` sebelum `onDestroy()` sempat dipanggil. Android mendefinisikan daur hidup ini secara ketat untuk membantu pengembang menghemat baterai dan mengelola memori di perangkat yang terbatas.
