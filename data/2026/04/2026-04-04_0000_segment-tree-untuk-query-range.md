## ⚡ Competitive Programming Concept: Segment Tree untuk Query Range

**Category:** Data Structure
**TL;DR:** Segment Tree adalah struktur data pohon yang memungkinkan query range dan update titik dalam waktu logaritmik.

Segment Tree adalah struktur data berbasis pohon biner yang digunakan untuk menyimpan informasi tentang interval atau segmen. Kegunaan utamanya adalah untuk secara efisien melakukan operasi (seperti mencari minimum, maksimum, atau jumlah) pada sub-range dari sebuah array, serta melakukan update pada elemen individual. Setiap node dalam Segment Tree merepresentasikan sebuah interval dalam array asli, dengan root merepresentasikan seluruh array. Node anak dari sebuah node yang merepresentasikan interval `[L, R]` akan merepresentasikan interval `[L, Mid]` dan `[Mid+1, R]`, di mana `Mid = (L+R)/2`.

Pembangunan Segment Tree dilakukan secara rekursif. Dimulai dari root yang mencakup seluruh array `[0, N-1]`. Untuk setiap node yang merepresentasikan interval `[L, R]`, jika `L == R`, node tersebut adalah leaf dan menyimpan nilai `array[L]`. Jika tidak, kita bagi interval menjadi dua `[L, Mid]` dan `[Mid+1, R]`, lalu secara rekursif membangun anak kiri dan anak kanan. Setelah anak-anak dibangun, node saat ini menggabungkan hasil dari kedua anaknya (misalnya, menjumlahkan nilai mereka). Proses konstruksi membutuhkan waktu `O(N)` karena setiap elemen array asli akan muncul di leaf node dan setiap node di atasnya akan dihitung sekali.

Operasi query untuk mendapatkan hasil pada range `[queryL, queryR]` juga rekursif. Jika interval node saat ini `[nodeL, nodeR]` sepenuhnya berada di luar `[queryL, queryR]`, kita kembalikan nilai identitas (misalnya 0 untuk sum, atau infiniti untuk min/max). Jika `[nodeL, nodeR]` sepenuhnya berada di dalam `[queryL, queryR]`, kita kembalikan nilai yang disimpan di node saat ini. Jika tidak, kita secara rekursif mencari di anak kiri dan anak kanan, lalu menggabungkan hasilnya. Operasi update untuk mengubah nilai pada indeks `idx` mirip: kita cari leaf yang merepresentasikan `idx`, ubah nilainya, lalu propagasikan perubahan ke atas sampai root dengan memperbarui nilai di setiap node yang dilalui.

Secara keseluruhan, konstruksi Segment Tree membutuhkan waktu `O(N)`. Setiap operasi query range dan update titik membutuhkan waktu `O(log N)`. Ruang memori yang dibutuhkan adalah `O(N)` karena jumlah node dalam Segment Tree bisa mencapai `2N - 1` atau hingga `4N` untuk implementasi array yang lebih sederhana, untuk array berukuran `N`.

```
const int N_MAX = 100005;
int arr[N_MAX];
int tree[4 * N_MAX]; // Array untuk Segment Tree, ukuran 4*N cukup

void build(int node, int start, int end) {
    if (start == end) { tree[node] = arr[start]; return; }
    int mid = (start + end) / 2;
    build(2 * node, start, mid);
    build(2 * node + 1, mid + 1, end);
    tree[node] = tree[2 * node] + tree[2 * node + 1]; // Contoh: sum query
}

int query(int node, int start, int end, int l, int r) {
    if (r < start || end < l) return 0; // Range node di luar range query
    if (l <= start && end <= r) return tree[node]; // Range node sepenuhnya di dalam range query
    int mid = (start + end) / 2;
    int p1 = query(2 * node, start, mid, l, r);
    int p2 = query(2 * node + 1, mid + 1, end, l, r);
    return p1 + p2;
}
```

💡 **Fun Fact:** Segment Tree seringkali disebut sebagai salah satu "senjata Swiss Army Knife" di competitive programming karena kemampuannya menyelesaikan berbagai masalah range query dan update dengan satu struktur yang sama. Ada juga varian canggihnya seperti Lazy Segment Tree untuk range update yang lebih kompleks.
