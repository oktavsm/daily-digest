## 🤖 Android Concept: Activity dan Siklus Hidupnya

**Category:** UI
**TL;DR:** Activity adalah komponen fundamental di Android yang menyediakan satu layar antarmuka pengguna, berfungsi sebagai titik masuk interaksi pengguna dengan aplikasi Anda, dan memiliki siklus hidup yang terdefinisi.

Activity adalah blok bangunan fundamental dari setiap aplikasi Android yang menyediakan antarmuka pengguna (UI) tunggal untuk berinteraksi dengan pengguna. Bayangkan Activity sebagai satu "layar" di aplikasi Anda; misalnya, layar utama yang menampilkan daftar item, layar detail produk, atau layar pengaturan, semuanya biasanya diimplementasikan sebagai Activity terpisah. Saat pengguna meluncurkan aplikasi Anda atau berpindah antar bagiannya, mereka sebenarnya berpindah dari satu Activity ke Activity lain.

Setiap Activity bertanggung jawab untuk menggambar layoutnya dan merespons interaksi pengguna. Di dalamnya, Anda menempatkan elemen-elemen UI seperti tombol (`Button`), teks (`TextView`), atau kotak input (`EditText`). Aktivitas ini bertindak sebagai wadah atau "host" untuk semua elemen visual ini, mengelola bagaimana mereka ditampilkan dan bagaimana mereka merespons sentuhan atau input lainnya dari pengguna.

Salah satu konsep paling penting dari Activity adalah "Siklus Hidup" (Lifecycle) nya. Ini adalah serangkaian state yang dilewati Activity dari saat pertama kali dibuat hingga dihancurkan. Sistem Android akan memanggil berbagai metode callback seperti `onCreate()`, `onStart()`, `onResume()`, `onPause()`, `onStop()`, dan `onDestroy()` saat Activity berpindah antar state ini. Memahami lifecycle ini sangat krusial agar aplikasi Anda berjalan dengan baik, menghemat baterai, dan tidak kehilangan data, misalnya saat pengguna beralih aplikasi atau memutar perangkat. Metode `onCreate()` adalah tempat utama Anda melakukan inisialisasi awal, seperti mengatur tampilan UI menggunakan `setContentView()`.

```
package com.example.myfirstapp

import android.os.Bundle
import androidx.appcompat.app.AppCompatActivity

class MainActivity : AppCompatActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main) // Mengatur layout untuk Activity ini
    }
}
```
