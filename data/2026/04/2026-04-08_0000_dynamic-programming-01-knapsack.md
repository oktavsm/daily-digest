## ⚡ Competitive Programming Concept: Dynamic Programming: 0/1 Knapsack

**Category:** DP
**TL;DR:** Masalah 0/1 Knapsack menggunakan Dynamic Programming bertujuan memilih item dengan nilai maksimum yang bisa dimuat dalam kapasitas tas tertentu, di mana setiap item hanya bisa diambil sekali atau tidak sama sekali.

Masalah 0/1 Knapsack adalah salah satu masalah klasik dalam ilmu komputer yang sering muncul di kompetisi. Diberikan sejumlah N item, di mana setiap item memiliki berat (weight) dan nilai (value), serta sebuah 'knapsack' (tas) dengan kapasitas maksimum W. Tugas kita adalah memilih subset item yang bisa dimasukkan ke dalam knapsack sehingga total nilai item yang dipilih maksimal, tanpa melebihi kapasitas W. Penamaan '0/1' menunjukkan bahwa setiap item hanya bisa diambil secara keseluruhan (1) atau tidak diambil sama sekali (0), tidak bisa diambil sebagian. Pendekatan greedy (misalnya, selalu mengambil item dengan rasio nilai/berat terbaik) tidak selalu memberikan solusi optimal untuk masalah ini.

Untuk menyelesaikan masalah ini dengan Dynamic Programming, kita mendefinisikan sebuah state `dp[i][j]` sebagai nilai maksimum yang bisa diperoleh dengan mempertimbangkan `i` item pertama dan kapasitas knapsack `j`. Rekurensi untuk mengisi tabel DP ini adalah sebagai berikut: untuk setiap item ke-`i` (dengan berat `weights[i-1]` dan nilai `values[i-1]`) dan setiap kapasitas `j`:

1. Jika berat item ke-`i` lebih besar dari kapasitas `j` (`weights[i-1] > j`), maka item tersebut tidak bisa dimasukkan. Jadi, nilai maksimumnya sama dengan nilai maksimum yang diperoleh dengan `i-1` item pertama dan kapasitas `j`: `dp[i][j] = dp[i-1][j]`.
2. Jika berat item ke-`i` kurang dari atau sama dengan kapasitas `j` (`weights[i-1] <= j`), kita punya dua pilihan: (a) tidak mengambil item ke-`i`, atau (b) mengambil item ke-`i`. Jika tidak mengambilnya, nilainya adalah `dp[i-1][j]`. Jika mengambilnya, nilainya adalah `values[i-1]` ditambah nilai maksimum yang bisa diperoleh dari `i-1` item pertama dengan kapasitas sisa `j - weights[i-1]`, yaitu `values[i-1] + dp[i-1][j - weights[i-1]]`. Kita ambil yang terbesar dari kedua pilihan ini: `dp[i][j] = max(dp[i-1][j], values[i-1] + dp[i-1][j - weights[i-1]])`.

Basis kasusnya adalah `dp[0][j] = 0` (tidak ada item, nilai 0) dan `dp[i][0] = 0` (kapasitas 0, nilai 0). Setelah tabel DP terisi, jawaban akhir adalah `dp[N][W]`. Kompleksitas waktu untuk mengisi tabel DP berukuran N x W ini adalah O(N*W), karena setiap sel dihitung dalam waktu O(1). Kompleksitas ruangnya juga O(N*W), meskipun bisa dioptimalkan menjadi O(W) dengan hanya menyimpan dua baris atau bahkan satu baris tabel DP karena perhitungan `dp[i]` hanya bergantung pada `dp[i-1]`.

```
int knapsack_0_1(int W, const std::vector<int>& weights, const std::vector<int>& values) {
    int n = weights.size();
    std::vector<std::vector<int>> dp(n + 1, std::vector<int>(W + 1, 0));

    for (int i = 1; i <= n; ++i) {
        for (int w = 1; w <= W; ++w) {
            if (weights[i-1] <= w) {
                dp[i][w] = std::max(dp[i-1][w], values[i-1] + dp[i-1][w - weights[i-1]]);
            } else {
                dp[i][w] = dp[i-1][w];
            }
        }
    }
    return dp[n][W];
}
```

💡 **Fun Fact:** Masalah Knapsack pertama kali diidentifikasi pada awal abad ke-20 dan merupakan salah satu dari 21 masalah NP-lengkap (NP-complete) asli yang diidentifikasi oleh Richard Karp pada tahun 1972. Meskipun NP-lengkap secara umum, ia memiliki solusi pseudo-polinomial (seperti DP ini) yang efisien jika kapasitas knapsack W tidak terlalu besar.
