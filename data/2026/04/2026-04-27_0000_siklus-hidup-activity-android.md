## 🤖 Android Concept: Siklus Hidup Activity Android

**Category:** Architecture
**TL;DR:** Memahami Siklus Hidup Activity sangat krusial untuk mengelola bagaimana layar aplikasi Anda dibuat, dihancurkan, dan merespon interaksi pengguna atau sistem, mencegah crash dan kebocoran memori.

Dalam pengembangan aplikasi Android, `Activity` adalah komponen fundamental yang merepresentasikan satu layar dalam antarmuka pengguna Anda. Bayangkan setiap layar yang Anda lihat di aplikasi sebagai sebuah Activity. Namun, sebuah Activity tidak hanya sekadar muncul dan menghilang; ia melewati serangkaian tahapan yang disebut "Siklus Hidup Activity" (Activity Lifecycle). Memahami siklus ini sangat penting karena menentukan kapan dan bagaimana resource dialokasikan, data disimpan, dan interaksi pengguna ditangani secara efektif.

Siklus hidup ini didefinisikan oleh serangkaian metode callback yang dipanggil oleh sistem Android saat Activity berpindah dari satu status ke status lainnya, seperti saat pertama kali dibuat, menjadi terlihat, aktif di latar depan, dihentikan sementara, atau dihancurkan. Metode-metode kunci yang harus Anda kenali meliputi `onCreate()`, `onStart()`, `onResume()`, `onPause()`, `onStop()`, dan `onDestroy()`. Setiap metode ini memiliki peran spesifik dan harus Anda override untuk menjalankan logika tertentu pada tahapan yang tepat.

Misalnya, `onCreate()` adalah tempat Anda biasanya melakukan inisialisasi awal seperti mengatur tata letak (layout) dan mengikat data. `onResume()` dipanggil saat Activity berinteraksi dengan pengguna, menjadikannya tempat ideal untuk memulai animasi atau koneksi yang hanya diperlukan saat Activity aktif. Sebaliknya, `onPause()` adalah tempat untuk menyimpan data sementara, menghentikan operasi yang memakan resource, atau mempersiapkan Activity untuk tidak lagi berada di latar depan.

Mengelola siklus hidup dengan benar mencegah berbagai masalah umum seperti kebocoran memori, konsumsi daya berlebihan, atau hilangnya data saat pengguna berpindah antar aplikasi atau menerima panggilan telepon. Dengan memahami setiap tahapan dan kapan harus melakukan tugas tertentu, Anda dapat membangun aplikasi Android yang tangguh, responsif, dan memberikan pengalaman pengguna yang mulus.

```
import android.os.Bundle
import android.util.Log
import androidx.appcompat.app.AppCompatActivity

class MainActivity : AppCompatActivity() {

    private val TAG = "MainActivity"

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)
        Log.d(TAG, "onCreate: Activity dibuat dan layout diatur.")
    }

    override fun onResume() {
        super.onResume()
        Log.d(TAG, "onResume: Activity siap berinteraksi dengan pengguna.")
    }

    override fun onPause() {
        super.onPause()
        Log.d(TAG, "onPause: Activity akan berhenti sementara.")
    }
}
```

💡 **Fun Fact:** Tahukah Anda bahwa pengelolaan siklus hidup Activity yang buruk adalah penyebab umum dari 'Application Not Responding' (ANR) atau kebocoran memori, terutama saat perangkat dirotasi atau pengguna beralih aplikasi?
