## 🤖 Android Concept: Activity: Layar Aplikasi Android

**Category:** UI
**TL;DR:** Activity adalah komponen fundamental yang merepresentasikan satu layar antarmuka pengguna (UI) dalam aplikasi Android Anda, tempat pengguna berinteraksi.

Dalam pengembangan aplikasi Android, Activity adalah salah satu blok bangunan paling dasar dan krusial. Bisa dibayangkan, setiap layar atau tampilan interaktif yang Anda lihat di aplikasi, seperti halaman beranda, halaman detail produk, atau layar pengaturan, umumnya diwakili oleh sebuah Activity. Activity menyediakan jendela tempat aplikasi Anda dapat menggambar antarmuka pengguna dan berinteraksi dengan pengguna melalui berbagai elemen UI seperti tombol, teks, dan gambar.

Setiap Activity memiliki siklus hidup (lifecycle) yang terdefinisi dengan baik, yang mencakup berbagai tahapan mulai dari saat Activity dibuat hingga dihancurkan. Beberapa tahapan penting antara lain `onCreate()`, yang dipanggil saat Activity pertama kali dibuat dan merupakan tempat umum untuk melakukan inisialisasi tata letak (layout) UI; `onStart()`, saat Activity mulai terlihat oleh pengguna; `onResume()`, saat Activity berada di latar depan dan siap berinteraksi; hingga `onPause()`, `onStop()`, dan `onDestroy()`.

Untuk membuat sebuah Activity di Kotlin, Anda akan mendefinisikan sebuah kelas yang mewarisi dari `AppCompatActivity` (atau `ComponentActivity` untuk kasus yang lebih spesifik). Di dalam metode `onCreate()`, Anda biasanya memanggil `setContentView(R.layout.nama_layout_anda)` untuk menghubungkan Activity dengan file layout XML yang mendefinisikan tampilan UI layar tersebut. Dari sana, Anda bisa mulai menambahkan logika untuk menangani interaksi pengguna, seperti klik tombol atau input teks.

Aplikasi Android seringkali terdiri dari beberapa Activity yang saling berinteraksi. Anda dapat berpindah dari satu Activity ke Activity lain menggunakan `Intent`. `Intent` bertindak sebagai “niat” atau “maksud” untuk melakukan sesuatu, misalnya memulai Activity baru, mengirim data ke Activity lain, atau bahkan berinteraksi dengan komponen aplikasi lain. Mekanisme ini memungkinkan aplikasi Anda memiliki alur navigasi yang kompleks dan dinamis antara berbagai layarnya.

```
import android.os.Bundle
import androidx.appcompat.app.AppCompatActivity

class MainActivity : AppCompatActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        // Menghubungkan Activity ini dengan tampilan antarmuka (UI) yang didefinisikan di XML
        setContentView(R.layout.activity_main)
        // Di sinilah inisialisasi komponen UI lainnya dilakukan,
        // seperti menemukan tombol dan menetapkan pendengar klik.
    }
}
```

💡 **Fun Fact:** Setiap kali Anda berpindah antar layar di aplikasi Android, Anda sebenarnya sedang menumpuk atau melepaskan Activity dari "back stack" (tumpukan belakang). Ini memungkinkan Anda untuk menekan tombol 'kembali' di perangkat dan menelusuri kembali layar-layar yang telah Anda kunjungi sebelumnya secara berurutan.
