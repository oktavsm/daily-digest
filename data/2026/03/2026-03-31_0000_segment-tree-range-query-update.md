## ⚡ Competitive Programming Concept: Segment Tree: Range Query Update

**Category:** Data Structure
**TL;DR:** Segment Tree adalah struktur data pohon yang memungkinkan query dan update pada rentang (range) dengan efisiensi logaritma.

Segment Tree adalah struktur data berbasis pohon biner yang sangat efisien untuk mengelola array dan melakukan operasi pada rentang (sub-array). Setiap node dalam pohon ini merepresentasikan sebuah "segmen" atau rentang dari array asli. Node daun (leaf nodes) merepresentasikan elemen tunggal dari array, sementara node internal menyimpan hasil komputasi (misalnya, jumlah, minimum, atau maksimum) dari rentang yang dicakup oleh anak-anaknya. Konsep utamanya adalah memecah masalah rentang besar menjadi sub-masalah rentang yang lebih kecil secara rekursif.

Pembangunan (build) Segment Tree dilakukan secara rekursif. Dimulai dari root yang mencakup seluruh array, kita membagi rentang menjadi dua bagian di tengah dan secara rekursif membangun sub-pohon untuk masing-masing bagian, hingga mencapai rentang berukuran satu (elemen daun). Nilai node internal dihitung dengan menggabungkan nilai dari kedua anak-anaknya. Untuk query pada rentang tertentu `[L, R]`, kita melakukan traversal pohon: jika rentang node saat ini `[start, end]` sepenuhnya berada di dalam `[L, R]`, kita kembalikan nilai node tersebut. Jika tidak, kita rekursif memanggil fungsi query pada anak kiri dan/atau kanan yang tumpang tindih dengan `[L, R]`, lalu menggabungkan hasilnya.

Operasi update, seperti mengubah nilai elemen tunggal pada indeks `idx`, juga dilakukan dengan traversal dari root ke daun yang sesuai dengan `idx`. Setelah nilai daun diperbarui, kita kembali ke atas (backtrack) dan memperbarui nilai node-node di jalur tersebut, memastikan informasi agregat di setiap node tetap konsisten. Baik operasi pembangunan pohon, query, maupun update pada Segment Tree memiliki kompleksitas waktu O(N) untuk pembangunan dan **O(log N)** untuk setiap query atau update. Ini jauh lebih cepat dibandingkan dengan O(N) yang diperlukan untuk operasi serupa pada array biasa, terutama jika ada banyak query dan update.

```
vector<int> tree; // Diasumsikan sudah diinisialisasi dan dibangun.
// Fungsi query untuk mendapatkan jumlah rentang [l, r] (0-indexed)
int query(int node, int start, int end, int l, int r) {
    if (r < start || end < l) return 0; // Rentang tidak tumpang tindih
    if (l <= start && end <= r) return tree[node]; // Rentang sepenuhnya di dalam query
    int mid = (start + end) / 2;
    return query(2 * node, start, mid, l, r) + query(2 * node + 1, mid + 1, end, l, r);
}
```

💡 **Fun Fact:** Segment Tree bisa diperluas dengan teknik 'Lazy Propagation' untuk menangani update pada seluruh rentang (range updates) secara efisien, bukan hanya update pada satu titik. Konsepnya juga terkait erat dengan Fenwick Tree (Binary Indexed Tree), namun Segment Tree umumnya lebih fleksibel untuk berbagai jenis operasi agregasi.
