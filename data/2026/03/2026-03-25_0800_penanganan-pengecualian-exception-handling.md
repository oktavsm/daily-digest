## ☕ Java Concept: Penanganan Pengecualian (Exception Handling)

**Category:** Exception
**TL;DR:** Mempelajari cara mengelola dan merespons kesalahan tak terduga yang terjadi saat program Java berjalan agar program tetap stabil dan memberikan pengalaman yang baik bagi pengguna.

Ketika kita menulis program, seringkali ada hal-hal tak terduga yang bisa terjadi saat program dieksekusi, seperti mencoba membagi angka dengan nol, mengakses elemen di luar batas array, atau mencoba membuka file yang tidak ada. Kejadian-kejadian ini disebut "pengecualian" (exceptions). Tanpa penanganan yang tepat, pengecualian ini akan menyebabkan program kita berhenti secara tiba-tiba (crash), yang tentu saja buruk bagi pengguna. Penanganan pengecualian memungkinkan kita untuk "menangkap" kesalahan ini dan meresponsnya dengan cara yang terkontrol.

Di Java, kita menggunakan blok `try-catch` untuk menangani pengecualian. Kode yang berpotensi menimbulkan pengecualian ditempatkan di dalam blok `try`. Jika pengecualian terjadi di dalam blok `try`, eksekusi kode di `try` akan dihentikan, dan kendali akan langsung berpindah ke blok `catch` yang sesuai. Blok `catch` mendefinisikan jenis pengecualian apa yang ingin ditangkap dan tindakan apa yang harus diambil untuk menanganinya, misalnya mencetak pesan kesalahan, mencoba tindakan alternatif, atau memberi tahu pengguna.

Ada banyak jenis pengecualian di Java, seperti `ArithmeticException` (untuk masalah matematika), `NullPointerException` (ketika mencoba menggunakan variabel yang bernilai `null`), atau `IOException` (untuk masalah input/output file). Pengecualian dikelompokkan menjadi `Checked Exceptions` (yang harus ditangani secara eksplisit oleh programmer) dan `Unchecked Exceptions` (seperti `RuntimeException` yang tidak wajib ditangani, tapi sangat disarankan). Terkadang, kita juga bisa menambahkan blok `finally` setelah `try-catch`. Kode di blok `finally` akan selalu dieksekusi, terlepas dari apakah pengecualian terjadi atau tidak, atau apakah pengecualian ditangkap atau tidak. Ini sangat berguna untuk membersihkan sumber daya seperti menutup koneksi database atau file.

```
public class Calculator {
    public static void main(String[] args) {
        try {
            int a = 10;
            int b = 0;
            int result = a / b; // Potensi ArithmeticException
            System.out.println("Hasil: " + result);
        } catch (ArithmeticException e) {
            System.err.println("Kesalahan! Tidak bisa membagi dengan nol.");
            System.err.println("Pesan error: " + e.getMessage());
        } finally {
            System.out.println("Operasi pembagian selesai diproses.");
        }
    }
}
```

💡 **Fun Fact:** Konsep penanganan pengecualian di Java sangat dipengaruhi oleh bahasa pemrograman C++ dan Smalltalk, namun Java menyederhanakannya dengan membedakan `checked` dan `unchecked` exceptions untuk mendorong pengembangan perangkat lunak yang lebih robust.
