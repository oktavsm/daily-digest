## 🌐 Networking Concept: Konfigurasi Static Routing Dasar

**Category:** Routing
**TL;DR:** Static routing adalah metode routing di mana administrator secara manual mengkonfigurasi jalur spesifik untuk paket data menuju jaringan tujuan.

Static routing adalah metode routing di mana administrator jaringan secara manual mengkonfigurasi jalur spesifik yang akan diambil oleh paket data untuk mencapai jaringan tujuan. Berbeda dengan routing dinamis yang secara otomatis mempelajari rute dari router lain menggunakan protokol routing, static routing memerlukan intervensi manual untuk setiap perubahan topologi. Metode ini sangat cocok untuk jaringan kecil dengan topologi sederhana, atau untuk tujuan khusus seperti mengkonfigurasi jalur default (default route) ke internet pada router edge.

Dalam konfigurasi static routing, sebuah rute ditentukan menggunakan alamat jaringan tujuan, subnet mask-nya, dan kemudian salah satu dari dua opsi: alamat IP hop berikutnya (next-hop IP address) atau antarmuka keluar (exit interface). Router akan menggunakan informasi ini untuk membuat entri dalam tabel routing-nya. Ketika sebuah paket tiba dengan tujuan yang cocok dengan entri static route, router akan meneruskan paket tersebut melalui next-hop atau antarmuka yang telah ditentukan. Administrative distance untuk static route biasanya adalah 1 (atau 0 jika mengacu langsung ke exit interface), menjadikannya lebih disukai daripada rute yang dipelajari secara dinamis (misalnya OSPF atau EIGRP) jika ada rute tujuan yang sama.

Keuntungan utama dari static routing meliputi keamanan yang lebih tinggi karena tidak ada pertukaran informasi routing dengan router lain, overhead CPU dan bandwidth yang lebih rendah karena tidak ada proses protokol routing, dan kemudahan implementasi pada jaringan kecil. Namun, kekurangannya menjadi jelas pada jaringan yang lebih besar atau dinamis. Setiap kali ada perubahan topologi atau penambahan jaringan baru, administrator harus memperbarui konfigurasi routing secara manual pada setiap router yang terpengaruh, yang sangat tidak skalabel dan rawan kesalahan. Selain itu, static routing tidak dapat secara otomatis beradaptasi terhadap kegagalan jalur, memerlukan konfigurasi redundant jalur secara manual.

Di Cisco Packet Tracer, static routing adalah konsep fundamental yang sering diajarkan untuk memahami dasar-dasar bagaimana router meneruskan paket. Mahasiswa dapat dengan mudah membangun topologi jaringan multi-router dan mengkonfigurasi static route untuk memungkinkan komunikasi antar jaringan yang berbeda, mempelajari bagaimana menentukan next-hop dan memecahkan masalah konektivitas dasar. Ini memberikan dasar yang kuat sebelum beralih ke protokol routing dinamis yang lebih kompleks.

```
[LAN A: 192.168.1.0/24]
        |
        R1 (GigabitEthernet0/0: 192.168.1.1)
        |
        | (Serial0/0/0: 10.0.0.1/30)
        +--------------------------+
                                   |
                                   R2 (Serial0/0/0: 10.0.0.2/30)
                                   |
                                   R2 (GigabitEthernet0/1: 192.168.2.1)
                                   |
                            [LAN B: 192.168.2.0/24]
```

**Cisco CLI:**
```
ip route 192.168.2.0 255.255.255.0 10.0.0.2
```

💡 **Fun Fact:** Meskipun banyak protokol routing dinamis yang canggih, static route masih sering digunakan di jaringan besar untuk tujuan spesifik seperti default route (jalur ke internet) atau sebagai backup route untuk skenario failover.
