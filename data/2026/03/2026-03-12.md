## 🌐 Networking Concept: Inter-VLAN Routing (Router-on-a-Stick)

**Category:** Routing
**TL;DR:** Memungkinkan perangkat di Virtual LAN (VLAN) yang berbeda untuk berkomunikasi menggunakan satu interface fisik router.

Virtual LAN (VLAN) adalah metode untuk membagi satu switch fisik menjadi beberapa jaringan logis yang terisolasi, seolah-olah Anda memiliki beberapa switch terpisah. Setiap VLAN memiliki broadcast domainnya sendiri, yang meningkatkan keamanan jaringan, mengurangi lalu lintas broadcast, dan memudahkan manajemen jaringan dengan mengelompokkan perangkat berdasarkan fungsi atau departemen, terlepas dari lokasi fisik mereka. Tanpa VLAN, semua perangkat yang terhubung ke switch yang sama akan berada dalam satu broadcast domain besar.

Secara default, perangkat yang berada dalam VLAN yang berbeda tidak dapat berkomunikasi satu sama lain. Mereka terisolasi pada Layer 2. Agar perangkat di VLAN yang berbeda dapat saling berkomunikasi, diperlukan perangkat Layer 3 yang berfungsi sebagai router. Inilah peran dari Inter-VLAN Routing, di mana router akan menerima lalu lintas dari satu VLAN, merutekannya, dan kemudian meneruskannya ke VLAN lain yang dituju.

Salah satu implementasi Inter-VLAN Routing yang populer adalah "Router-on-a-Stick". Metode ini menggunakan satu interface fisik pada router yang dikonfigurasi sebagai trunk port. Pada interface fisik router tersebut, dibuatlah beberapa sub-interface logis, di mana setiap sub-interface mewakili satu VLAN dan memiliki alamat IP gateway untuk VLAN tersebut. Komunikasi antara router dan switch dilakukan melalui satu kabel fisik yang membawa lalu lintas dari berbagai VLAN, yang diidentifikasi menggunakan tag protokol IEEE 802.1Q.

Ketika paket dari satu VLAN tiba di router melalui trunk port, router akan memeriksa tag 802.1Q untuk menentukan VLAN asal. Kemudian, router akan merutekan paket tersebut berdasarkan tujuan IP-nya ke sub-interface yang sesuai untuk VLAN tujuan, menambahkan tag 802.1Q yang benar, dan mengirimkannya kembali melalui trunk port fisik yang sama ke switch. Switch kemudian akan meneruskan paket ke perangkat tujuan dalam VLAN yang benar berdasarkan tag tersebut. Metode ini efisien karena hanya membutuhkan satu port router dan satu port switch, cocok untuk jaringan skala kecil hingga menengah.

```
     +----------+      +------------------+      +-------------+
     | PC1      |      |                  |      |             |
     | VLAN 10  |------| Fa0/1 (Acc V10)  |      | Fa0/0.10    |
     +----------+      |                  |      |   (GW V10)  |
                       |     Switch       |      |             |
     +----------+      | Fa0/24 (Trunk)   |------| Fa0/0 (Trunk)|
     | PC2      |      |                  |      |             |
     | VLAN 20  |------| Fa0/2 (Acc V20)  |      | Fa0/0.20    |
     +----------+      |                  |      |   (GW V20)  |
                       +------------------+      +-------------+
                                                      Router

```

**Cisco CLI:**
```
encapsulation dot1Q 10
```

💡 **Fun Fact:** "Router-on-a-Stick" mendapatkan namanya karena router berfungsi seperti 'tongkat' atau satu kabel yang 'menyimpan' banyak jaringan (VLAN) di dalamnya, menghubungkan mereka ke switch.
