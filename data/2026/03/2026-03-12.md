## 🌐 Networking Concept: Konfigurasi Routing Statis Dasar

**Category:** Routing
**TL;DR:** Mengkonfigurasi jalur tetap secara manual pada router untuk mengarahkan lalu lintas data antar jaringan yang tidak terhubung langsung.

Routing statis adalah metode konfigurasi router di mana administrator jaringan secara manual mendefinisikan jalur spesifik untuk pengiriman paket data ke jaringan tujuan tertentu. Berbeda dengan routing dinamis yang menggunakan protokol untuk secara otomatis bertukar informasi rute, routing statis memerlukan intervensi manual untuk setiap perubahan topologi. Metode ini sering digunakan pada jaringan berskala kecil, atau untuk tujuan khusus seperti menyediakan rute backup atau rute default.

Cara kerja routing statis cukup sederhana: administrator menginstruksikan router tentang bagaimana mencapai suatu jaringan yang tidak terhubung langsung dengannya. Konfigurasi ini melibatkan penentuan alamat jaringan tujuan, subnet mask jaringan tersebut, dan kemudian "next-hop IP address" (alamat IP antarmuka router berikutnya di jalur) atau "exit interface" (antarmuka lokal router yang akan digunakan untuk mengirim paket). Router akan mengikuti jalur yang telah ditetapkan ini setiap kali ada paket yang ditujukan ke jaringan tujuan tersebut.

Kelebihan utama routing statis meliputi kesederhanaan implementasi pada jaringan kecil, peningkatan keamanan karena tidak ada pertukaran informasi routing dengan router lain, dan penggunaan sumber daya router yang rendah. Namun, kekurangannya adalah kurangnya skalabilitas pada jaringan besar karena setiap perubahan topologi memerlukan konfigurasi ulang manual, dan rentan terhadap kesalahan manusia. Selain itu, routing statis tidak dapat beradaptasi secara otomatis jika ada kegagalan jalur.

Dalam praktik dengan Cisco Packet Tracer, kita akan mensimulasikan skenario di mana dua atau lebih jaringan terpisah ingin saling berkomunikasi melalui beberapa router. Kita akan mengkonfigurasi rute statis pada setiap router agar setiap jaringan dapat 'menemukan' jaringan yang lain. Ini adalah latihan dasar yang sangat baik untuk memahami bagaimana router membuat keputusan pengiriman paket dan bagaimana mengelola konektivitas antar segmen jaringan yang berbeda.

```
Net A (192.168.1.0/24)         Net B (10.0.0.0/30)           Net C (172.16.1.0/24)
   |                             |         |                     |
[PC1] -- G0/0 [ Router1 ] G0/1 -- G0/0 [ Router2 ] G0/1 -- [PC2]
              192.168.1.1    10.0.0.1  10.0.0.2    172.16.1.1

Contoh: Router1 perlu route ke Net C (172.16.1.0) melalui Router2 (10.0.0.2).
```

**Cisco CLI:**
```
ip route 172.16.1.0 255.255.255.0 10.0.0.2
```

💡 **Fun Fact:** Meskipun 'statis', route ini dapat memiliki administrative distance (AD) yang berbeda. Secara default, route statis yang menunjuk ke next-hop IP memiliki AD 1, sedangkan route statis yang menunjuk ke exit interface memiliki AD 0, menjadikannya rute yang paling disukai oleh router.
