## ☕ Java Concept: Penanganan Exception Dasar

**Category:** Exception
**TL;DR:** Mekanisme Java untuk mengelola error tak terduga (runtime errors) agar program tidak crash dan tetap berjalan dengan elegan.

Dalam pemrograman, seringkali terjadi hal-hal yang tidak terduga saat program sedang berjalan, seperti mencoba membagi angka dengan nol, mengakses elemen di luar batas array, atau membaca file yang tidak ada. Kejadian-kejadian inilah yang dikenal sebagai "exception" atau pengecualian. Jika tidak ditangani, exception dapat menyebabkan program berhenti mendadak (crash), yang tentu saja tidak diinginkan dalam aplikasi yang stabil dan ramah pengguna. Penanganan exception adalah cara kita mengantisipasi dan merespons situasi-situasi ini.

Java menyediakan mekanisme penanganan exception yang kuat menggunakan blok `try-catch`. Kode yang berpotensi menimbulkan exception diletakkan di dalam blok `try`. Jika exception terjadi di dalam blok `try`, eksekusi kode akan segera berhenti di titik tersebut, dan kontrol akan dialihkan ke blok `catch` yang sesuai. Blok `catch` inilah tempat kita menulis kode untuk merespons atau memperbaiki masalah yang terjadi, misalnya dengan menampilkan pesan error kepada pengguna atau melakukan logging untuk tujuan debugging.

Dengan penanganan exception, kita bisa membuat program menjadi lebih tangguh (robust). Alih-alih crash, program bisa "memulihkan diri" dari kesalahan atau setidaknya memberikan informasi yang jelas kepada pengguna tentang apa yang salah, sehingga pengalaman pengguna menjadi lebih baik. Ini juga membantu developer dalam melakukan debugging karena lokasi dan jenis kesalahan seringkali bisa dilacak melalui informasi exception yang tertangkap.

Selain `try` dan `catch`, ada juga blok `finally` (meskipun tidak wajib). Kode di dalam blok `finally` akan selalu dieksekusi, terlepas dari apakah exception terjadi atau tidak. Ini sangat berguna untuk membersihkan sumber daya, seperti menutup file atau koneksi database, yang harus selalu dilakukan agar tidak terjadi kebocoran sumber daya, menjadikannya pilihan ideal untuk memastikan sumber daya dilepaskan dengan benar.

```
public class ExceptionDemo {
    public static void main(String[] args) {
        try {
            int numerator = 10;
            int denominator = 0;
            int result = numerator / denominator; // Ini akan melempar ArithmeticException
            System.out.println("Hasil pembagian: " + result);
        } catch (ArithmeticException e) {
            System.err.println("Terjadi kesalahan aritmatika: " + e.getMessage());
            System.err.println("Tidak bisa membagi dengan nol!");
        }
        System.out.println("Program selesai, meskipun ada error!");
    }
}
```

💡 **Fun Fact:** Sistem exception Java, yang diperkenalkan sejak versi 1.0, adalah salah satu fitur awal yang membedakannya dari banyak bahasa lain pada masanya, membantu developer menulis kode yang lebih tangguh dan mudah di-debug.
