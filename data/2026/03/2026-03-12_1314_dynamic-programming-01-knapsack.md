## ⚡ Competitive Programming Concept: Dynamic Programming: 0/1 Knapsack

**Category:** DP
**TL;DR:** Mencari subset item dengan berat total tidak melebihi kapasitas tas dan nilai total maksimum menggunakan Dynamic Programming.

Konsep Dynamic Programming (DP) adalah metode kuat untuk memecahkan masalah dengan memecahnya menjadi sub-masalah yang lebih kecil dan menyimpan hasilnya untuk menghindari perhitungan ulang. Salah satu masalah klasik yang dipecahkan dengan DP adalah masalah 0/1 Knapsack. Dalam masalah ini, Anda diberikan N buah item, di mana setiap item memiliki berat (`w_i`) dan nilai (`v_i`). Anda juga memiliki sebuah tas (knapsack) dengan kapasitas maksimum (`W`). Tujuan Anda adalah memilih subset item sedemikian rupa sehingga total berat item yang dipilih tidak melebihi `W`, dan total nilai item yang dipilih adalah yang paling maksimum. Masalah ini disebut 0/1 karena setiap item hanya bisa diambil utuh (1) atau tidak sama sekali (0), tidak bisa sebagian.

Untuk menyelesaikan masalah 0/1 Knapsack, kita mendefinisikan tabel DP, umumnya dalam bentuk 2D. Misalkan `dp[i][j]` menyatakan nilai maksimum yang dapat diperoleh dengan mempertimbangkan `i` item pertama (dari indeks 0 hingga `i-1`) dan kapasitas tas sebesar `j`. Untuk mengisi tabel ini, kita mempertimbangkan item ke-`i` (item dengan indeks `i-1` dalam array `weights` dan `values`): Jika berat item ke-`i` (`weights[i-1]`) lebih besar dari kapasitas `j`, maka item ini tidak dapat diambil, sehingga `dp[i][j] = dp[i-1][j]`. Jika berat item ke-`i` tidak lebih besar dari kapasitas `j`, kita punya dua pilihan: tidak mengambil item ke-`i` (nilainya `dp[i-1][j]`) atau mengambil item ke-`i` (nilainya `values[i-1] + dp[i-1][j - weights[i-1]]`). Kita mengambil nilai maksimum dari kedua pilihan ini: `dp[i][j] = max(dp[i-1][j], values[i-1] + dp[i-1][j - weights[i-1]])`.

Base case untuk tabel DP ini adalah `dp[0][j] = 0` untuk semua `j` (tidak ada item, nilai 0) dan `dp[i][0] = 0` untuk semua `i` (kapasitas 0, nilai 0). Kita mengisi tabel secara iteratif, mulai dari `i=1` hingga `N` dan `j=0` hingga `W`. Setelah tabel terisi penuh, hasil akhir yang kita cari adalah `dp[N][W]`. Karena kita mengisi tabel berukuran `N x W` dan setiap sel membutuhkan waktu konstan `O(1)` untuk dihitung, kompleksitas waktu keseluruhan algoritma ini adalah `O(N * W)`. Kompleksitas ruangnya juga `O(N * W)` untuk menyimpan tabel DP, meskipun dapat dioptimalkan menjadi `O(W)` dengan hanya menyimpan dua baris atau satu baris sebelumnya.

```
vector<int> weights = {10, 20, 30};
vector<int> values = {60, 100, 120};
int N = weights.size(); // Jumlah item
int W = 50; // Kapasitas tas

vector<vector<int>> dp(N + 1, vector<int>(W + 1, 0));

for (int i = 1; i <= N; ++i) {
    for (int w = 0; w <= W; ++w) {
        if (weights[i-1] <= w) {
            dp[i][w] = max(dp[i-1][w], values[i-1] + dp[i-1][w - weights[i-1]]);
        } else {
            dp[i][w] = dp[i-1][w];
        }
    }
}
// Nilai maksimum adalah dp[N][W]
```

💡 **Fun Fact:** Masalah Knapsack adalah salah satu dari 21 masalah NP-complete yang diidentifikasi oleh Richard Karp pada tahun 1972. Ini berarti tidak ada algoritma waktu polinomial yang diketahui untuk menyelesaikannya secara umum, meskipun DP memberikan solusi pseudo-polinomial (bergantung pada nilai W).
