## ⚡ Competitive Programming Concept: Dynamic Programming: Memoization & Tabulation

**Category:** DP
**TL;DR:** Metode optimasi rekursi dengan menyimpan hasil sub-problem untuk menghindari komputasi berulang, dibagi menjadi pendekatan top-down (memoization) dan bottom-up (tabulation).

Dynamic Programming (DP) adalah teknik optimasi yang digunakan untuk memecahkan masalah kompleks dengan memecahnya menjadi sub-masalah yang lebih kecil. DP efektif ketika masalah menunjukkan dua properti utama: _optimal substructure_ (solusi optimal untuk masalah dapat dibangun dari solusi optimal sub-masalah) dan _overlapping subproblems_ (sub-masalah yang sama muncul berulang kali). Tujuan utamanya adalah menghindari komputasi berulang pada sub-masalah yang sama.

Ada dua pendekatan utama dalam DP. Pendekatan pertama adalah **Memoization (Top-Down DP)**. Dalam pendekatan ini, kita memulai dari masalah utama dan secara rekursif memecahkannya menjadi sub-masalah. Hasil dari setiap sub-masalah disimpan dalam tabel (sering disebut array `dp` atau map) setelah dihitung pertama kali. Jika sub-masalah yang sama muncul lagi, hasilnya langsung diambil dari tabel, bukan dihitung ulang. Ini menggabungkan rekursi dengan caching.

Pendekatan kedua adalah **Tabulation (Bottom-Up DP)**. Berbeda dengan memoization, tabulation membangun solusi dari sub-masalah terkecil hingga masalah utama secara iteratif. Kita mengisi tabel `dp` dimulai dari kasus dasar (base cases) dan secara bertahap menghitung solusi untuk masalah yang lebih besar berdasarkan solusi sub-masalah yang sudah dihitung sebelumnya. Ini seringkali lebih mudah diimplementasikan tanpa rekursi, sehingga menghindari isu stack overflow pada input yang sangat besar.

Kedua pendekatan ini secara drastis mengurangi kompleksitas waktu dari yang mungkin eksponensial (tanpa DP) menjadi polinomial. Misalnya, untuk menghitung deret Fibonacci, kompleksitas waktu tanpa DP adalah O(2^N), sedangkan dengan DP (baik memoization maupun tabulation) menjadi O(N). Kompleksitas ruang biasanya O(N) atau O(N*M) tergantung pada dimensi ruang status yang diperlukan untuk menyimpan hasil sub-masalah.

```
std::vector<int> dp; // Harus diinisialisasi: dp.assign(n + 1, -1);
int fib_memo(int n) {
    if (n <= 1) return n;
    if (dp[n] != -1) return dp[n];
    return dp[n] = fib_memo(n - 1) + fib_memo(n - 2);
}
```

💡 **Fun Fact:** Istilah 'Dynamic Programming' diciptakan oleh Richard Bellman pada tahun 1950-an. Ia memilih kata 'dynamic' karena terdengar lebih menarik di mata para atasan di Pentagon, dan 'programming' mengacu pada 'perencanaan' atau 'penjadwalan', bukan pemrograman komputer.
