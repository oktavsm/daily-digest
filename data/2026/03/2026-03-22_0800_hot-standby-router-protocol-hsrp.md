## 🌐 Networking Concept: Hot Standby Router Protocol (HSRP)

**Category:** Routing
**TL;DR:** HSRP menyediakan redundansi default gateway untuk host di LAN dengan memungkinkan beberapa router berbagi satu alamat IP virtual dan MAC address.

Hari ini kita akan mendalami konsep penting untuk ketersediaan jaringan yang tinggi, yaitu Hot Standby Router Protocol (HSRP). Dalam jaringan lokal (LAN), setiap host membutuhkan alamat default gateway untuk berkomunikasi ke luar subnet-nya. Jika router yang berfungsi sebagai default gateway tunggal mengalami kegagalan, seluruh host di subnet tersebut akan terisolasi. HSRP dirancang untuk mengatasi masalah single point of failure ini dengan menyediakan redundansi gateway.

HSRP bekerja dengan mengelompokkan dua atau lebih router ke dalam sebuah 'grup standby'. Dalam grup ini, satu router akan berperan sebagai router 'Active', yang bertanggung jawab meneruskan semua lalu lintas yang ditujukan ke alamat IP virtual dan alamat MAC virtual yang dibagikan oleh grup. Router lainnya akan berfungsi sebagai router 'Standby', yang siap mengambil alih peran Active jika router Active saat ini gagal. Host di LAN hanya perlu dikonfigurasi dengan alamat IP virtual ini sebagai default gateway mereka.

Untuk menentukan siapa yang menjadi Active atau Standby, HSRP menggunakan sistem prioritas. Router dengan prioritas tertinggi akan menjadi Active. Fitur 'preemption' memungkinkan router dengan prioritas lebih tinggi untuk mengambil alih peran Active jika ia online kembali setelah mengalami kegagalan. HSRP juga menggunakan pesan 'hello' periodik antar anggota grup untuk memantau status satu sama lain. Jika router Active tidak mengirimkan hello dalam waktu tertentu, router Standby akan mengambil alih. Hal ini memastikan transisi yang mulus dan cepat, menjaga konektivitas jaringan tetap stabil.

Dengan Cisco Packet Tracer, kita bisa mensimulasikan skenario ini dengan mudah. Kita dapat mengkonfigurasi dua router (atau multilayer switch) sebagai anggota grup HSRP, melihat bagaimana mereka memilih Active/Standby, dan menguji proses failover dengan mematikan salah satu interface router. Konsep ini sangat relevan dalam desain jaringan enterprise dan data center untuk memastikan ketersediaan layanan yang tidak terputus.

```
standby 1 ip 192.168.1.254
```

💡 **Fun Fact:** HSRP adalah protokol milik proprietary Cisco, yang berarti hanya bekerja pada perangkat Cisco. Namun, karena dominasi Cisco, HSRP menjadi standar de facto dan banyak digunakan. Protokol serupa yang bersifat standar terbuka adalah Virtual Router Redundancy Protocol (VRRP).
