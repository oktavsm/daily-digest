## 🤖 Android Concept: Activity: Layar Aplikasi Pertama

**Category:** UI
**TL;DR:** Activity adalah komponen utama yang menyediakan satu layar antarmuka pengguna (UI) tempat interaksi pengguna terjadi.

Pada dasarnya, setiap aplikasi Android yang Anda gunakan terdiri dari satu atau lebih "Activity". Bayangkan Activity sebagai satu "layar" dalam aplikasi Anda. Misalnya, saat Anda membuka aplikasi WhatsApp dan melihat daftar chat, itu adalah sebuah Activity. Ketika Anda mengklik salah satu chat dan masuk ke percakapan, itu adalah Activity lain. Activity bertugas untuk mengelola tampilan UI dan berinteraksi dengan pengguna di layar tersebut.

Setiap Activity memiliki siklus hidup (lifecycle) yang terdefinisi dengan baik, mulai dari saat ia dibuat, terlihat oleh pengguna, berinteraksi, berhenti sementara, hingga akhirnya dihancurkan. Metode-metode seperti `onCreate()`, `onStart()`, `onResume()`, `onPause()`, `onStop()`, dan `onDestroy()` adalah bagian dari siklus hidup ini. Memahami lifecycle sangat penting agar aplikasi Anda berperilaku benar, terutama saat pengguna berpindah antar layar, menerima panggilan telepon, atau memutar orientasi perangkat.

Untuk membuat Activity baru, Anda perlu mendeklarasikan sebuah class yang mewarisi dari `AppCompatActivity` dan mengimplementasikan metode `onCreate()`. Di dalam `onCreate()`, Anda biasanya akan memanggil `setContentView()` untuk menghubungkan Activity dengan file layout XML yang mendefinisikan tampilan UI-nya. Activity ini kemudian harus didaftarkan di file `AndroidManifest.xml` agar sistem Android dapat mengenalinya dan meluncurkannya.

Activity merupakan fondasi utama dari interaksi pengguna di aplikasi Android. Meskipun ada pendekatan modern seperti Jetpack Compose yang mengurangi kebutuhan akan file layout XML terpisah, konsep dasar Activity sebagai unit layar yang mengelola UI dan interaksi tetap relevan. Menguasai Activity adalah langkah pertama dan paling fundamental dalam membangun aplikasi Android yang fungsional.

```
package com.example.myapp

import android.os.Bundle
import androidx.appcompat.app.AppCompatActivity

class MainActivity : AppCompatActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main) // Menghubungkan ke layout XML
    }
}
```

💡 **Fun Fact:** Activity sudah ada sejak versi paling awal Android 1.0, bahkan sebelum ada App Store (sekarang Google Play Store). Pada awalnya, Activity tidak memiliki `AppCompatActivity`; itu ditambahkan kemudian untuk menyediakan kompatibilitas fitur-fitur baru pada versi Android yang lebih lama.
