## ⚡ CompProg Concept: Segment Tree

**Category:** Data Structure
**TL;DR:** Segment Tree adalah struktur data pohon yang memungkinkan query range dan update titik/range secara efisien pada sebuah array.

Segment Tree adalah struktur data pohon biner yang digunakan untuk menyimpan informasi tentang interval atau 'segmen' dalam sebuah array. Setiap node di Segment Tree merepresentasikan suatu sub-interval dari array asli, dengan node akar merepresentasikan seluruh array, dan node daun merepresentasikan elemen tunggal. Struktur ini sangat efisien untuk melakukan query pada rentang tertentu (misalnya, mencari sum, minimum, atau maksimum dalam sebuah rentang) dan melakukan update pada elemen individual atau rentang elemen. Ini adalah salah satu struktur data fundamental dalam competitive programming.

Pembangunan (building) Segment Tree dilakukan secara rekursif. Kita membagi array menjadi dua bagian pada setiap langkah, membangun pohon untuk setiap bagian, dan kemudian menggabungkan hasilnya. Sebagai contoh, jika sebuah node merepresentasikan interval `[L, R]`, anak kirinya akan merepresentasikan `[L, (L+R)/2]` dan anak kanannya merepresentasikan `[(L+R)/2 + 1, R]`. Nilai yang disimpan di node internal biasanya merupakan hasil operasi (misal: penjumlahan, minimum) dari nilai-nilai di anak-anaknya. Proses query bekerja dengan memecah rentang query menjadi beberapa sub-rentang yang persis ditutupi oleh node-node di pohon, lalu menggabungkan hasilnya. Update satu titik juga bersifat rekursif, menelusuri jalur dari akar ke daun yang relevan, memperbarui nilainya, lalu memperbarui semua node di jalur kembali ke atas.

Segment Tree menawarkan efisiensi tinggi untuk operasi yang sering muncul di competitive programming. Proses pembangunan (build) pohon membutuhkan waktu O(N), di mana N adalah ukuran array, karena setiap elemen array hanya diakses sekali untuk mengisi node daun dan kemudian propagate ke atas. Baik operasi query maupun update membutuhkan waktu O(log N). Ini karena pada setiap operasi, kita hanya menelusuri sebuah jalur tunggal dari akar ke daun (untuk update) atau paling banyak mengunjungi O(log N) node yang menutupi rentang query. Kompleksitas ruang yang dibutuhkan adalah O(N) karena pohon biner akan memiliki sekitar 2N hingga 4N node, tergantung implementasi.

```
vector<long long> tree;
vector<int> arr; // Input array
int N_orig; // Size of input array

void build(int node, int start, int end) {
    if (start == end) {
        tree[node] = arr[start];
    } else {
        int mid = (start + end) / 2;
        build(2 * node, start, mid);
        build(2 * node + 1, mid + 1, end);
        tree[node] = tree[2 * node] + tree[2 * node + 1];
    }
}
```

💡 **Fun Fact:** Segment Tree adalah generalisasi dari range sum query dan bisa dimodifikasi untuk lazy propagation, yang memungkinkan update range secara efisien pada rentang waktu yang sama. Konsep dasarnya mirip dengan prinsip Divide and Conquer.
