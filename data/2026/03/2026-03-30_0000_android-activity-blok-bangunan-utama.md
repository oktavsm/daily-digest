## 🤖 Android Concept: Android Activity: Blok Bangunan Utama

**Category:** UI
**TL;DR:** Activity adalah komponen fundamental yang menyediakan satu layar antarmuka pengguna (UI) bagi pengguna untuk berinteraksi dalam aplikasi Android.

Activity adalah salah satu blok bangunan paling dasar di aplikasi Android Anda. Bayangkan Activity sebagai satu layar tunggal dalam aplikasi Anda, seperti layar login, layar daftar produk, atau layar detail profil. Setiap kali pengguna melihat atau berinteraksi dengan sesuatu di layar, kemungkinan besar mereka sedang berinteraksi dengan sebuah Activity. Tujuan utamanya adalah menyediakan tempat bagi antarmuka pengguna (UI) Anda dan mengatur interaksi pengguna.

Setiap Activity memiliki siklus hidup (lifecycle) yang terdefinisi dengan baik, mulai dari dibuat hingga dihancurkan. Beberapa metode penting dalam siklus hidup ini meliputi `onCreate()`, `onStart()`, `onResume()`, `onPause()`, `onStop()`, dan `onDestroy()`. Memahami siklus hidup ini sangat krusial karena Anda perlu mengelola sumber daya, menyimpan status aplikasi, dan merespons interaksi pengguna pada tahap yang tepat. Misalnya, `onCreate()` adalah tempat ideal untuk melakukan inisialisasi awal, seperti memuat tata letak UI (layout).

Anda membuat Activity dengan meng-extend class `AppCompatActivity` atau `Activity`. Dalam metode `onCreate()`, Anda biasanya memanggil `setContentView()` untuk mengaitkan Activity dengan file layout XML yang mendefinisikan tampilan UI-nya. Untuk berpindah dari satu Activity ke Activity lain, Anda menggunakan objek `Intent`, yang bertindak seperti "niat" untuk melakukan suatu tindakan, misalnya memulai Activity baru atau mengirim data antar komponen.

Meskipun Jetpack Compose semakin populer untuk membangun UI, Activity tetap merupakan pondasi untuk banyak aspek aplikasi Android. Ini adalah titik masuk utama bagi sebagian besar interaksi pengguna dan bertanggung jawab untuk mengatur bagaimana UI ditampilkan dan bagaimana aplikasi merespons masukan pengguna. Menguasai konsep Activity adalah langkah pertama yang esensial dalam membangun aplikasi Android yang berfungsi dengan baik.

```
package com.example.myfirstapp

import android.os.Bundle
import androidx.appcompat.app.AppCompatActivity

class MainActivity : AppCompatActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main) // Mengaitkan layout XML
        // Di sini Anda bisa menambahkan logika UI lainnya
    }
}
```

💡 **Fun Fact:** Tahukah Anda bahwa nama Activity merujuk pada gagasan bahwa komponen ini adalah tempat aplikasi melakukan 'aktivitas' tertentu, seperti menampilkan daftar atau mengedit data?
