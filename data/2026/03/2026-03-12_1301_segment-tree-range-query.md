## ⚡ Competitive Programming Concept: Segment Tree: Range Query

**Category:** Data Structure
**TL;DR:** Segment Tree adalah struktur data pohon biner yang efisien untuk melakukan operasi query dan update pada rentang (range) indeks dalam sebuah array dengan kompleksitas waktu logaritmik.

Segment Tree adalah struktur data berbentuk pohon biner yang memungkinkan kita melakukan operasi pada rentang (range) elemen dalam array dengan sangat efisien. Setiap node dalam pohon merepresentasikan sebuah rentang indeks dari array asli. Node akar merepresentasikan seluruh array, dan node-node turunannya merepresentasikan sub-rentang yang lebih kecil. Sebuah node internal akan memiliki dua anak yang masing-masing merepresentasikan separuh dari rentang induknya. Node daun merepresentasikan elemen tunggal dari array asli. Struktur ini sangat berguna untuk operasi yang bersifat asosiatif dan komutatif, seperti mencari jumlah, minimum, maksimum, atau XOR dari sebuah rentang.

Proses pembangunan (build) Segment Tree dilakukan secara rekursif. Dimulai dari akar yang merepresentasikan seluruh array, kita membagi rentang menjadi dua sub-rentang yang sama besar untuk anak kiri dan kanan. Proses ini berlanjut hingga mencapai node daun yang merepresentasikan satu elemen. Nilai pada node induk dihitung berdasarkan nilai anak-anaknya (misalnya, jumlah anak kiri + jumlah anak kanan untuk operasi sum). Proses build memakan waktu O(N) karena setiap elemen array dan setiap node di pohon dikunjungi sekali. Operasi query (misalnya, mencari sum dari indeks L sampai R) juga dilakukan secara rekursif, membagi rentang query menjadi bagian-bagian yang dapat dijawab oleh node-node di pohon. Jika rentang node sepenuhnya tercakup dalam rentang query, kita langsung mengembalikan nilai node tersebut; jika tidak, kita rekursif ke anak-anak yang relevan. Kompleksitas waktu untuk query adalah O(log N).

Selain query, Segment Tree juga mendukung operasi update pada titik (point update) atau rentang (range update). Untuk point update, kita mencari node daun yang merepresentasikan indeks yang diupdate, mengubah nilainya, lalu secara rekursif memperbarui semua node induk di jalur dari daun tersebut hingga ke akar. Setiap node induk akan menghitung ulang nilainya berdasarkan anak-anaknya. Proses ini juga memiliki kompleksitas waktu O(log N). Untuk range update yang lebih kompleks (misalnya menambahkan nilai ke seluruh rentang), seringkali digunakan teknik yang disebut 'Lazy Propagation' untuk menjaga efisiensi agar tetap O(log N).

Secara keseluruhan, Segment Tree menawarkan keseimbangan yang sangat baik antara kecepatan query dan update. Dengan kompleksitas waktu O(log N) untuk kedua operasi tersebut, serta kompleksitas ruang O(N) (biasanya sekitar 2N hingga 4N node), menjadikannya salah satu struktur data fundamental dan paling sering digunakan dalam competitive programming untuk memecahkan berbagai masalah yang melibatkan manipulasi dan pertanyaan pada rentang data.

```
// Asumsi 'tree' adalah array global untuk menyimpan node segment tree,
// dan operasi yang diinginkan adalah penjumlahan rentang.
int query(int node, int start, int end, int L, int R) {
    // Rentang query (L, R) tidak tumpang tindih dengan rentang node (start, end)
    if (R < start || end < L) return 0; // Return identity element for sum
    // Rentang query (L, R) sepenuhnya mencakup rentang node (start, end)
    if (L <= start && end <= R) return tree[node];
    // Rentang query tumpang tindih sebagian, rekursif ke anak
    int mid = (start + end) / 2;
    int p1 = query(2 * node, start, mid, L, R);
    int p2 = query(2 * node + 1, mid + 1, end, L, R);
    return p1 + p2; // Gabungkan hasil dari anak-anak
}
```

💡 **Fun Fact:** Segment Tree sering disebut sebagai 'generalization of prefix sums' karena ia memungkinkan query rentang dan update titik dalam waktu logaritmik, tidak hanya untuk sum tetapi juga operasi lain seperti min, max, atau XOR, yang tidak bisa dilakukan hanya dengan prefix sums.
