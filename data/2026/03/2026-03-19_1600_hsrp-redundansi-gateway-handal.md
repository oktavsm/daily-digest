## 🌐 Networking Concept: HSRP: Redundansi Gateway Handal

**Category:** Routing
**TL;DR:** HSRP memastikan ketersediaan gateway default yang tinggi dengan memungkinkan dua atau lebih router berbagi satu alamat IP virtual dan MAC address.

Mahasiswa sekalian, bayangkan sebuah jaringan LAN tempat semua perangkat mengandalkan satu router sebagai gateway default mereka. Apa yang terjadi jika router tersebut tiba-tiba mati? Seluruh perangkat dalam segmen LAN tersebut akan kehilangan akses ke jaringan luar, internet, atau bahkan segmen jaringan lain. Ini adalah sebuah single point of failure yang kritis dan dapat menyebabkan downtime yang signifikan.

Di sinilah HSRP (Hot Standby Router Protocol) datang sebagai solusi elegan. HSRP adalah protokol redundansi first-hop milik Cisco yang memungkinkan dua atau lebih router untuk bekerja sama, berbagi satu alamat IP dan MAC address virtual. Alamat IP virtual inilah yang akan dikonfigurasi sebagai gateway default pada semua perangkat akhir di LAN Anda. Dengan demikian, dari sudut pandang perangkat klien, gateway mereka selalu aktif dan tidak pernah berubah.

Bagaimana HSRP mencapai ini? Router dalam grup HSRP akan mengambil peran Active dan Standby. Router Active adalah router yang secara aktif merutekan traffic untuk alamat IP virtual tersebut, sementara router Standby siap mengambil alih peran Active jika router Active gagal. Mereka saling bertukar pesan 'hello' secara teratur untuk memantau status satu sama lain. Pemilihan router Active didasarkan pada nilai prioritas yang dikonfigurasi; router dengan prioritas tertinggi akan menjadi Active. Fitur preemption juga dapat diaktifkan, memastikan router dengan prioritas lebih tinggi yang baru pulih dapat mengambil kembali peran Active.

Dengan HSRP, kegagalan pada satu router fisik tidak akan menyebabkan gangguan layanan bagi pengguna. Perangkat akhir tidak perlu mengetahui perubahan router fisik di belakang layar, karena mereka selalu berkomunikasi dengan alamat IP virtual yang sama. Ini meningkatkan ketersediaan jaringan secara drastis, menjamin konektivitas yang lebih stabil. Di Cisco Packet Tracer, kita bisa mensimulasikan skenario ini untuk melihat bagaimana failover HSRP bekerja secara otomatis dan transparan.

```
standby 1 ip 192.168.1.254
```

💡 **Fun Fact:** Tahukah Anda bahwa HSRP adalah protokol milik eksklusif Cisco? Pesaing terbukanya adalah VRRP (Virtual Router Redundancy Protocol), yang bekerja dengan konsep serupa dan dapat diimplementasikan di perangkat multi-vendor.
