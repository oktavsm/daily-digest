## ☕ Java Concept: Penggunaan `ArrayList` untuk Data Dinamis

**Category:** Collections
**TL;DR:** `ArrayList` adalah struktur data dinamis yang memungkinkan kita menyimpan koleksi objek dan mengubah ukurannya sesuai kebutuhan.

Dalam pemrograman Java, seringkali kita perlu menyimpan sekumpulan data yang jumlahnya tidak tetap atau dapat berubah selama program berjalan. Array standar memiliki ukuran tetap setelah dibuat, yang bisa menjadi batasan. Di sinilah `ArrayList` menjadi solusi yang sangat praktis. `ArrayList` adalah bagian dari Java Collections Framework, yang menyediakan implementasi struktur data seperti list, set, dan map. Secara fundamental, `ArrayList` dapat dibayangkan sebagai "array yang fleksibel", di mana kita bisa dengan mudah menambah atau mengurangi elemen tanpa perlu khawatir tentang kapasitas awal.

Untuk mulai menggunakan `ArrayList`, kita perlu mendeklarasikannya dan menginisialisasinya. Saat mendeklarasikan, penting untuk menentukan jenis (tipe) objek yang akan disimpan di dalamnya, misalnya `ArrayList<String>` untuk menyimpan teks atau `ArrayList<Integer>` untuk bilangan bulat. Penentuan tipe ini, yang dikenal sebagai *generics*, membantu Java memastikan keamanan tipe (type safety) sehingga kita tidak secara tidak sengaja memasukkan objek dengan tipe yang salah. Setelah `ArrayList` dibuat, kita bisa menambahkan elemen menggunakan metode `add()`, mengambil elemen dengan `get(index)`, atau mengetahui jumlah elemen saat ini dengan `size()`.

`ArrayList` sangat sering digunakan karena kemudahan dan fleksibilitasnya dalam mengelola daftar data. Selain menambah dan mengambil elemen, kita juga dapat menghapus elemen dengan `remove(index)` atau `remove(object)`, mencari elemen, dan melakukan iterasi (mengulang) di seluruh elemen menggunakan *enhanced for-loop* (`for-each`). Kemampuannya untuk secara otomatis menyesuaikan ukurannya membuat `ArrayList` menjadi pilihan utama ketika kita perlu mengelola koleksi objek yang jumlahnya bervariasi, seperti daftar nama pelanggan, nilai ujian siswa, atau item dalam keranjang belanja.

```
import java.util.ArrayList;

public class MahasiswaApp {
    public static void main(String[] args) {
        ArrayList<String> daftarNamaMahasiswa = new ArrayList<>();

        daftarNamaMahasiswa.add("Budi");
        daftarNamaMahasiswa.add("Ani");
        daftarNamaMahasiswa.add("Cahyo");

        System.out.println("Daftar Mahasiswa: " + daftarNamaMahasiswa);
        System.out.println("Mahasiswa pertama: " + daftarNamaMahasiswa.get(0));

        daftarNamaMahasiswa.remove(1); // Menghapus "Ani"
        System.out.println("Daftar setelah penghapusan: " + daftarNamaMahasiswa);
    }
}
```

💡 **Fun Fact:** Secara internal, `ArrayList` sebenarnya menggunakan array biasa untuk menyimpan elemen-elemennya. Ketika `ArrayList` mencapai kapasitas maksimum array internalnya, ia akan secara otomatis membuat array baru yang lebih besar (biasanya 1.5 kali ukuran semula) dan menyalin semua elemen lama ke array baru tersebut. Proses ini disebut *resizing*, dan merupakan salah satu alasan mengapa `ArrayList` bisa "dinamis".
