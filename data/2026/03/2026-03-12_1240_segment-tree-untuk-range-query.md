## ⚡ Competitive Programming Concept: Segment Tree untuk Range Query

**Category:** Data Structure
**TL;DR:** Segment Tree adalah struktur data pohon biner yang efisien untuk melakukan query pada range (misalnya sum, min, max) dan update pada titik tunggal atau range dalam array, dengan kompleksitas waktu logaritmik.

Segment Tree adalah struktur data berbasis pohon biner yang dirancang untuk menyimpan informasi tentang interval atau segmen. Kegunaan utamanya adalah untuk mendukung dua jenis operasi secara efisien: melakukan query pada suatu range (misalnya mencari jumlah, nilai minimum, atau nilai maksimum dalam subarray antara indeks L dan R) dan melakukan update pada elemen tunggal atau seluruh range dalam array. Keunggulan utamanya terletak pada kompleksitas waktu yang logaritmik (O(log N)) untuk operasi-operasi tersebut, jauh lebih cepat dibandingkan dengan O(N) jika dilakukan secara naif pada array biasa.

Struktur Segment Tree dibangun secara rekursif, di mana setiap node dalam pohon merepresentasikan sebuah interval dari array asli. Node root mewakili seluruh array [0, N-1]. Setiap node internal memiliki dua anak: anak kiri merepresentasikan paruh kiri dari interval node tersebut, dan anak kanan merepresentasikan paruh kanan. Proses ini berlanjut hingga mencapai leaf node, yang masing-masing merepresentasikan elemen tunggal dari array asli. Dengan cara ini, setiap query range dapat dipecah menjadi gabungan beberapa interval yang sudah dihitung dan disimpan di node-node pohon, memungkinkan pencarian atau agregasi nilai yang cepat.

Tiga operasi utama pada Segment Tree adalah `build`, `query`, dan `update`. Operasi `build` membangun pohon dari array awal, dengan kompleksitas waktu O(N) karena setiap elemen dan node diproses sekali. Operasi `query` mengambil informasi dari range [L, R], melibatkan traversal pohon yang mengunjungi paling banyak O(log N) node, sehingga kompleksitasnya O(log N). Operasi `update` mengubah nilai elemen pada indeks tertentu dan memperbarui nilai pada node-node ancestor yang terpengaruh, juga dengan kompleksitas O(log N). Segment Tree sangat fleksibel dan dapat digunakan untuk berbagai operasi asosiatif seperti penjumlahan, minimum, maksimum, GCD, atau XOR, dan bahkan bisa diperluas dengan Lazy Propagation untuk update range yang lebih efisien.

```
vector<int> tree;
vector<int> arr; // Diasumsikan arr sudah terisi dan 0-indexed

// N adalah ukuran array asli, tree biasanya membutuhkan 4*N ruang
void build(int node_idx, int start, int end) {
    if (start == end) {
        tree[node_idx] = arr[start]; // Leaf node menyimpan nilai asli
    } else {
        int mid = (start + end) / 2;
        build(2 * node_idx, start, mid); // Bangun anak kiri
        build(2 * node_idx + 1, mid + 1, end); // Bangun anak kanan
        tree[node_idx] = tree[2 * node_idx] + tree[2 * node_idx + 1]; // Agregasi hasil (contoh: sum)
    }
}
```

💡 **Fun Fact:** Segment Trees pertama kali diperkenalkan oleh Jon Bentley pada tahun 1977. Nama 'Segment Tree' itu sendiri tidak mengacu pada bentuk pohonnya yang bersegmen, melainkan pada kemampuan struktur ini untuk mengelola dan melakukan query secara efisien pada segmen atau interval data.
