## ⚡ Competitive Programming Concept: Longest Common Subsequence (LCS)

**Category:** DP
**TL;DR:** Mencari urutan karakter terpanjang yang muncul di dua string, tidak harus berurutan, namun mempertahankan urutan relatifnya.

Longest Common Subsequence (LCS) adalah masalah klasik dalam pemrograman dinamis (DP) yang bertujuan untuk menemukan urutan karakter terpanjang yang merupakan subsequence dari dua string yang diberikan. Berbeda dengan substring yang harus berurutan, subsequence tidak perlu berurutan di string aslinya, namun harus mempertahankan urutan relatif karakter-karakternya. Misalnya, "ACE" adalah subsequence dari "ABCDE", tetapi "AEC" bukan. Karena sifat optimal substructure (solusi optimal dapat dibangun dari solusi optimal sub-masalah) dan overlapping subproblems (sub-masalah yang sama dihitung berulang kali), DP menjadi pendekatan yang sangat cocok untuk masalah ini.

Untuk menyelesaikan LCS dengan DP, kita bisa mendefinisikan `dp[i][j]` sebagai panjang LCS dari prefix pertama `i` karakter dari string pertama (`s1[0...i-1]`) dan prefix pertama `j` karakter dari string kedua (`s2[0...j-1]`). Terdapat dua kasus utama saat membandingkan karakter `s1[i-1]` dan `s2[j-1]`. Jika kedua karakter tersebut sama, maka kita bisa menambahkan 1 ke panjang LCS dari `s1[0...i-2]` dan `s2[0...j-2]`. Jadi, `dp[i][j] = 1 + dp[i-1][j-1]`.

Jika `s1[i-1]` dan `s2[j-1]` berbeda, maka karakter tersebut tidak bisa menjadi bagian dari LCS pada posisi tersebut secara bersamaan. Dalam kasus ini, panjang LCS `dp[i][j]` adalah maksimum dari dua kemungkinan: LCS dari `s1[0...i-1]` dan `s2[0...j-2]` (mengabaikan `s2[j-1]`), atau LCS dari `s1[0...i-2]` dan `s2[0...j-1]` (mengabaikan `s1[i-1]`). Secara matematis, `dp[i][j] = max(dp[i-1][j], dp[i][j-1])`. Kasus dasar (base cases) adalah ketika salah satu string kosong, maka LCS-nya adalah 0, yaitu `dp[0][j] = 0` dan `dp[i][0] = 0` untuk semua `i` dan `j`.

Tabel DP `dp[N+1][M+1]` (di mana N dan M adalah panjang string) diisi secara iteratif, biasanya dari kiri atas ke kanan bawah. Iterasi dimulai dari `i=1` hingga `N` dan `j=1` hingga `M`. Setelah tabel terisi penuh, `dp[N][M]` akan berisi panjang LCS dari kedua string. Kompleksitas waktu untuk pendekatan ini adalah O(N * M) karena setiap sel di tabel DP dihitung sekali, dan setiap perhitungan membutuhkan waktu O(1). Kompleksitas ruang juga O(N * M) untuk menyimpan tabel DP.

```
int solve_lcs(std::string s1, std::string s2) {
    int n = s1.length(), m = s2.length();
    std::vector<std::vector<int>> dp(n + 1, std::vector<int>(m + 1, 0));
    for (int i = 1; i <= n; ++i) {
        for (int j = 1; j <= m; ++j) {
            if (s1[i-1] == s2[j-1]) {
                dp[i][j] = 1 + dp[i-1][j-1];
            } else {
                dp[i][j] = std::max(dp[i-1][j], dp[i][j-1]);
            }
        }
    }
    return dp[n][m];
}
```

💡 **Fun Fact:** Masalah LCS memiliki aplikasi luas, termasuk dalam bioinformatika untuk menyelaraskan urutan DNA atau protein, dan dalam sistem kontrol versi seperti 'diff' tools untuk menunjukkan perbedaan antara dua versi file teks.
