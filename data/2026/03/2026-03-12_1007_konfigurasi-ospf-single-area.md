## 🌐 Networking Concept: Konfigurasi OSPF Single Area

**Category:** Routing
**TL;DR:** OSPF adalah routing protocol link-state yang efisien untuk menentukan jalur terbaik dalam jaringan IP dengan cepat dan skalabel.

Hari ini kita akan mendalami Open Shortest Path First (OSPF), sebuah Interior Gateway Protocol (IGP) yang sangat populer dan sering digunakan dalam jaringan skala menengah hingga besar. Berbeda dengan Distance Vector Protocol seperti RIP, OSPF menggunakan pendekatan Link-State. Artinya, setiap router OSPF akan membangun peta topologi lengkap jaringannya, bukan hanya mengandalkan informasi jarak dari tetangga. Hal ini membuat OSPF lebih cepat berkonvergensi ketika terjadi perubahan topologi dan lebih efisien dalam penggunaan bandwidth setelah konvergensi awal.

Dalam OSPF, router bertukar informasi tentang status link (Link-State Advertisements atau LSA) dengan router tetangga. Informasi LSA ini kemudian digunakan untuk membangun Link-State Database (LSDB) yang identik pada semua router di area yang sama. Setelah LSDB terbentuk, setiap router menjalankan algoritma Shortest Path First (SPF) dari Dijkstra untuk menghitung jalur terpendek ke semua tujuan dalam jaringan. Hasil perhitungan SPF ini kemudian digunakan untuk mengisi tabel routing, memastikan setiap paket dikirim melalui jalur paling optimal.

Konsep 'Area' adalah fitur kunci OSPF untuk skalabilitas. Untuk pembelajaran awal di Packet Tracer, kita akan fokus pada konfigurasi OSPF dalam 'Single Area', biasanya menggunakan Area 0 (Backbone Area). Seluruh router dan subnet yang ingin berpartisipasi dalam routing OSPF harus didefinisikan dalam area ini. Konfigurasi single area menyederhanakan proses pemahaman dasar OSPF sebelum kita masuk ke konfigurasi multi-area yang lebih kompleks, di mana area-area terpisah terhubung ke area backbone untuk mengurangi beban komputasi dan manajemen pada router.

Dengan OSPF, kita dapat membangun jaringan yang redundan dan efisien. Dalam praktiknya di Cisco Packet Tracer, Anda akan mengkonfigurasi OSPF pada setiap router dan mengiklankan (advertise) jaringan-jaringan yang terhubung langsung dengannya ke dalam area OSPF yang telah ditentukan. Selanjutnya, kita akan memverifikasi tabel routing dan status tetangga OSPF untuk memastikan protokol berjalan dengan benar, memungkinkan komunikasi antar subnet yang sebelumnya terisolasi.

```
     +-----+       +-----+       +-----+ 
     | R1  |-------| R2  |-------| R3  | 
     +-----+       +-----+       +-----+ 
        |             |             |    
        |   OSPF Area 0             |    
        +---------------------------+    
          (Berbagai Subnet LAN/WAN)    
```

**Cisco CLI:**
```
network 192.168.1.0 0.0.0.255 area 0
```

💡 **Fun Fact:** OSPF menggunakan 'cost' sebagai metrik untuk menentukan jalur terbaik. Secara default, cost dihitung berdasarkan bandwidth link, di mana link dengan bandwidth lebih tinggi memiliki cost yang lebih rendah, sehingga lebih diutamakan.
