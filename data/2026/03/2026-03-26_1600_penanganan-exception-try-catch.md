## ☕ Java Concept: Penanganan Exception: try-catch

**Category:** Exception
**TL;DR:** Konsep penanganan exception `try-catch` digunakan untuk mengantisipasi dan menangani kesalahan tak terduga (runtime errors) dalam program agar tidak crash dan tetap berjalan lancar.

Dalam pemrograman, tidak jarang kita menghadapi situasi di mana program bisa mengalami kesalahan saat dijalankan, misalnya membagi dengan nol, mengakses indeks array yang tidak ada, atau mencoba membaca file yang hilang. Kesalahan-kesalahan ini disebut 'exceptions' dan jika tidak ditangani, dapat menyebabkan program Anda berhenti secara tiba-tiba (crash) dengan pesan error yang kurang informatif.

Java menyediakan mekanisme yang elegan untuk menangani exception menggunakan blok `try-catch`. Blok `try` adalah tempat Anda meletakkan kode yang berpotensi menimbulkan exception. Jika sebuah exception terjadi di dalam blok `try`, eksekusi kode di dalam blok `try` akan dihentikan dan kontrol akan segera dialihkan ke blok `catch` yang sesuai.

Blok `catch` berfungsi untuk 'menangkap' dan 'menangani' exception yang terjadi. Setiap blok `catch` mendeklarasikan jenis exception yang ingin ditanganinya. Ketika sebuah exception ditangkap, kode di dalam blok `catch` akan dieksekusi, memungkinkan Anda untuk memberikan pesan kesalahan yang lebih ramah, melakukan pembersihan sumber daya, atau bahkan mencoba pemulihan dari kesalahan tersebut. Ini membuat program Anda lebih tangguh dan mudah digunakan.

Penerapan `try-catch` sangat praktis dalam pengembangan aplikasi sehari-hari. Bayangkan sebuah aplikasi yang meminta input angka dari pengguna, lalu mencoba mengkonversinya. Jika pengguna memasukkan teks alih-alih angka, tanpa `try-catch` program akan crash. Dengan `try-catch`, Anda bisa menangkap `NumberFormatException` dan meminta pengguna memasukkan ulang input yang benar, menciptakan pengalaman pengguna yang jauh lebih baik.

```
public class ExceptionDemo {
    public static void main(String[] args) {
        try {
            int numerator = 10;
            int denominator = 0; // Akan melempar ArithmeticException
            int result = numerator / denominator;
            System.out.println("Hasil: " + result);
        } catch (ArithmeticException e) {
            System.err.println("Error: Tidak bisa membagi dengan nol!");
            System.err.println("Pesan sistem: " + e.getMessage());
        }
        System.out.println("Program selesai berjalan.");
    }
}
```

💡 **Fun Fact:** Java adalah salah satu bahasa pemrograman awal yang secara eksplisit memasukkan mekanisme penanganan exception (menggunakan keyword `try`, `catch`, `finally`) sebagai bagian integral dari sintaksisnya, mendorong pengembang untuk membangun aplikasi yang lebih robust dan andal sejak dini.
