## 🤖 Android Concept: Memahami Activity pada Android

**Category:** Architecture
**TL;DR:** Activity adalah komponen inti aplikasi Android yang menyediakan satu layar interaktif untuk pengguna berinteraksi.

Activity adalah fondasi aplikasi Android Anda. Bayangkan setiap layar dalam aplikasi Anda — layar login, layar daftar produk, layar detail — semuanya diwakili oleh sebuah Activity. Ini adalah tempat Anda meletakkan elemen UI seperti tombol dan teks, serta logika bisnis yang terkait dengan layar tersebut, memungkinkan pengguna berinteraksi langsung dengan aplikasi.

Setiap Activity memiliki siklus hidup (lifecycle) yang terdefinisi dengan baik, mulai dari dibuat (`onCreate()`), terlihat oleh pengguna (`onStart()`, `onResume()`), hingga tidak terlihat (`onPause()`, `onStop()`) dan akhirnya dihancurkan (`onDestroy()`). Memahami lifecycle ini sangat krusial untuk mengelola resource dengan benar, menyimpan data, dan memastikan aplikasi berjalan efisien serta bebas error, terutama saat pengguna berpindah antar layar atau menerima panggilan telepon.

Untuk membuat sebuah Activity, Anda perlu membuat class Kotlin yang mewarisi dari `AppCompatActivity` (atau `ComponentActivity` secara modern). Di dalam metode `onCreate()`, Anda biasanya memanggil `setContentView()` untuk mengaitkan layout XML dengan Activity tersebut. Setiap Activity juga harus didaftarkan dalam file `AndroidManifest.xml` agar sistem Android dapat mengenalinya dan meluncurkannya. Aplikasi Android seringkali memiliki lebih dari satu Activity, dan untuk berpindah antar Activity, kita menggunakan objek `Intent`.

```
package com.example.myfirstapp

import android.os.Bundle
import androidx.appcompat.app.AppCompatActivity

class MainActivity : AppCompatActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main) // Mengaitkan layout XML
    }
}
```

💡 **Fun Fact:** Meskipun Activity adalah fondasi, tren modern dalam pengembangan Android bergerak menuju arsitektur 'Single-Activity', di mana satu Activity utama berfungsi sebagai host untuk banyak Fragment, menyederhanakan manajemen lifecycle dan navigasi.
