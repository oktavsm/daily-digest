## ⚡ Competitive Programming Concept: Dynamic Programming: Memoization

**Category:** DP
**TL;DR:** Strategi DP top-down yang menyimpan hasil sub-masalah yang sudah dihitung untuk menghindari komputasi berulang.

Memoization adalah salah satu pendekatan utama dalam Dynamic Programming (DP) yang bekerja secara "top-down". Ini menggabungkan rekursi standar dengan penyimpanan (caching) hasil dari sub-masalah yang sudah dihitung. Ketika sebuah fungsi rekursif dipanggil untuk menyelesaikan sub-masalah tertentu, pertama-tama ia memeriksa apakah hasil untuk sub-masalah tersebut sudah pernah dihitung dan disimpan. Jika ya, hasil yang tersimpan akan langsung dikembalikan, menghindari komputasi ulang yang tidak perlu.

Jika hasil sub-masalah belum ada, fungsi akan menghitungnya seperti biasa, lalu menyimpan hasilnya ke dalam struktur data (biasanya array atau hash map) sebelum mengembalikannya. Proses ini memastikan bahwa setiap sub-masalah hanya dihitung sekali. Kondisi dasar (base cases) dari rekursi juga harus didefinisikan dengan jelas. Dengan cara ini, masalah kompleks dipecah menjadi sub-masalah yang lebih kecil, dan karena adanya "overlapping subproblems", banyak sub-masalah yang sama akan muncul berulang kali, di situlah memoization menjadi sangat efisien.

Kompleksitas waktu memoization umumnya adalah jumlah state unik dikalikan dengan biaya komputasi untuk setiap state. Misalnya, untuk masalah seperti deret Fibonacci, ada N state unik, dan setiap state dihitung dalam O(1), sehingga total kompleksitas waktu menjadi O(N). Keuntungan utamanya adalah kemudahan dalam mengekspresikan logika rekursif secara langsung, yang seringkali lebih intuitif daripada pendekatan "bottom-up" (tabulation) untuk beberapa jenis masalah.

```
std::vector<long long> memo;
long long fib(int n) {
    if (n <= 1) return n;
    if (memo[n] != -1) return memo[n];
    return memo[n] = fib(n - 1) + fib(n - 2);
}
// Untuk menggunakan: memo.assign(N + 1, -1); long long result = fib(N);
```

💡 **Fun Fact:** Memoization berasal dari kata 'memo' yang berarti 'mengingat' atau 'mencatat', bukan dari kata 'memori'. Ini adalah salah satu teknik kunci untuk mengatasi masalah dengan sifat 'overlapping subproblems' dalam Dynamic Programming.
