## 🤖 Android Concept: Activity: Pintu Masuk Aplikasi

**Category:** UI
**TL;DR:** Activity adalah komponen dasar yang menyediakan layar untuk berinteraksi dengan pengguna di aplikasi Android dan memiliki siklus hidup yang dikelola sistem.

Selamat pagi, teman-teman! Hari ini kita akan menyelami jantung aplikasi Android, yaitu **Activity**. Bayangkan aplikasi Android sebagai sebuah rumah. Setiap ruangan di rumah itu adalah sebuah Activity. Activity adalah komponen fundamental yang menyediakan satu layar (atau bagian dari layar) untuk berinteraksi dengan pengguna. Misalnya, saat Anda membuka aplikasi WhatsApp dan melihat daftar chat, itu adalah sebuah Activity. Ketika Anda mengklik salah satu chat dan masuk ke layar percakapan, itu Activity lain.

Setiap Activity memiliki "siklus hidup" atau *lifecycle* yang dikelola oleh sistem Android. Ini berarti Activity akan melalui berbagai tahapan, mulai dari dibuat (`onCreate`), terlihat oleh pengguna (`onStart`, `onResume`), berhenti sebentar (`onPause`), hingga dimatikan sepenuhnya (`onDestroy`). Memahami lifecycle ini sangat penting karena Anda akan menulis kode yang merespons tahapan-tahapan tersebut, misalnya saat Activity baru dibuat atau akan dihancurkan, untuk memastikan aplikasi berjalan mulus.

Untuk membuat Activity, Anda biasanya membuat kelas Kotlin yang mewarisi `AppCompatActivity` (subclass dari `Activity`) dan meng-override metode `onCreate()`. Di sinilah Anda mengatur tata letak (layout) UI dengan memanggil `setContentView()`. Selain itu, setiap Activity harus dideklarasikan dalam file `AndroidManifest.xml` agar sistem Android tahu Activity mana yang tersedia dalam aplikasi Anda. Ini seperti mendaftarkan setiap ruangan di rumah Anda agar tamu tahu ada ruangan apa saja.

Aplikasi modern jarang hanya memiliki satu Activity. Kita sering perlu berpindah dari satu Activity ke Activity lain. Untuk melakukan ini, kita menggunakan objek `Intent`. `Intent` seperti pesan yang memberi tahu sistem Android "Saya ingin memulai Activity ini" atau "Saya ingin melakukan aksi ini". Ini adalah cara utama untuk navigasi antar layar di aplikasi Android Anda dan memungkinkan interaksi antar komponen aplikasi.

```
class MainActivity : AppCompatActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main) // Menghubungkan ke layout XML
        // Kode inisialisasi UI atau data lainnya di sini
    }
}
```

💡 **Fun Fact:** Tahukah kamu? Setiap kali kamu memutar layar ponsel (dari portrait ke landscape atau sebaliknya), Activity yang sedang berjalan akan dihancurkan dan dibuat ulang! Ini adalah bagian dari manajemen siklus hidupnya untuk menyesuaikan dengan konfigurasi baru secara tepat.
