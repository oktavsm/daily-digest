## ⚡ Competitive Programming Concept: 0/1 Knapsack Problem (DP)

**Category:** DP
**TL;DR:** Menentukan subset item dengan total nilai maksimum yang dapat dimasukkan ke dalam 'knapsack' dengan kapasitas terbatas, di mana setiap item hanya bisa diambil satu kali.

Masalah 0/1 Knapsack adalah problem klasik dalam optimasi kombinatorial. Bayangkan Anda memiliki sebuah tas (knapsack) dengan kapasitas berat `W` dan sejumlah `N` item. Setiap item `i` memiliki berat `w_i` dan nilai `v_i`. Tujuan kita adalah memilih subset item yang dapat dimasukkan ke dalam knapsack sehingga total beratnya tidak melebihi `W`, dan total nilai dari item-item yang dipilih adalah maksimum. Disebut "0/1" karena setiap item hanya bisa diambil (1) atau tidak diambil (0), tidak bisa sebagian.

Masalah ini tidak bisa diselesaikan dengan strategi greedy sederhana karena pilihan lokal terbaik belum tentu mengarah ke solusi global terbaik. Dynamic Programming (DP) adalah pendekatan yang tepat di sini karena masalah ini memiliki properti *optimal substructure* (solusi optimal dari masalah besar dibangun dari solusi optimal sub-masalah) dan *overlapping subproblems* (sub-masalah yang sama dihitung berkali-kali). Kita mendefinisikan `dp[i][j]` sebagai nilai maksimum yang dapat dicapai dengan mempertimbangkan `i` item pertama dan kapasitas knapsack `j`.

Untuk membangun tabel DP, kita iterasi dari item pertama hingga terakhir, dan untuk setiap item, kita iterasi melalui semua kemungkinan kapasitas. Untuk setiap `dp[i][j]`, ada dua kemungkinan: 1) Kita tidak mengambil item ke-`i`. Maka nilai maksimumnya sama dengan `dp[i-1][j]`. 2) Kita mengambil item ke-`i`. Ini hanya mungkin jika kapasitas `j` cukup untuk berat item `i` (`j >= w_i`). Jika ya, nilai maksimumnya adalah nilai item ke-`i` ditambah nilai maksimum yang bisa dicapai dari `i-1` item pertama dengan sisa kapasitas `j - w_i`, yaitu `v_i + dp[i-1][j - w_i]`. Oleh karena itu, `dp[i][j] = max(dp[i-1][j], v_i + dp[i-1][j - w_i])` jika `j >= w_i`, atau `dp[i-1][j]` jika `j < w_i`.

Basis kasusnya adalah `dp[0][j] = 0` (0 item, nilai 0) dan `dp[i][0] = 0` (kapasitas 0, nilai 0). Solusi akhir terletak di `dp[N][W]`. Kompleksitas waktu untuk pendekatan ini adalah O(N * W), di mana N adalah jumlah item dan W adalah kapasitas knapsack. Kompleksitas ruang juga O(N * W) untuk tabel DP 2D. Namun, kompleksitas ruang dapat dioptimalkan menjadi O(W) dengan hanya menggunakan satu baris DP dan mengiterasi kapasitas dari belakang untuk menghindari penggunaan nilai dari iterasi item saat ini.

```
// N items, W capacity
// weights: vector<int> w, values: vector<int> v
vector<vector<int>> dp(N + 1, vector<int>(W + 1, 0));

for (int i = 1; i <= N; ++i) {
    for (int j = 1; j <= W; ++j) {
        if (w[i-1] <= j) { // If item i can fit
            dp[i][j] = max(dp[i-1][j], v[i-1] + dp[i-1][j - w[i-1]]);
        } else { // Item i cannot fit
            dp[i][j] = dp[i-1][j];
        }
    }
}
// Result is dp[N][W]
```

💡 **Fun Fact:** Masalah Knapsack pertama kali diformulasikan pada akhir abad ke-19 dan memiliki banyak aplikasi praktis, mulai dari alokasi sumber daya hingga optimasi investasi dan pemilihan portofolio.
