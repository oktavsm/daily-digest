## 🤖 Android Concept: Activity: Blok Bangunan Aplikasi

**Category:** UI
**TL;DR:** Activity adalah komponen inti yang mewakili satu layar tunggal dalam aplikasi Android Anda, di mana pengguna dapat berinteraksi.

Activity adalah salah satu komponen fundamental dalam pengembangan aplikasi Android. Secara sederhana, setiap Activity merepresentasikan satu layar atau satu antarmuka pengguna (UI) dalam aplikasi Anda. Bayangkan aplikasi Anda sebagai sebuah buku, di mana setiap halaman adalah sebuah Activity. Ketika Anda beralih dari satu layar ke layar lain, misalnya dari daftar email ke detail email, Anda berpindah dari satu Activity ke Activity lainnya.

Setiap Activity memiliki siklus hidup (lifecycle) yang terdefinisi dengan baik, diatur oleh sistem operasi Android. Ini mencakup tahapan penting seperti `onCreate()`, `onStart()`, `onResume()`, `onPause()`, `onStop()`, dan `onDestroy()`. Bagi pemula, metode `onCreate()` sangatlah penting, karena di sinilah Activity pertama kali dibuat, dan Anda akan mengatur tampilan (layout) awal serta menginisialisasi komponen UI.

Untuk membuat sebuah Activity di Kotlin, Anda perlu membuat kelas baru yang mewarisi (inherit) dari `AppCompatActivity`. Di dalam metode `onCreate()`, Anda akan menggunakan `setContentView()` untuk 'mengikat' Activity Anda ke file layout XML yang sudah Anda desain. File layout XML inilah yang mendefinisikan tata letak dan elemen-elemen UI seperti tombol, teks, dan gambar yang akan ditampilkan di layar. Selain itu, setiap Activity harus didaftarkan dalam file `AndroidManifest.xml` agar sistem Android dapat mengenali dan meluncurkannya.

```
package com.example.myfirstapp

import android.os.Bundle
import androidx.appcompat.app.AppCompatActivity

class MainActivity : AppCompatActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main) // Menghubungkan ke layout XML
    }
}
```

💡 **Fun Fact:** Tahukah Anda bahwa Activity adalah konsep inti sejak Android versi pertama dirilis? Desain Activity yang fleksibel memungkinkan sistem Android untuk mengelola bagaimana aplikasi dibuka, ditutup, dan berinteraksi satu sama lain, bahkan antara aplikasi yang berbeda!
