## ☕ Java Concept: Penanganan Exception dengan Try-Catch

**Category:** Exception
**TL;DR:** Penanganan exception adalah cara Java mengelola error runtime agar program tidak crash dan tetap berjalan secara terkontrol.

Dalam pemrograman, seringkali terjadi kondisi yang tidak terduga saat program sedang berjalan, misalnya pengguna memasukkan data yang salah atau file yang ingin dibuka tidak ditemukan. Kondisi ini disebut "exception" atau pengecualian. Tanpa penanganan yang tepat, exception dapat menyebabkan program berhenti mendadak (crash), yang tentu saja buruk bagi pengalaman pengguna dan stabilitas aplikasi. Penanganan exception adalah mekanisme vital dalam Java untuk mengelola error runtime ini secara elegan.

Java menyediakan blok `try-catch` sebagai fondasi utama untuk menangani exception. Kode yang berpotensi menimbulkan exception diletakkan di dalam blok `try`. Jika exception terjadi di dalam blok `try`, eksekusi kode di blok `try` akan dihentikan, dan kendali program akan dialihkan ke blok `catch` yang relevan. Blok `catch` inilah tempat kita mendefinisikan apa yang harus dilakukan ketika exception tertentu tertangkap, misalnya menampilkan pesan kesalahan kepada pengguna, mencatat error, atau mencoba pemulihan.

Ada berbagai jenis exception di Java, seperti `NumberFormatException` ketika mencoba mengubah teks yang bukan angka menjadi angka, atau `ArrayIndexOutOfBoundsException` saat mencoba mengakses indeks array yang tidak valid. Penting untuk menangani exception secara spesifik jika memungkinkan, agar kita bisa memberikan penanganan yang paling sesuai untuk setiap jenis masalah. Namun, kita juga bisa menangkap `Exception` generik sebagai *fallback* untuk jenis exception yang tidak terduga, dan blok `finally` memastikan kode tertentu selalu dieksekusi terlepas dari apakah exception terjadi atau tidak.

Dengan menerapkan penanganan exception, program kita menjadi jauh lebih robust dan stabil. Ini mencegah program crash, memberikan umpan balik yang lebih baik kepada pengguna saat terjadi masalah, dan memudahkan pengembang untuk melacak dan memperbaiki bug. Penanganan exception adalah praktik wajib untuk setiap aplikasi Java yang serius agar dapat beroperasi dengan andal dalam berbagai skenario.

```
public class ExceptionDemo {
    public static void main(String[] args) {
        try {
            String str = "abc"; // String ini tidak bisa diubah jadi angka
            int num = Integer.parseInt(str); // Baris ini akan memicu NumberFormatException
            System.out.println("Angka: " + num);
        } catch (NumberFormatException e) {
            System.err.println("Error: Input bukan angka yang valid! Pesan: " + e.getMessage());
        } finally {
            System.out.println("Blok finally selalu dijalankan, baik ada exception atau tidak.");
        }
    }
}
```

💡 **Fun Fact:** Salah satu desain filosofi awal Java adalah 'checked exceptions', di mana kompilator memaksa programmer untuk secara eksplisit menangani atau mendeklarasikan exception tertentu. Ini bertujuan untuk membuat kode lebih robust dengan memastikan potensi masalah sudah dipikirkan sejak awal.
