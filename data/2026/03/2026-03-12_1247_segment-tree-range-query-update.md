## ⚡ Competitive Programming Concept: Segment Tree: Range Query Update

**Category:** Data Structure
**TL;DR:** Segment Tree adalah struktur data pohon yang efisien untuk melakukan operasi query (misalnya sum, min, max) dan update pada rentang (range) elemen dalam array.

Segment Tree adalah struktur data berbasis pohon yang dirancang untuk secara efisien melakukan operasi "range query" dan "point update" atau "range update" pada array. Bayangkan kita memiliki sebuah array dan sering kali perlu menemukan jumlah elemen dalam sub-array tertentu, atau mencari nilai minimum/maksimum dalam suatu rentang, lalu kemudian mengubah nilai satu atau beberapa elemen. Jika kita melakukan ini dengan array biasa, range query bisa memakan waktu O(N) dan point update O(1), atau sebaliknya jika menggunakan prefix sums. Segment Tree menawarkan solusi di mana kedua operasi tersebut dapat dilakukan dengan jauh lebih cepat.

Struktur Segment Tree adalah pohon biner lengkap atau hampir lengkap di mana setiap node merepresentasikan sebuah interval atau "segmen" dari array asli. Node akar (root) merepresentasikan seluruh array [0...N-1]. Setiap node internal [L...R] memiliki dua anak: anak kiri merepresentasikan segmen [L...(L+R)/2] dan anak kanan merepresentasikan segmen [(L+R)/2 + 1...R]. Node daun (leaf) merepresentasikan segmen yang hanya berisi satu elemen, yaitu [i...i]. Dengan struktur ini, setiap query atau update pada suatu rentang dapat dipecah menjadi beberapa segmen yang direpresentasikan oleh node-node di pohon, dan hasilnya digabungkan.

Operasi build (membangun pohon dari array awal) akan membutuhkan waktu O(N), karena kita mengunjungi setiap node daun sekali dan setiap node internal sekali. Untuk operasi point update (mengubah nilai satu elemen) atau range query (menanyakan nilai pada suatu rentang), waktu yang dibutuhkan adalah O(log N). Ini karena pada setiap level pohon, kita hanya perlu mengunjungi jalur dari akar hingga node daun yang relevan (untuk update) atau beberapa jalur yang membentuk rentang query. Jika ada kebutuhan untuk "range update" (mengubah nilai pada seluruh rentang) secara efisien, Segment Tree dapat diperluas dengan teknik "Lazy Propagation", yang juga mempertahankan kompleksitas waktu O(log N).

Segment Tree sangat populer karena fleksibilitasnya. Dapat digunakan untuk mencari sum, min, max, GCD, XOR, dan operasi asosiatif lainnya pada rentang. Contoh penggunaannya termasuk mencari rentang terpanjang dengan properti tertentu, menghitung inversi dalam array, atau bahkan sebagai dasar untuk struktur data yang lebih kompleks seperti 2D Segment Tree.

```
// Basic Segment Tree untuk mencari sum
const int N_MAX = 100005;
int tree[4 * N_MAX]; // Array untuk menyimpan pohon
int arr[N_MAX];      // Array asli

void build(int node, int start, int end) { // Kompleksitas: O(N)
    if(start == end) tree[node] = arr[start];
    else { int mid = (start+end)/2;
        build(2*node, start, mid); build(2*node+1, mid+1, end);
        tree[node] = tree[2*node] + tree[2*node+1]; }
}

int query(int node, int start, int end, int l, int r) { // Kompleksitas: O(log N)
    if(r < start || end < l) return 0; // Tidak ada overlap, return identitas (0 untuk sum)
    if(l <= start && end <= r) return tree[node]; // Overlap penuh
    int mid = (start+end)/2;
    return query(2*node, start, mid, l, r) + query(2*node+1, mid+1, end, l, r);
}
```

💡 **Fun Fact:** Segment Tree bisa dimodifikasi untuk hampir semua operasi asosiatif pada rentang, menjadikannya salah satu struktur data paling serbaguna di competitive programming.
