## ☕ Java Concept: ArrayList - List Dinamis Java

**Category:** Collections
**TL;DR:** ArrayList adalah implementasi List yang memungkinkan Anda menyimpan dan mengelola koleksi objek dengan ukuran yang bisa berubah secara dinamis.

ArrayList adalah salah satu kelas yang paling sering digunakan dalam kerangka kerja Collections di Java, yang berada di bawah paket `java.util`. Berbeda dengan array biasa yang ukurannya tetap setelah dideklarasikan, ArrayList menawarkan fleksibilitas untuk menyimpan koleksi objek yang ukurannya dapat berubah secara dinamis. Ini berarti Anda dapat menambah atau menghapus elemen dari ArrayList kapan saja, dan ArrayList akan secara otomatis menyesuaikan kapasitasnya. Konsep ini sangat berguna ketika Anda tidak tahu persis berapa banyak data yang perlu Anda simpan saat program berjalan.

Untuk menggunakan ArrayList, Anda perlu mengimpornya terlebih dahulu (`import java.util.ArrayList;`). Kemudian, Anda dapat membuat objek ArrayList dan menentukan jenis objek yang akan disimpannya menggunakan generics, misalnya `ArrayList<String>` untuk menyimpan string atau `ArrayList<Integer>` untuk menyimpan angka bulat. Anda dapat menambahkan elemen menggunakan metode `add()`, mengakses elemen berdasarkan indeks menggunakan `get()`, dan menghapus elemen menggunakan `remove()`. ArrayList menjaga urutan elemen sesuai dengan urutan saat mereka ditambahkan, dan memungkinkan elemen duplikat.

Keunggulan utama ArrayList terletak pada kemampuannya untuk mengelola koleksi data tanpa perlu khawatir tentang alokasi memori secara manual. Metode-metode seperti `size()` untuk mendapatkan jumlah elemen, `isEmpty()` untuk memeriksa apakah ArrayList kosong, dan `clear()` untuk menghapus semua elemen, membuat manipulasi koleksi data menjadi sangat intuitif. ArrayList sangat ideal digunakan ketika Anda sering perlu mengakses elemen secara acak (berdasarkan indeks) dan urutan elemen adalah penting.

Secara internal, ArrayList sebenarnya menggunakan array biasa untuk menyimpan elemen-elemennya. Ketika ArrayList mendeteksi bahwa array internalnya sudah penuh saat Anda mencoba menambahkan elemen baru, ia akan secara otomatis membuat array baru yang lebih besar (biasanya 1.5 kali ukuran semula) dan menyalin semua elemen dari array lama ke array baru tersebut. Proses ini memungkinkan pertumbuhan dinamis, meskipun operasi `add` sesekali bisa lebih lambat karena proses penyalinan ini.

```
import java.util.ArrayList;

public class DemoArrayList {
    public static void main(String[] args) {
        ArrayList<String> daftarNama = new ArrayList<>(); // Deklarasi
        daftarNama.add("Alice"); // Tambah elemen
        daftarNama.add("Bob");
        daftarNama.add("Charlie");

        System.out.println("Nama pertama: " + daftarNama.get(0)); // Akses elemen
        System.out.println("Ukuran list: " + daftarNama.size()); // Dapatkan ukuran

        daftarNama.remove(1); // Hapus elemen "Bob" (indeks 1)
        System.out.println("Setelah hapus, nama kedua: " + daftarNama.get(1)); // Menampilkan "Charlie"
    }
}
```

💡 **Fun Fact:** Sebelum Java 5, ArrayList tidak memiliki dukungan Generics. Anda harus meng-casting setiap objek yang diambil dari ArrayList secara manual, yang sering menimbulkan kesalahan `ClassCastException` saat runtime jika ada tipe data yang salah dimasukkan. Generics ditambahkan untuk membuat Collections seperti ArrayList menjadi tipe-aman.
