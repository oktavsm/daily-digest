## ⚡ Competitive Programming Concept: Segment Tree

**Category:** Data Structure
**TL;DR:** Struktur data berbasis pohon yang efisien untuk melakukan operasi query pada rentang (range query) dan pembaruan pada titik (point update) atau rentang (range update) dalam sebuah array.

Segment Tree adalah struktur data pohon biner yang digunakan untuk menyimpan informasi tentang interval atau 'segmen'. Tujuan utamanya adalah untuk memungkinkan query pada rentang tertentu dari sebuah array (misalnya, mencari jumlah, minimum, maksimum dalam sub-array dari indeks L hingga R) dan melakukan pembaruan elemen tunggal atau rentang elemen dengan cepat. Setiap node dalam Segment Tree merepresentasikan sebuah segmen atau interval dari array asli, dengan node root merepresentasikan seluruh array, dan node daun merepresentasikan elemen individual dari array.

Proses pembangunan (build) Segment Tree melibatkan pembagian array secara rekursif menjadi dua bagian hingga mencapai elemen tunggal. Setiap node internal menyimpan hasil operasi yang relevan (misalnya, jumlah) dari segmen yang diwakilinya, yang biasanya merupakan gabungan dari nilai-nilai yang disimpan oleh kedua anak-anaknya. Kompleksitas waktu untuk membangun Segment Tree adalah O(N), di mana N adalah ukuran array, karena setiap elemen dari array asli diproses sekali.

Ketika melakukan query untuk rentang [L, R], algoritma secara rekursif menelusuri pohon, menggabungkan informasi dari node-node yang segmennya sepenuhnya berada dalam rentang query. Jika suatu node sepenuhnya berada di luar rentang query, ia diabaikan. Jika hanya sebagian, ia terus menelusuri anak-anaknya. Proses ini menjamin bahwa query dapat dilakukan dalam waktu O(log N) karena pada setiap level pohon, hanya sejumlah kecil node yang perlu dikunjungi. Demikian pula, operasi pembaruan (update) pada titik tunggal juga memerlukan waktu O(log N) karena hanya satu jalur dari root ke daun yang perlu diperbarui.

Untuk operasi yang lebih kompleks seperti pembaruan pada rentang (range update), Segment Tree dapat diperluas dengan teknik 'lazy propagation'. Teknik ini menunda pembaruan pada node-node yang tidak sepenuhnya berada dalam rentang query, dan 'menyebarkan' pembaruan tersebut ke anak-anaknya hanya saat node tersebut perlu diakses. Dengan lazy propagation, operasi range update juga dapat dilakukan dalam waktu O(log N), menjadikan Segment Tree salah satu struktur data yang sangat serbaguna dan sering muncul dalam kompetisi pemrograman.

```
long long tree[4 * MAXN]; // MAXN is max size of array

// Example: Query for range sum
long long query(int node_idx, int current_start, int current_end, int query_l, int query_r) {
    if (query_r < current_start || current_end < query_l) return 0; // No overlap
    if (query_l <= current_start && current_end <= query_r) return tree[node_idx]; // Full overlap
    int mid = (current_start + current_end) / 2;
    long long p1 = query(2 * node_idx, current_start, mid, query_l, query_r);
    long long p2 = query(2 * node_idx + 1, mid + 1, current_end, query_l, query_r);
    return p1 + p2;
}
```

💡 **Fun Fact:** Meskipun sering digunakan dalam kompetisi pemrograman, Segment Tree memiliki aplikasi praktis di luar itu, misalnya dalam sistem informasi geografis (GIS) untuk query spasial atau dalam basis data untuk mengelola data interval. Konsep dasarnya, yaitu divide-and-conquer pada interval, sangat umum dalam ilmu komputer.
