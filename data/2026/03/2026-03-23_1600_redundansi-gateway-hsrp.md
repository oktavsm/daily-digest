## 🌐 Networking Concept: Redundansi Gateway HSRP

**Category:** Routing
**TL;DR:** HSRP memastikan ketersediaan gateway default yang tinggi dengan memungkinkan dua router atau lebih berbagi satu alamat IP virtual.

Di jaringan mana pun, gateway default adalah titik kritis. Jika router yang berfungsi sebagai gateway utama mati, semua host di LAN akan kehilangan konektivitas ke jaringan luar. Hot Standby Router Protocol (HSRP) dirancang oleh Cisco untuk mengatasi masalah single point of failure ini dengan menyediakan mekanisme redundansi otomatis. Ini memungkinkan dua atau lebih router untuk berkolaborasi dan tampil sebagai satu gateway logis bagi perangkat di segmen LAN yang sama, menjaga konektivitas jaringan tetap berjalan bahkan saat ada kegagalan perangkat.

HSRP bekerja dengan menetapkan satu alamat IP virtual dan alamat MAC virtual yang dibagikan oleh sekelompok router. Dari kelompok ini, satu router akan menjadi router "Active" yang bertanggung jawab meneruskan semua lalu lintas untuk alamat IP virtual tersebut. Router lainnya akan menjadi router "Standby" yang memantau router Active dan siap mengambil alih fungsi gateway jika router Active gagal. Router berkomunikasi satu sama lain menggunakan pesan "hello" secara berkala untuk memverifikasi ketersediaan masing-masing dan status rekan-rekannya.

Prioritas digunakan untuk menentukan router mana yang akan menjadi Active dan Standby; router dengan prioritas tertinggi (secara default 100) menjadi Active. Jika kedua router memiliki prioritas yang sama, router dengan alamat IP tertinggi akan dipilih. Selain itu, fitur "preemption" dapat diaktifkan agar router dengan prioritas lebih tinggi yang baru pulih dapat mengambil alih peran Active. Implementasi HSRP di Cisco Packet Tracer memungkinkan mahasiswa untuk mensimulasikan kegagalan router dan mengamati bagaimana lalu lintas secara transparan beralih ke router Standby, memastikan kelangsungan layanan tanpa interupsi bagi end-device.

```
standby 1 ip 192.168.1.254
```

💡 **Fun Fact:** HSRP adalah protokol redundansi gateway yang bersifat proprietary Cisco, tetapi ada alternatif standar terbuka seperti VRRP (Virtual Router Redundancy Protocol) yang memiliki tujuan serupa dan bisa berinteroperasi dengan perangkat multi-vendor.
