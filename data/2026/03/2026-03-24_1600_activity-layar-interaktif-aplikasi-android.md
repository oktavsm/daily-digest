## 🤖 Android Concept: Activity: Layar Interaktif Aplikasi Android

**Category:** Architecture
**TL;DR:** Activity adalah komponen dasar yang menyediakan layar tunggal dengan antarmuka pengguna (UI) bagi user untuk berinteraksi dalam aplikasi Android.

Activity adalah salah satu blok bangunan fundamental dalam setiap aplikasi Android. Bayangkan aplikasi Anda sebagai sebuah rumah; Activity adalah masing-masing ruangan di dalamnya, seperti ruang tamu, kamar tidur, atau dapur. Setiap Activity bertanggung jawab untuk menyediakan satu layar tunggal dengan antarmuka pengguna yang memungkinkan user berinteraksi, misalnya layar login, layar daftar produk, atau layar pengaturan. Ketika Anda berpindah dari satu layar ke layar lain dalam sebuah aplikasi, sebenarnya Anda sedang berpindah dari satu Activity ke Activity lainnya.

Setiap Activity memiliki siklus hidup (lifecycle) yang terdefinisi dengan baik, yang mencakup berbagai tahapan mulai dari saat Activity pertama kali dibuat hingga dihancurkan. Tahap yang paling penting dan sering digunakan oleh developer adalah `onCreate()`. Metode `onCreate()` dipanggil sistem Android saat Activity pertama kali dibuat. Di sinilah kita biasanya melakukan inisialisasi awal, seperti mengatur layout (tampilan) yang akan digunakan Activity dan menginisialisasi komponen-komponen UI seperti tombol atau teks.

Untuk menampilkan antarmuka pengguna, setiap Activity akan dihubungkan dengan sebuah file layout XML. Anda akan menggunakan metode `setContentView()` di dalam `onCreate()` untuk "mengaitkan" Activity Anda dengan layout XML tersebut. Setelah layout dimuat, Anda bisa mengakses elemen-elemen UI di dalamnya (seperti `Button`, `TextView`, `ImageView`) dari kode Kotlin Anda dan menambahkan logika bisnis atau penanganan event (misalnya, apa yang terjadi saat tombol diklik).

Memahami Activity adalah langkah pertama dan paling krusial dalam mengembangkan aplikasi Android. Meskipun ada komponen lain seperti Fragment yang kini sering digunakan untuk bagian-bagian layar, konsep dasar Activity tetap menjadi fondasi bagaimana aplikasi Android beroperasi dan bagaimana user bernavigasi dari satu "tempat" ke "tempat" lain di dalam aplikasi Anda.

```
package com.example.myfirstapp

import android.os.Bundle
import androidx.appcompat.app.AppCompatActivity

class MainActivity : AppCompatActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main) // Mengaitkan dengan layout XML
    }
}
```

💡 **Fun Fact:** Pada versi Android awal, sebelum ada desain material dan arsitektur yang lebih kompleks, Activity adalah satu-satunya cara utama untuk membangun setiap "layar" di aplikasi. Bahkan hingga kini, setiap Fragment tetap harus "menumpang" pada sebuah Activity untuk bisa ditampilkan.
