## ⚡ Competitive Programming Concept: Knapsack Problem: Dynamic Programming

**Category:** DP
**TL;DR:** Masalah Knapsack adalah masalah optimisasi di mana kita harus memilih item-item dengan berat dan nilai tertentu untuk dimasukkan ke dalam 'knapsack' (ransel) dengan kapasitas terbatas agar total nilai item maksimal.

Masalah Knapsack (khususnya 0/1 Knapsack) adalah masalah klasik dalam pemrograman dinamis. Diberikan N item, di mana setiap item i memiliki berat (weight) w_i dan nilai (value) v_i. Kita juga memiliki sebuah 'knapsack' dengan kapasitas maksimal W. Tujuan kita adalah memilih subset item-item yang total beratnya tidak melebihi W, tetapi memiliki total nilai yang semaksimal mungkin.

Pendekatan Dinamik (DP) yang paling umum melibatkan pembuatan tabel dp[i][j] yang merepresentasikan nilai maksimum yang bisa dicapai dengan menggunakan i item pertama dan kapasitas knapsack j. Untuk setiap item i dan kapasitas j, kita memiliki dua pilihan: tidak memasukkan item i ke dalam knapsack, atau memasukkannya (jika beratnya memungkinkan). Rekurensi dasarnya adalah: jika w_i > j, maka dp[i][j] = dp[i-1][j] (item terlalu berat). Jika w_i <= j, maka dp[i][j] = max(dp[i-1][j], dp[i-1][j - w_i] + v_i). Pilihan pertama berarti kita tidak mengambil item i, sedangkan pilihan kedua berarti kita mengambil item i dan menambahkan nilainya, sambil mengurangi kapasitas knapsack yang tersedia.

Kompleksitas waktu dari pendekatan DP dua dimensi ini adalah O(N*W), di mana N adalah jumlah item dan W adalah kapasitas knapsack. Kita juga dapat mengoptimalkan ruang menjadi O(W) dengan menyadari bahwa dp[i][j] hanya bergantung pada baris sebelumnya (i-1). Ini dilakukan dengan mengiterasi kapasitas j dari W ke w_i, untuk memastikan bahwa setiap dp[j] yang diperbarui menggunakan nilai dari 'baris sebelumnya' (yang belum diperbarui di iterasi saat ini).

```
int solveKnapsack(int W, const std::vector<int>& w, const std::vector<int>& v) {
    int N = w.size();
    std::vector<int> dp(W + 1, 0);

    for (int i = 0; i < N; ++i) {
        for (int j = W; j >= w[i]; --j) {
            dp[j] = std::max(dp[j], dp[j - w[i]] + v[i]);
        }
    }
    return dp[W];
}
```

💡 **Fun Fact:** Masalah Knapsack adalah salah satu dari 21 masalah NP-lengkap pertama yang diidentifikasi oleh Richard Karp pada tahun 1972, menjadikannya tonggak penting dalam teori kompleksitas komputasi. Meskipun demikian, ada algoritma pseudo-polinomial seperti DP ini, yang efisien jika W tidak terlalu besar.
