## 🤖 Android Concept: Siklus Hidup Activity

**Category:** Architecture
**TL;DR:** Memahami bagaimana sebuah Activity dibuat, diatur, dan dihancurkan oleh sistem Android sangat penting untuk mengelola data dan UI aplikasi dengan benar.

Setiap kali pengguna berinteraksi dengan sebuah layar di aplikasi Android Anda, biasanya itu adalah sebuah `Activity`. `Activity` adalah komponen aplikasi yang menyediakan satu layar dengan antarmuka pengguna tempat pengguna dapat berinteraksi. Namun, sebuah `Activity` tidak hanya "berjalan" begitu saja; ia melewati serangkaian tahapan yang disebut siklus hidup (lifecycle), yang dikelola oleh sistem Android. Memahami siklus hidup ini sangat krusial untuk membuat aplikasi yang stabil, efisien, dan responsif.

Ketika sebuah `Activity` pertama kali dibuat, metode `onCreate()` dipanggil. Ini adalah tempat Anda melakukan inisialisasi satu kali, seperti menyetel tata letak (`setContentView`) dan mengikat data ke UI. Setelah `onCreate()`, `onStart()` dipanggil saat Activity menjadi terlihat oleh pengguna, diikuti oleh `onResume()` saat Activity berada di latar depan dan dapat berinteraksi dengan pengguna. `onResume()` adalah tahap di mana Activity aktif sepenuhnya.

Jika pengguna beralih ke aplikasi lain atau ada dialog muncul di atas Activity Anda, `onPause()` akan dipanggil. Di sini Anda harus melepaskan sumber daya yang intensif atau menyimpan data yang belum disimpan karena Activity mungkin akan dihentikan. Jika Activity tidak lagi terlihat oleh pengguna (misalnya, berpindah ke Activity lain atau menekan tombol home), `onStop()` akan dipanggil. Setelah `onStop()`, sistem mungkin menghancurkan Activity untuk menghemat memori.

Metode `onDestroy()` adalah callback terakhir yang dipanggil sebelum Activity dihancurkan sepenuhnya dari memori. Ini adalah tempat yang tepat untuk melepaskan semua sumber daya yang masih aktif yang belum dilepaskan di `onPause()` atau `onStop()`, seperti koneksi database atau thread yang berjalan. Penting untuk diingat bahwa siklus hidup `Activity` dapat berubah sewaktu-waktu oleh sistem, sehingga pengelolaan state dan sumber daya di setiap callback menjadi sangat penting.

```
import android.os.Bundle
import androidx.appcompat.app.AppCompatActivity
import android.util.Log

class MainActivity : AppCompatActivity() {

    private val TAG = "MainActivityLifecycle"

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)
        Log.d(TAG, "onCreate() dipanggil")
    }

    override fun onResume() {
        super.onResume()
        Log.d(TAG, "onResume() dipanggil")
    }

    override fun onPause() {
        super.onPause()
        Log.d(TAG, "onPause() dipanggil")
    }

    override fun onDestroy() {
        super.onDestroy()
        Log.d(TAG, "onDestroy() dipanggil")
    }
}
```

💡 **Fun Fact:** Android dapat mematikan Activity Anda kapan saja untuk menghemat memori, bahkan tanpa memanggil `onDestroy()` jika Activity sudah di-background terlalu lama. Ini adalah alasan mengapa sangat penting untuk menyimpan data penting pengguna di `onSaveInstanceState()` dan mengelolanya saat Activity dibuat ulang, bukan hanya di `onDestroy()`.
