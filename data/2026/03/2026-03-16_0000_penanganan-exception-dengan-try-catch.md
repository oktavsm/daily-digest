## ☕ Java Concept: Penanganan Exception dengan try-catch

**Category:** Exception
**TL;DR:** Penanganan exception menggunakan blok `try-catch` memungkinkan program Java untuk mengatasi error tak terduga tanpa berhenti secara paksa.

Dalam pemrograman, sebuah "exception" (pengecualian) adalah peristiwa yang terjadi selama eksekusi program dan mengganggu alur normal instruksi. Ini bisa disebabkan oleh berbagai hal, seperti mencoba mengakses file yang tidak ada, membagi angka dengan nol, atau mencoba mengonversi teks menjadi angka yang tidak valid. Jika sebuah exception tidak ditangani, program biasanya akan "crash" atau berhenti secara tiba-tiba, yang tentu saja merupakan pengalaman buruk bagi pengguna.

Java menyediakan mekanisme yang kuat untuk menangani exception, yaitu dengan menggunakan blok `try-catch`. Blok `try` digunakan untuk mengelilingi kode yang berpotensi memicu exception. Jika sebuah exception terjadi di dalam blok `try`, eksekusi kode di `try` akan dihentikan, dan kontrol akan dialihkan ke blok `catch` yang sesuai. Blok `catch` berfungsi untuk "menangkap" dan mengelola exception tersebut, sehingga Anda bisa memberikan pesan error yang informatif kepada pengguna atau melakukan tindakan pemulihan.

Dengan menggunakan `try-catch`, Anda dapat membuat aplikasi yang lebih robust dan "ramah pengguna". Alih-alih program berhenti mendadak, Anda bisa menampilkan pesan error yang jelas, meminta input ulang, atau mencatat detail kesalahan untuk debugging. Ada juga blok opsional `finally` yang selalu dieksekusi, terlepas dari apakah exception terjadi atau tidak. Blok `finally` sangat berguna untuk membersihkan sumber daya (misalnya, menutup file atau koneksi database) yang dibuka di blok `try`, memastikan tidak ada kebocoran sumber daya.

```
try {
    int angka = Integer.parseInt("bukan_angka"); // Akan memicu NumberFormatException
    System.out.println("Angka yang diproses: " + angka);
} catch (NumberFormatException e) {
    System.out.println("Terjadi kesalahan konversi: Input bukan angka yang valid!");
    System.out.println("Detail error: " + e.getMessage());
} finally {
    System.out.println("Blok finally selalu dieksekusi, membersihkan sumber daya jika ada.");
}
```

💡 **Fun Fact:** Konsep penanganan exception seperti `try-catch` sangat penting dalam membangun perangkat lunak yang andal. Bahkan, banyak API Java mengharuskan Anda untuk menangani exception tertentu (checked exceptions) secara eksplisit, menunjukkan betapa pentingnya Java menempatkan ketahanan terhadap kesalahan.
