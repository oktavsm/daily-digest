## 🤖 Android Concept: Activity: Blok Bangunan Aplikasi Android

**Category:** Architecture
**TL;DR:** Activity adalah komponen utama yang menyediakan layar tunggal untuk berinteraksi dengan pengguna, mengelola siklus hidup tampilan dan interaksi.

Activity adalah salah satu komponen fundamental dalam pengembangan aplikasi Android yang paling sering Anda temui. Bayangkan Activity sebagai satu "layar" atau satu "halaman" dalam aplikasi Anda. Setiap kali Anda membuka aplikasi, melihat daftar item, atau mengisi formulir, kemungkinan besar Anda sedang berinteraksi dengan sebuah Activity. Activity bertanggung jawab untuk menyediakan jendela visual tempat antarmuka pengguna (UI) aplikasi Anda digambar dan tempat pengguna dapat berinteraksi.

Setiap Activity memiliki siklus hidup (lifecycle) yang terdefinisi dengan baik, mulai dari saat dibuat hingga dihancurkan. Beberapa tahapan penting dalam siklus hidup ini antara lain `onCreate()`, `onStart()`, `onResume()`, `onPause()`, `onStop()`, dan `onDestroy()`. Memahami siklus hidup ini sangat krusial karena Anda akan menempatkan logika aplikasi Anda pada tahapan yang tepat, misalnya, menginisialisasi tampilan di `onCreate()`, atau menyimpan data sementara di `onPause()` saat Activity tidak lagi aktif namun masih terlihat.

Aplikasi Android jarang hanya memiliki satu Activity. Seringkali, Anda perlu berpindah dari satu Activity ke Activity lainnya, misalnya dari layar login ke layar utama. Untuk melakukan ini, Anda menggunakan `Intent`. `Intent` adalah objek yang berfungsi sebagai pesan untuk meminta sistem Android melakukan sesuatu, seperti meluncurkan Activity baru. Dengan Intent, Anda tidak hanya bisa berpindah Activity, tapi juga bisa membawa data dari satu Activity ke Activity lainnya.

Secara keseluruhan, Activity adalah tulang punggung interaksi pengguna dalam aplikasi Android Anda. Ia mengatur bagaimana UI ditampilkan, bagaimana data awal dimuat, dan bagaimana aplikasi merespons peristiwa pengguna. Dengan Activity, Anda dapat membangun alur navigasi yang kompleks dan fungsionalitas yang kaya, menjadikannya titik awal yang penting dalam perjalanan Anda sebagai developer Android.

```
package com.example.myapp

import android.os.Bundle
import androidx.appcompat.app.AppCompatActivity

class MainActivity : AppCompatActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main) // Menghubungkan layout XML
        // Inisialisasi tampilan atau logika awal aplikasi di sini
    }
}
```

💡 **Fun Fact:** Tahukah Anda? Activity pada Android dirancang untuk dihentikan dan dimulai kembali kapan saja oleh sistem, misalnya saat pengguna beralih antar aplikasi atau saat perangkat berotasi. Inilah mengapa pengelolaan siklus hidup Activity sangat vital!
