## ⚡ Competitive Programming Concept: Longest Common Subsequence (LCS)

**Category:** DP
**TL;DR:** Algoritma pemrograman dinamis untuk menemukan urutan karakter terpanjang yang sama yang muncul di dua string, dengan urutan relatif karakter tetap terjaga.

Longest Common Subsequence (LCS) adalah masalah klasik dalam pemrograman dinamis yang bertujuan untuk menemukan urutan karakter terpanjang yang merupakan subsequence dari dua string yang diberikan. Subsequence berarti karakter-karakter tersebut tidak harus berurutan secara berturut-turut di string aslinya, tetapi urutan relatifnya harus tetap terjaga. Misalnya, "ACE" adalah subsequence dari "ABCDE", tetapi "AEC" bukan. Masalah LCS memiliki banyak aplikasi, seperti mendeteksi plagiarisme, bioinformatika untuk perbandingan DNA, dan fitur "diff" pada editor teks atau sistem kontrol versi.

Pendekatan untuk menyelesaikan LCS adalah menggunakan pemrograman dinamis. Kita akan membuat tabel `dp[i][j]` yang menyimpan panjang LCS dari prefix pertama string `s1` dengan panjang `i` dan prefix pertama string `s2` dengan panjang `j`. Ukuran tabel adalah `(N+1) x (M+1)`, di mana `N` adalah panjang `s1` dan `M` adalah panjang `s2`. Basis kasusnya adalah ketika salah satu string kosong (`i=0` atau `j=0`), maka panjang LCS-nya adalah 0.

Untuk mengisi tabel `dp[i][j]`, kita pertimbangkan karakter `s1[i-1]` dan `s2[j-1]`. Jika kedua karakter ini sama, berarti kita bisa memperpanjang LCS yang ditemukan dari prefix sebelumnya. Jadi, `dp[i][j] = 1 + dp[i-1][j-1]`. Jika karakter-karakter tersebut tidak sama, maka kita tidak bisa memasukkan keduanya ke dalam LCS. Kita harus memilih yang terbaik dari dua kemungkinan: menghilangkan `s1[i-1]` (maka kita melihat `dp[i-1][j]`) atau menghilangkan `s2[j-1]` (maka kita melihat `dp[i][j-1]`). Oleh karena itu, `dp[i][j] = max(dp[i-1][j], dp[i][j-1]).

Setelah seluruh tabel terisi, nilai `dp[N][M]` akan memberikan panjang Longest Common Subsequence dari kedua string lengkap. Kompleksitas waktu untuk algoritma ini adalah O(N*M) karena kita mengisi setiap sel di tabel `N x M` sekali, dan setiap pengisian membutuhkan waktu konstan. Kompleksitas ruangnya juga O(N*M) untuk menyimpan tabel DP.

```
int lcs(const std::string& s1, const std::string& s2) {
    int n = s1.length(), m = s2.length();
    std::vector<std::vector<int>> dp(n + 1, std::vector<int>(m + 1, 0));
    for (int i = 1; i <= n; ++i) {
        for (int j = 1; j <= m; ++j) {
            if (s1[i-1] == s2[j-1]) dp[i][j] = 1 + dp[i-1][j-1];
            else dp[i][j] = std::max(dp[i-1][j], dp[i][j-1]);
        }
    }
    return dp[n][m];
}
```

💡 **Fun Fact:** Algoritma LCS sering digunakan di fitur 'diff' pada sistem kontrol versi seperti Git untuk membandingkan dan menampilkan perubahan antara dua versi file, meskipun implementasi 'diff' modern biasanya menggunakan algoritma yang lebih canggih dari LCS murni.
