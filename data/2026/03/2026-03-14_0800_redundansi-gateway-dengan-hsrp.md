## 🌐 Networking Concept: Redundansi Gateway dengan HSRP

**Category:** Routing
**TL;DR:** Memastikan ketersediaan gateway default untuk host di LAN dengan mengkonfigurasi beberapa router sebagai satu gateway virtual menggunakan HSRP.

Dalam lingkungan jaringan modern, ketersediaan adalah kunci. Salah satu titik kegagalan tunggal yang paling sering terlupakan adalah router gateway default untuk segmen LAN. Jika router ini mati atau mengalami masalah, semua perangkat di segmen tersebut akan kehilangan konektivitas ke jaringan eksternal, meskipun ada router lain yang berfungsi di segmen yang sama. Inilah masalah krusial yang dipecahkan oleh First Hop Redundancy Protocols (FHRPs), seperti HSRP (Hot Standby Router Protocol) dari Cisco.

HSRP bekerja dengan mengelompokkan dua atau lebih router fisik menjadi satu router logis atau virtual. Kelompok router ini berbagi satu alamat IP virtual dan alamat MAC virtual. Salah satu router dalam kelompok akan berperan sebagai router "Aktif" yang memproses semua lalu lintas yang ditujukan ke alamat IP virtual. Router lain akan menjadi router "Standby" yang memantau router Aktif dan siap mengambil alih peran jika router Aktif gagal. Semua host di LAN akan dikonfigurasi untuk menggunakan alamat IP virtual ini sebagai gateway default mereka.

Ketika router Aktif gagal, router Standby akan mendeteksinya (biasanya melalui pesan hello yang terhenti) dan secara otomatis beralih menjadi router Aktif, mengambil alih alamat IP dan MAC virtual. Transisi ini terjadi dengan cepat dan transparan bagi host, yang berarti perangkat di LAN tidak perlu mengubah konfigurasi gateway mereka dan akan tetap memiliki konektivitas ke jaringan luar. HSRP juga memungkinkan konfigurasi prioritas, di mana router dengan prioritas tertinggi akan menjadi Aktif, dan fitur "preempt" agar router prioritas tinggi secara otomatis merebut kembali peran Aktif setelah pulih dari kegagalan.

Di Cisco Packet Tracer, Anda dapat dengan mudah mensimulasikan HSRP dengan menyiapkan dua router yang terhubung ke satu switch, dan beberapa PC sebagai host. Konfigurasikan interface router yang terhubung ke switch dengan alamat IP pribadi dan kemudian terapkan HSRP pada interface tersebut, mendefinisikan grup HSRP, IP virtual, dan prioritas. Anda kemudian dapat mematikan salah satu router atau interface-nya untuk mengamati bagaimana router standby mengambil alih peran aktif, memastikan konektivitas host tetap terjaga.

```
interface GigabitEthernet0/1
 ip address 192.168.1.1 255.255.255.0
 standby 1 ip 192.168.1.254
 standby 1 priority 150
 standby 1 preempt
```

💡 **Fun Fact:** Meskipun HSRP adalah protokol milik Cisco, ada standar terbuka yang setara bernama VRRP (Virtual Router Redundancy Protocol) yang berfungsi serupa. Cisco juga memiliki GLBP (Gateway Load Balancing Protocol) yang memungkinkan semua router di grup secara aktif memproses lalu lintas, bukan hanya satu.
