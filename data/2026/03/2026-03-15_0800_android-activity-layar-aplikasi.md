## 🤖 Android Concept: Android Activity: Layar Aplikasi

**Category:** UI
**TL;DR:** Activity adalah komponen fundamental Android yang merepresentasikan satu layar antarmuka pengguna (UI) dalam aplikasi.

Sebagai seorang developer Android, Activity adalah salah satu fondasi utama yang akan sering Anda temui. Bayangkan setiap 'layar' yang Anda lihat di aplikasi, seperti halaman login, daftar produk, atau detail sebuah item, itu umumnya diwakili oleh sebuah Activity. Activity berfungsi sebagai wadah untuk menempatkan elemen-elemen UI (tombol, teks, gambar) dan mengelola interaksi pengguna pada layar tersebut. Setiap kali pengguna berpindah dari satu layar ke layar lain dalam aplikasi, kemungkinan besar ada perpindahan atau perubahan Activity yang terjadi.

Setiap Activity memiliki siklus hidup (lifecycle) yang dikelola secara otomatis oleh sistem Android, seperti saat Activity dibuat, mulai terlihat oleh pengguna, berinteraksi, berhenti sementara, hingga dihancurkan. Metode paling krusial yang akan kita gunakan adalah `onCreate()`. Metode ini adalah titik awal ketika sebuah Activity pertama kali dibuat. Di sinilah kita melakukan inisialisasi awal, seperti menghubungkan Activity dengan file layout XML yang mendefinisikan tampilan UI-nya.

Untuk menampilkan antarmuka pengguna pada Activity, kita perlu menautkannya dengan sebuah file layout XML. Perintah `setContentView()` adalah jembatan yang menghubungkan kode Activity Kotlin kita dengan desain XML tersebut, sehingga tampilan yang telah kita buat bisa muncul di layar. Selanjutnya, untuk berpindah dari satu Activity ke Activity lain, Android menggunakan konsep `Intent`. `Intent` seperti sebuah pesan yang memberi tahu sistem Android apa yang ingin kita lakukan, misalnya meluncurkan Activity baru atau mengirim data antar Activity.

Memahami Activity dan siklus hidupnya sangat penting karena ini adalah inti dari bagaimana aplikasi Android berinteraksi dengan pengguna dan bagaimana navigasi antar layar diatur. Ini adalah langkah pertama untuk membangun aplikasi yang interaktif dan responsif.

```
package com.example.myfirstapp

import android.os.Bundle
import androidx.appcompat.app.AppCompatActivity

class MainActivity : AppCompatActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main) // Menghubungkan Activity dengan layout XML
    }
}
```

💡 **Fun Fact:** Android Activity pertama kali diperkenalkan dengan rilis Android 1.0 pada tahun 2008. Konsep Activity stack-nya memungkinkan pengguna untuk menavigasi aplikasi dengan mudah menggunakan tombol 'Back', mirip dengan tab browser.
