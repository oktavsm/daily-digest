## ⚡ Competitive Programming Concept: Knapsack Problem (0/1)

**Category:** DP
**TL;DR:** Masalah optimasi klasik di mana Anda memilih item dengan bobot dan nilai untuk memaksimalkan total nilai dalam batas kapasitas tertentu.

The 0/1 Knapsack Problem adalah masalah optimasi klasik dalam Dynamic Programming. Diberikan `N` item, di mana setiap item `i` memiliki berat `w_i` dan nilai `v_i`, serta sebuah kapasitas maksimum knapsack `W`. Tujuannya adalah memilih subset item yang dapat dimasukkan ke dalam knapsack sehingga total berat tidak melebihi `W`, sambil memaksimalkan total nilai dari item-item yang dipilih. Setiap item hanya bisa dipilih sekali (0 atau 1, diambil atau tidak diambil).

Untuk menyelesaikan masalah ini dengan DP, kita bisa mendefinisikan `dp[j]` sebagai nilai maksimum yang bisa diperoleh dengan kapasitas knapsack `j`. Saat memproses item ke-`i`, kita akan mempertimbangkan apakah akan mengambil item tersebut atau tidak. Jika kita mengambil item ke-`i` dengan berat `w_i` dan nilai `v_i`, maka nilai maksimum untuk kapasitas `j` akan menjadi `v_i` ditambah nilai maksimum yang bisa didapat dari kapasitas `j - w_i` (dengan item-item sebelumnya). Jika tidak mengambil, nilai maksimumnya sama dengan nilai maksimum untuk kapasitas `j` dari item-item sebelumnya.

Recurrence relation-nya adalah `dp[j] = max(dp[j], dp[j - w_i] + v_i)` untuk setiap `j` dari `W` hingga `w_i`. Perlu diingat bahwa iterasi `j` harus mundur (dari `W` ke `w_i`) untuk memastikan setiap item dipertimbangkan hanya sekali dan tidak diambil lebih dari sekali. Kompleksitas waktu untuk solusi ini adalah `O(N*W)`, di mana `N` adalah jumlah item dan `W` adalah kapasitas knapsack. Kompleksitas ruang dapat dioptimalkan menjadi `O(W)` dengan pendekatan array 1D.

```
vector<int> dp(W + 1, 0);
for (int i = 0; i < N; ++i) {
    int current_weight = w[i];
    int current_value = v[i];
    for (int j = W; j >= current_weight; --j) {
        dp[j] = max(dp[j], dp[j - current_weight] + current_value);
    }
}
// Nilai maksimum ada di dp[W]
```

💡 **Fun Fact:** Knapsack Problem adalah salah satu dari 21 masalah NP-Complete pertama yang diidentifikasi oleh Richard Karp pada tahun 1972, menjadikannya patokan penting dalam teori kompleksitas komputasi.
