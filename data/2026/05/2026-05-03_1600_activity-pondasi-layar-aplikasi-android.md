## 🤖 Android Concept: Activity: Pondasi Layar Aplikasi Android

**Category:** Architecture
**TL;DR:** Activity adalah komponen fundamental dalam aplikasi Android yang menyediakan satu layar antarmuka pengguna, berfungsi sebagai pintu masuk utama interaksi pengguna.

Activity adalah salah satu dari empat komponen inti aplikasi Android (bersama dengan Service, Broadcast Receiver, dan Content Provider). Secara sederhana, Activity adalah layar tunggal yang memungkinkan pengguna berinteraksi dengan aplikasi Anda. Misalnya, layar login, layar daftar produk, atau layar pengaturan, semuanya bisa direpresentasikan sebagai Activity yang berbeda. Setiap aplikasi Android biasanya memiliki setidaknya satu Activity utama yang menjadi titik awal saat aplikasi diluncurkan.

Setiap Activity memiliki siklus hidup (lifecycle) yang dikelola oleh sistem Android. Ini berarti Activity dapat berada dalam berbagai status, seperti sedang dibuat (onCreate), terlihat oleh pengguna (onStart), berinteraksi aktif (onResume), hingga dihancurkan (onDestroy). Memahami siklus hidup ini sangat krusial untuk mengelola sumber daya, menyimpan data saat aplikasi berhenti, dan memastikan pengalaman pengguna yang mulus. Fungsi onCreate() adalah tempat paling umum untuk melakukan inisialisasi awal, seperti mengatur tata letak (layout) dan mengikat tampilan (views).

Anda membuat Activity dengan membuat class Kotlin yang mewarisi AppCompatActivity (atau Activity). Setiap Activity harus dideklarasikan dalam file AndroidManifest.xml agar sistem Android dapat mengenalinya dan bagaimana ia harus diluncurkan. Untuk berpindah dari satu Activity ke Activity lain, Anda menggunakan objek Intent, yang bertindak sebagai "niat" untuk melakukan suatu operasi, misalnya meluncurkan komponen aplikasi lain atau memanggil Activity tertentu.

Activity bertanggung jawab penuh untuk mengelola interaksi pengguna pada layarnya. Ini termasuk merespons sentuhan, menampilkan input, dan memperbarui tampilan berdasarkan logika aplikasi. Dengan mengorganisir fungsionalitas aplikasi ke dalam Activity yang berbeda, pengembang dapat menciptakan alur navigasi yang jelas dan memisahkan tanggung jawab, membuat aplikasi lebih modular dan mudah dikelola.

```
package com.example.myapp

import android.os.Bundle
import androidx.appcompat.app.AppCompatActivity

class MainActivity : AppCompatActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main) // Menghubungkan layout XML
    }
}
```

💡 **Fun Fact:** Tahukah kamu? Setiap aplikasi Android modern biasanya memiliki satu atau lebih Activity, dan Activity utama sering disebut sebagai 'launcher activity' karena ia adalah titik masuk saat aplikasi diluncurkan dari home screen.
