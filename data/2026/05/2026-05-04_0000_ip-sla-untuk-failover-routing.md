## 🌐 Networking Concept: IP SLA untuk Failover Routing

**Category:** Routing
**TL;DR:** Menggunakan IP Service Level Agreement (SLA) untuk memantau konektivitas jalur utama dan secara otomatis mengalihkan lalu lintas ke jalur cadangan jika terjadi kegagalan.

Dalam skenario jaringan yang menggunakan static routing, kegagalan pada jalur utama dapat menyebabkan "blackholing" traffic karena router terus mencoba mengirim data ke jalur yang tidak aktif. Meskipun dynamic routing (seperti OSPF atau EIGRP) dapat mengatasi ini, ada situasi di mana static routing tetap diinginkan, misalnya untuk kesederhanaan, mengontrol titik keluar tertentu, atau sebagai backup untuk routing dinamis. Di sinilah IP Service Level Agreement (SLA) berperan penting.

IP SLA adalah fitur pada perangkat Cisco yang memungkinkan router secara aktif memantau kondisi jaringan atau ketersediaan host tertentu dengan mengirimkan trafik probe. Contoh probe yang umum adalah ICMP echo (ping) ke IP address tujuan atau mencoba koneksi TCP ke port tertentu. Hasil dari operasi IP SLA ini dapat digunakan untuk menilai apakah suatu jalur atau layanan tertentu masih berfungsi dengan baik atau tidak.

Kunci integrasi IP SLA dengan routing adalah kemampuannya untuk dikaitkan (track) dengan objek routing, khususnya static route. Ketika sebuah operasi IP SLA berhasil memverifikasi ketersediaan jalur utama, objek tracking akan berstatus "up", dan static route yang terkait akan aktif dalam tabel routing. Namun, jika operasi IP SLA gagal mendeteksi target (misalnya, ping gagal), objek tracking akan berstatus "down", dan static route yang terkait secara otomatis akan dihapus dari tabel routing.

Ketika static route utama dihapus, router dapat mengaktifkan "floating static route" yang sudah dikonfigurasi sebelumnya. Floating static route adalah static route cadangan dengan administrative distance (AD) yang lebih tinggi. Karena AD jalur utama (yang sekarang dihapus) lebih rendah, jalur cadangan ini sebelumnya tidak aktif. Begitu jalur utama tidak ada, jalur cadangan dengan AD lebih tinggi ini menjadi rute terbaik dan otomatis masuk ke tabel routing, memastikan kontinuitas layanan. Saat jalur utama pulih, IP SLA akan mendeteksinya, static route utama kembali diaktifkan, dan floating static route akan kembali tidak aktif.

```
ip route 0.0.0.0 0.0.0.0 10.0.0.1 track 1
```

💡 **Fun Fact:** IP SLA tidak hanya untuk routing! Hasil operasi IP SLA juga bisa digunakan untuk memicu perubahan pada konfigurasi Quality of Service (QoS) atau untuk mengirim notifikasi SNMP saat kondisi tertentu terpenuhi.
