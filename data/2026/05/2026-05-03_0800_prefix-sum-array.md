## ⚡ Competitive Programming Concept: Prefix Sum Array

**Category:** Data Structure
**TL;DR:** Prefix Sum adalah teknik prekomputasi untuk menghitung jumlah elemen pada subarray atau range secara efisien (O(1) per query) setelah pembangunan awal O(N).

Konsep dasar prefix sum adalah membuat sebuah array baru, sebut saja `P`, di mana setiap elemen `P[i]` menyimpan jumlah dari elemen-elemen awal array asli `A` hingga indeks `i-1`. Tujuan utama dari teknik ini adalah untuk menjawab pertanyaan 'berapa total nilai dari indeks `L` hingga `R` di array asli `A`?' dengan sangat cepat, jauh lebih efisien daripada menjumlahkan elemen satu per satu setiap kali ada pertanyaan.

Misalkan kita memiliki array asli `A` berukuran `N`. Kita dapat membangun array prefix sum `P` berukuran `N+1` (menggunakan `P[0]=0` untuk kemudahan indeksasi). Elemen `P[i]` akan menyimpan jumlah `A[0] + A[1] + ... + A[i-1]`. Proses pembangunan array `P` ini dapat dilakukan dengan iterasi tunggal: `P[i] = P[i-1] + A[i-1]` untuk `i` dari `1` hingga `N`. Pembangunan array ini memiliki kompleksitas waktu `O(N)`.

Setelah array `P` berhasil dibuat, untuk menghitung jumlah elemen dari `A[L]` hingga `A[R]` (inklusi, dengan `L` dan `R` adalah indeks 0-based), kita cukup melakukan operasi `P[R+1] - P[L]`. Mengapa demikian? `P[R+1]` adalah total jumlah `A[0]` hingga `A[R]`, sementara `P[L]` adalah total jumlah `A[0]` hingga `A[L-1]`. Selisih kedua nilai ini akan menghasilkan tepat jumlah elemen dari `A[L]` hingga `A[R]`. Dengan demikian, setiap query range sum hanya membutuhkan waktu `O(1)`, sebuah peningkatan signifikan dari `O(N)` per query.

Prefix sum sangat powerful karena mengubah banyak query yang tadinya membutuhkan `O(N)` waktu menjadi `O(1)` per query setelah prekomputasi awal yang hanya `O(N)`. Teknik ini sangat berguna dalam masalah yang melibatkan banyak query jumlah subarray, atau untuk menemukan subarray dengan properti tertentu. Konsep ini juga dapat diperluas untuk array dua dimensi (2D Prefix Sum) atau untuk operasi lain selain penjumlahan (misalnya XOR prefix sum) di mana operasi bersifat asosiatif dan memiliki elemen identitas.

```
#include <vector>

// Fungsi untuk membangun array prefix sum
std::vector<long long> build_prefix_sum(const std::vector<int>& arr) {
    std::vector<long long> ps(arr.size() + 1, 0);
    for (int i = 0; i < arr.size(); ++i) {
        ps[i+1] = ps[i] + arr[i];
    }
    return ps;
}

// Fungsi untuk query jumlah range [L, R] (0-indexed)
long long query_range_sum(const std::vector<long long>& ps, int L, int R) {
    return ps[R + 1] - ps[L];
}
```

💡 **Fun Fact:** Meskipun sederhana, konsep prefix sum adalah salah satu teknik optimisasi paling dasar yang dapat mengubah kompleksitas waktu dari O(N) per query menjadi O(1) per query setelah prekomputasi, menjadikannya fondasi bagi banyak algoritma yang lebih canggih.
