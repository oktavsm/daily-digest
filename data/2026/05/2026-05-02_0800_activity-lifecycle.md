## 🤖 Android Concept: Activity Lifecycle

**Category:** Architecture
**TL;DR:** Activity adalah komponen utama yang menyediakan antarmuka pengguna pada sebuah layar, dan Lifecycle-nya mendefinisikan tahapan hidupnya yang dikelola oleh sistem Android.

Activity adalah 'layar' tunggal dalam aplikasi Android Anda. Ketika Anda membuka aplikasi dan melihat antarmuka pengguna (UI) tertentu, kemungkinan besar Anda sedang berinteraksi dengan sebuah Activity. Activity bertanggung jawab untuk mengelola UI dan merespons interaksi pengguna untuk layar tersebut. Setiap aplikasi Android setidaknya memiliki satu Activity sebagai titik masuk utamanya.

Sistem Android tidak hanya menampilkan Activity Anda, tetapi juga mengelolanya melalui serangkaian tahapan yang disebut Lifecycle. Memahami Lifecycle sangat penting karena Activity bisa dibuat, ditampilkan, berhenti sebentar, bahkan dihancurkan, tergantung pada tindakan pengguna atau peristiwa sistem (misalnya, telepon masuk, orientasi layar berubah, atau pengguna berpindah aplikasi). Anda harus menempatkan logika inisialisasi, penghematan data, atau pelepasan sumber daya pada tahapan Lifecycle yang tepat untuk mencegah kebocoran memori atau crash.

Lifecycle Activity didefinisikan oleh serangkaian metode callback yang bisa Anda override di kelas Activity Anda. Beberapa yang paling penting antara lain `onCreate()` (dipanggil saat Activity pertama kali dibuat dan melakukan inisialisasi awal), `onStart()` (Activity mulai terlihat oleh pengguna), `onResume()` (Activity mulai berinteraksi dengan pengguna), `onPause()` (Activity kehilangan fokus tetapi masih terlihat), `onStop()` (Activity tidak lagi terlihat), dan `onDestroy()` (Activity akan dihancurkan. Sistem Android akan memanggil metode-metode ini secara otomatis saat Activity berpindah dari satu status ke status lain, memungkinkan Anda mengelola sumber daya dengan efisien.

```
import android.os.Bundle
import android.util.Log
import androidx.appcompat.app.AppCompatActivity

class MainActivity : AppCompatActivity() {
    private val TAG = "MainActivityLifecycle"

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)
        Log.d(TAG, "onCreate: Activity Created")
    }

    override fun onResume() {
        super.onResume()
        Log.d(TAG, "onResume: Activity Resumed (Interactive)")
    }

    override fun onPause() {
        super.onPause()
        Log.d(TAG, "onPause: Activity Paused (Lost Focus)")
    }
}
```

💡 **Fun Fact:** Sebelum munculnya Fragments, Activity seringkali harus menangani semua kompleksitas UI dan logika aplikasi untuk satu layar. Hal ini membuatnya menjadi 'God Object' yang sulit di-maintain. Kini, Activity lebih sering berfungsi sebagai container untuk Fragments, yang membantu memecah kompleksitas UI.
