## 🤖 Android Concept: Activity: Fondasi Layar Aplikasi Android

**Category:** UI
**TL;DR:** Activity adalah komponen inti Android yang merepresentasikan satu layar dalam aplikasi dan mengelola interaksi pengguna serta siklus hidupnya.

Di dunia Android, setiap "layar" yang Anda lihat dalam sebuah aplikasi, seperti daftar email, halaman pengaturan, atau tampilan kamera, biasanya diwakili oleh sebuah `Activity`. Activity adalah komponen dasar yang menyediakan jendela untuk antarmuka pengguna (UI), tempat Anda bisa menempatkan elemen-elemen seperti tombol, teks, gambar, dan komponen interaktif lainnya. Ini adalah titik masuk utama di mana pengguna berinteraksi langsung dengan aplikasi Anda.

Setiap Activity memiliki siklus hidup yang terdefinisi dengan baik, mulai dari dibuat, aktif, hingga dihancurkan. Metode-metode seperti `onCreate()`, `onStart()`, `onResume()`, `onPause()`, `onStop()`, dan `onDestroy()` memungkinkan Anda untuk menjalankan kode pada tahapan-tahapan tertentu dalam kehidupan Activity. Sebagai contoh, `onCreate()` adalah tempat paling umum untuk menginisialisasi UI aplikasi Anda dan menyiapkan data awal, yang hanya dipanggil sekali saat Activity pertama kali dibuat.

Untuk membuat Activity, Anda perlu mendefinisikannya sebagai class Kotlin yang mewarisi dari `AppCompatActivity`. Selain itu, setiap Activity harus dideklarasikan dalam file `AndroidManifest.xml` aplikasi Anda agar sistem Android dapat mengenali dan meluncurkannya. Di dalam metode `onCreate()`, Anda akan menggunakan `setContentView(R.layout.nama_layout)` untuk menghubungkan Activity dengan file layout XML yang mendefinisikan tampilan antarmuka pengguna untuk layar tersebut.

Aplikasi Android seringkali terdiri dari beberapa Activity yang saling terhubung. Perpindahan antar Activity dilakukan menggunakan `Intent`, sebuah objek yang berfungsi sebagai "niat" untuk melakukan suatu tindakan, seperti memulai Activity lain. Memahami Activity sangat krusial karena ia adalah tulang punggung dari navigasi dan struktur visual aplikasi, memastikan pengalaman pengguna yang mulus saat mereka berpindah antar layar atau kembali ke aplikasi setelah terinterupsi.

```
import android.os.Bundle
import androidx.appcompat.app.AppCompatActivity

class MainActivity : AppCompatActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main) // Menghubungkan ke layout XML
    }
}
// (Activity ini juga perlu dideklarasikan di AndroidManifest.xml)
```

💡 **Fun Fact:** Tahukah Anda bahwa konsep Activity adalah salah satu pilar desain Android dari awal, dirancang untuk memungkinkan aplikasi berinteraksi satu sama lain dengan mulus melalui Intent, bahkan jika mereka dibuat oleh developer yang berbeda?
