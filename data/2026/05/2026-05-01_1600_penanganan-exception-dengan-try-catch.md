## ☕ Java Concept: Penanganan Exception dengan try-catch

**Category:** Exception
**TL;DR:** Memastikan program Java tetap berjalan lancar saat terjadi kesalahan tak terduga (runtime errors) dengan menangani Exception menggunakan blok try-catch.

Dalam pemrograman Java, `Exception` adalah sebuah peristiwa yang terjadi selama eksekusi program yang mengganggu alur normal instruksi. Ini bukan kesalahan sintaks yang dicek oleh compiler, melainkan kesalahan yang muncul saat program sedang berjalan (runtime error). Bayangkan Anda meminta program untuk membaca file yang ternyata tidak ada, atau mencoba membagi angka dengan nol. Kejadian-kejadian inilah yang disebut `Exception`.

Ketika sebuah `Exception` terjadi dan tidak ditangani, program akan langsung berhenti bekerja (crash) dan menampilkan pesan error yang mungkin membingungkan pengguna. Ini tentu bukan pengalaman yang baik. Untuk menghindari hal tersebut, Java menyediakan mekanisme `try-catch`. Konsepnya adalah 'mencoba' menjalankan kode yang berpotensi menimbulkan masalah, dan 'menangkap' (catch) kesalahan jika memang terjadi.

Struktur dasar `try-catch` melibatkan dua blok utama: blok `try` dan blok `catch`. Kode yang berpotensi menimbulkan `Exception` ditempatkan di dalam blok `try`. Jika `Exception` terjadi di dalam blok `try`, eksekusi kode di blok `try` akan langsung berhenti dan kontrol akan berpindah ke blok `catch` yang sesuai. Blok `catch` bertugas untuk 'menangkap' `Exception` dari jenis tertentu dan menjalankan kode penanganan yang sudah kita definisikan. Ini bisa berupa menampilkan pesan error yang lebih user-friendly, mencatat error untuk debugging, atau bahkan mencoba memperbaiki masalah. Kita bisa memiliki lebih dari satu blok `catch` untuk menangani jenis `Exception` yang berbeda secara spesifik.

Selain `try` dan `catch`, ada juga blok `finally` yang opsional. Kode di dalam blok `finally` akan *selalu* dieksekusi, tidak peduli apakah `Exception` terjadi atau tidak, atau apakah ia ditangkap atau tidak. Ini sangat berguna untuk membersihkan sumber daya seperti menutup `Scanner` atau koneksi database.

```
import java.util.Scanner;

public class DemoException {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);
        try {
            System.out.print("Masukkan angka pertama: ");
            int num1 = Integer.parseInt(scanner.nextLine());
            System.out.print("Masukkan angka kedua: ");
            int num2 = Integer.parseInt(scanner.nextLine());
            int result = num1 / num2;
            System.out.println("Hasil pembagian: " + result);
        } catch (ArithmeticException e) {
            System.out.println("Error: Tidak bisa membagi dengan nol!");
        } catch (NumberFormatException e) {
            System.out.println("Error: Input bukan angka yang valid.");
        } finally {
            scanner.close();
            System.out.println("Program selesai, sumber daya ditutup.");
        }
    }
}
```

💡 **Fun Fact:** Konsep `Exception` di Java adalah salah satu fitur utama yang membedakannya dari bahasa-bahasa sebelumnya seperti C, yang cenderung mengandalkan kode pengembalian (return codes) untuk menandakan kesalahan. Ini membuat kode Java lebih bersih dan lebih mudah dalam penanganan error.
